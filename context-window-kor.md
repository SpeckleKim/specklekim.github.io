# 컨텍스트 윈도우와 롱 컨텍스트

## 컨텍스트 윈도우란?

컨텍스트 윈도우(또는 컨텍스트 길이)는 LLM이 한 번에 처리할 수 있는 최대 토큰 수를 정의합니다. 이는 현대 언어 모델의 가장 기본적인 제약 중 하나입니다. BERT 같은 초기 트랜스포머는 512개 토큰의 컨텍스트 윈도우를 가졌고, GPT-2는 1,024개 토큰을 도입했습니다. 오늘날의 모델은 4K에서 100만 개 토큰까지 다양하지만, 이 제한은 여전히 중요한 설계 고려사항으로 남아있습니다.

### 컨텍스트가 중요한 이유

더 큰 컨텍스트 윈도우는 다음을 가능하게 합니다:
- **문서 수준 이해**: 잘림 없이 전체 연구 논문이나 책 처리
- **장시간 대화**: 여러 번의 긴 대화 유지
- **인-컨텍스트 학습**: 퓨샷 프롬프팅을 위해 많은 예제 제공
- **향상된 요약**: 긴 시퀀스 전체의 뉘앙스 포착
- **복잡한 추론**: 더 많은 중간 단계를 작업 메모리에 유지

그러나 컨텍스트 크기는 계산 비용을 야깁니다. 어텐션 메커니즘은 시퀀스 길이에 따라 이차적으로(O(n²)) 확장되므로, 큰 컨텍스트 윈도우는 학습과 추론 모두에서 비싸집니다.

## 도전과제: 왜 단순히 컨텍스트를 늘리지 않을까?

### 계산 복잡성

표준 트랜스포머 어텐션은 모든 토큰 쌍 간의 유사성 점수를 계산해야 합니다. 100만 개 토큰의 컨텍스트에서는 어텐션 헤드당 약 10^12 연산이 필요하므로 일반 하드웨어에서는 계산 불가능합니다. 이 O(n²) 확장이 근본적인 병목입니다.

### 학습 제약

대부분의 모델은 메모리와 시간 제약으로 인해 짧은 컨텍스트(2K-4K 토큰)로 학습합니다. 학습 노출 없이 추론 시간에 더 긴 컨텍스트에서 평가하면 분포 불일치가 발생하여 성능이 저하됩니다.

### 위치 보간 문제

트랜스포머는 학습 중에 절대 위치 임베딩을 학습합니다. 학습 길이를 초과하는 컨텍스트로 확장하려고 하면 모델이 본 적 없는 위치를 만나게 됩니다. 단순한 확장은 치명적인 성능 저하를 야기합니다.

## 확장 기술

### RoPE (회전 위치 임베딩)

RoPE는 복소 공간에서 위치를 회전 각도로 인코딩하여 학습 길이를 초과하는 더 매끄러운 외삽을 가능하게 합니다. 주파수 베이스(theta)를 조정함으로써 모델은 더 긴 시퀀스로 일반화할 수 있습니다. 많은 현대 모델(LLaMA, Qwen)이 RoPE를 사용하는 이유는 간단한 수정으로 더 나은 길이 일반화를 가능하게 하기 때문입니다.

### 어텐션 변형

**선형 어텐션**: 커널화된 함수나 반복 공식을 사용하여 표준 어텐션을 선형 복잡성(O(n))으로 근사합니다. Mamba와 Liquid Transformers 같은 모델이 이 방향을 탐색합니다.

**희소 어텐션**: 근처 토큰과 전역 토큰 사이의 어텐션만 계산합니다(Longformer, BigBird). 위치 i의 토큰은 i-w부터 i+w까지의 위치(로컬 윈도우)와 선택된 전역 토큰에만 어텐션을 적용합니다.

**링 어텐션**: 긴 시퀀스를 여러 GPU에 분배하고 링으로 어텐션을 계산하여 장치당 메모리를 감소시킵니다. 클러스터에서 400만+ 토큰 학습을 가능하게 합니다.

### 랜드마크 어텐션

이 기술은 작은 "랜드마크" 토큰 세트(요약 토큰)를 선택하고 이러한 랜드마크와만 전체 어텐션을 계산하며, 다른 어텐션은 근사합니다. 이는 O(n²)를 O(nk)로 감소시킵니다(k는 작음).

## 롱 컨텍스트 모델

### 128K-100만 토큰 모델

최근 발전으로 다음을 가능하게 합니다:
- **Claude 3 (200K 컨텍스트)**: 고급 어텐션 기법과 신중한 학습 절차 사용
- **GPT-4 Turbo (128K 컨텍스트)**: 원래 ChatGPT 컨텍스트의 25배 관리
- **LLaMA 2 변형 (100K+)**: 위치 보간과 지속적인 사전학습을 통해
- **Gemini Ultra (100만 토큰)**: 컨텍스트 길이의 한계 확장

이 모델들은 기법의 조합을 구현합니다: 최적화된 어텐션 패턴, 위치 임베딩 스케일링, 종종 더 긴 시퀀스의 지속적인 사전학습.

## "중간에 잃혀버린" 문제

경험적 발견에 따르면 큰 컨텍스트 윈도우에도 불구하고 모델은 입력의 중간에 있는 정보로 어려움을 겪습니다. 20개 이상의 문서를 처리할 때 모델은 중간에 배치된 관련 정보를 무시하고 시작과 끝에 부당하게 집중할 수 있습니다.

이는 다음과 같이 발생합니다:
- **어텐션 패턴**: 모델은 사전학습 중에 주로 최근과 초기 토큰에 어텐션하도록 학습
- **최적화 역학**: 중요 정보는 경계에서 더 현저함
- **효과적 컨텍스트**: "효과적인" 사용 가능한 컨텍스트는 명목상 길이보다 훨씬 작음

완화 전략:
- 가장 중요한 정보를 처음이나 마지막에 배치
- 검색 증강(RAG)을 사용하여 관련 문서만 선택
- 요약으로 계층적 처리 구현
- 지시 튜닝을 통해 어텐션 조정

## 실제 영향

### 개발자를 위해

롱 컨텍스트 모델을 사용할 때:
- 더 긴 컨텍스트가 더 나은 성능과 같지 않음; 관련성이 더 중요
- 컨텍스트 내 정보의 위치가 품질에 영향
- 온도와 디코딩 매개변수가 조정 필요할 수 있음
- 비용 모니터링—긴 컨텍스트는 토큰 비용을 곱셈

### 트레이드오프

- **추론 속도**: 긴 컨텍스트는 토큰 생성을 선형적으로 또는 더 느리게 함
- **비용**: 컨텍스트 길이에 비례한 가격 책정으로 긴 컨텍스트가 비쌈
- **하드웨어 요구사항**: 추론도 여러 100K 컨텍스트에 대해 상당한 GPU 메모리 필요

## 미래 방향

연구는 다음을 계속합니다:
- **효율적인 어텐션**: 알고리즘 혁신을 통한 O(n²) 스케일링 감소
- **계층적 처리**: 대규모 문서를 위한 다단계 요약
- **적응형 컨텍스트**: 작업당 최적 컨텍스트 길이를 결정하는 모델
- **하이브리드 접근**: 조밀하고 희소한 어텐션을 지능적으로 결합

앞으로의 경로는 단순히 컨텍스트 길이를 무한정 확장하기보다는 이 기법들의 조합을 포함할 가능성이 높습니다. 목표는 사용 가능하고 효과적인 컨텍스트이지 단순한 토큰 수가 아닙니다.
