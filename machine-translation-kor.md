# 기계번역

## 기계번역 소개

기계번역(Machine Translation, MT)은 계산 방법을 사용하여 한 언어(원문)에서 다른 언어(대상)로 텍스트를 자동으로 번역하는 작업입니다. 기계번역은 점점 더 다언어적인 우리의 세계에서 언어 장벽을 무너뜨리고 언어 간 실시간 통신 및 정보 접근을 가능하게 하는 데 필수적입니다.

이 분야는 수십 년에 걸쳐 광범위한 인간의 노동이 필요한 초기 규칙 기반 접근 방식에서 딥 러닝과 대규모 데이터셋을 활용하는 현대 신경 방법으로 극적으로 진화했습니다. 오늘날의 신경 기계번역 시스템은 양호한 자원을 갖춘 언어 쌍의 경우 인간 수준의 번역에 가까운 놀라운 품질을 달성합니다.

## 규칙 기반 기계번역

규칙 기반 기계번역은 1980년대-1990년대의 주요 접근 방식이었습니다. 텍스트를 번역하기 위해 명시적인 언어학 규칙과 도메인별 사전에 의존합니다.

**구성 요소:**
- **형태론적 분석**: 단어를 구성 요소로 분해
- **구문 분석**: 원문의 구조 파싱
- **전이 규칙**: 원문 구조를 대상 구조로 변환
- **생성**: 대상 언어 텍스트 생성

**장점:**
- 해석 가능; 규칙이 명시적이고 수정 가능
- 도메인 전문성 통합 가능
- 명확한 규칙이 있는 기술/전문 콘텐츠에서 좋은 성능

**단점:**
- 극도로 노동 집약적; 각 언어 쌍에 대해 전문가 언어학자 필요
- 취약함; 규칙 집합 외의 현상을 만날 때 중단
- 관용구, 은유 및 문맥 의존적 현상의 처리 불량
- 새로운 언어 쌍 배포에 비용이 많이 들고 느림

## 통계적 기계번역

통계적 기계번역은 2000년대-2010년대를 지배했습니다. 병렬 코퍼스(인간이 번역한 텍스트)에서 확률 모델을 사용하여 번역 패턴을 학습합니다.

**주요 구성 요소:**

**구 기반 모델**: 원문과 대상 언어의 구 간 매핑을 학습하며 Bayes 규칙을 사용하여 P(대상 | 원문)을 모델링합니다.

**언어 모델**: 대상 언어 유창성을 점수화합니다: P(대상) ∝ exp(Σ log P(단어_i | 이전 단어))

**정렬 모델**: 원문과 대상 간의 단어 대응을 식별하여 어떤 원문 단어가 어떤 대상 단어에 정렬되는지를 캡처합니다.

**디코딩**: P(대상 | 원문)을 최대화하는 대상 수열을 검색합니다. 계산 복잡성으로 인해 빔 검색을 사용합니다.

**장점:**
- 규칙 기반 기계번역에 비해 인간의 노력 감소
- 더 간단한 모델로 백오프를 통해 보이지 않는 구문 처리
- 규칙 기반 시스템보다 더 견고함

**단점:**
- 명시적 언어학 지식 제한(별도의 언어 및 번역 모델)
- 장거리 의존성 및 재정렬 처리 어려움
- 신중한 기능 엔지니어링 필요
- 더 많은 데이터로 번역 품질 정체

## 신경 기계번역(NMT)

신경 기계번역(Neural Machine Translation, NMT)은 2014년경 도입되었으며 엔드-투-엔드 번역을 모델링하기 위해 깊은 신경망을 사용합니다. 지배적인 아키텍처는 주의가 있는 인코더-디코더입니다.

**수열-대-수열 + 주의:**

1. **인코더**: 양방향 LSTM/GRU가 원문을 처리하여 문맥 벡터를 생성
2. **주의 메커니즘**: 각 대상 토큰을 생성할 때 관련 원문 토큰에 동적으로 초점을 맞춤
3. **디코더**: 또 다른 LSTM/GRU가 주의 가중치 원문 문맥을 사용하여 대상 수열을 토큰-바이-토큰으로 생성

주의 메커니즘은 중요했습니다—디코더가 모든 원문 정보를 단일 벡터로 압축하는 대신 원문 단어로 "돌아갈" 수 있도록 허용합니다.

**장점:**
- 엔드-투-엔드 학습; 별도의 언어 및 번역 모델 없음
- 구 기반 모델보다 장거리 의존성을 더 잘 처리
- 주의 메커니즘은 해석 가능성 제공
- 단일 신경망이 복잡한 통계 파이프라인을 대체

**제한 사항:**
- 여전히 RNN에 기반함; 순차 처리는 훈련 및 추론에 느림
- 사라지는 그래디언트로 인해 매우 긴 수열에서 어려움
- 순환 아키텍처로 인해 제한된 문맥 윈도우

## 트랜스포머 기반 기계번역

"Attention is All You Need"(2017)에서 소개된 트랜스포머는 순환을 순수 주의 메커니즘으로 대체하여 NMT를 혁신했습니다.

**아키텍처:**
- **멀티헤드 주의**: 원문-대상 관계의 여러 측면 캡처
- **위치 인코딩**: 순환 없이 단어 순서 인코드
- **피드포워드 네트워크**: 위치당 적용
- **계층 스택**: 깊이를 위한 여러 인코더 및 디코더 계층

**장점:**
- 병렬 처리: RNN과 달리 모든 토큰을 동시에 처리 가능
- 장거리 의존성: 자기 주의는 먼 토큰 간의 관계를 직접 모델링
- 높은 병렬화 가능: GPU/TPU를 사용하여 대규모 데이터셋에서 효율적인 훈련 가능
- 해석 가능한 주의: 어떤 원문 단어가 각 대상 단어에 영향을 미쳤는지 시각화 가능

트랜스포머 기반 시스템은 RNN 기반 NMT에 비해 상당한 개선을 달성했으며, 산업 및 연구에서 사실상의 표준이 되었습니다.

## 다언어 기계번역 모델

현대 기계번역 시스템은 종종 단일 모델로 많은 언어 쌍을 지원합니다:

**mBART (다언어 BART)**: 50개 이상의 언어에서 잡음 제거 목표로 사전 훈련된 수열-대-수열 트랜스포머입니다. 50개 이상의 언어 쌍 간에 번역할 수 있습니다.

**NLLB (No Language Left Behind)**: Meta의 202개 언어를 지원하는 모델(2023년 기준). 저자원 언어 번역을 가능하게 하는 자기 감독 목표를 포함한 대규모 비레이블 데이터로 훈련됨.

**장점:**
- 단일 모델이 많은 언어 쌍을 제공하여 저장소 및 배포 비용 감소
- 전이 학습: 고자원 쌍의 지식이 저자원 언어를 돕기
- 직접 병렬 데이터 없이 언어 쌍 간 제로샷 번역 가능

**과제:**
- 다양한 언어 쌍 간 품질 균형
- 일부 쌍에 대한 긍정적 전이; 다른 쌍에 대한 부정적 전이
- 언어 간 간섭; 모델은 때때로 코드 스위칭(언어 혼합) 생성

## 평가 지표

기계번역 품질 평가는 모든 원문에 대해 여러 정확한 번역이 존재하기 때문에 어렵습니다.

**BLEU (양언어 평가 언더스터디):**
- 시스템 출력과 참조 번역 간의 n-그램 중복을 비교
- 점수 0-100; 높을수록 좋음
- 빠르고 언어 독립적
- 단점: 인간 판단과 완벽하게 상관되지 않음; 정확히 일치하는 것에 편향

**COMET (번역 평가의 교차언어 최적화 지표):**
- 인간 판단에서 훈련된 학습 기반 지표
- 사전 훈련된 언어 모델을 사용하여 번역 품질 추정
- BLEU보다 인간 판단과 더 잘 정렬
- 계산 자원 필요하지만 연구에서 표준이 되고 있음

**인간 평가:**
- 금본위제; 인간이 유창성 및 적절성을 평가
- 비용이 많이 들고 시간 소모적
- 릴리스 품질 시스템에 필수적

## 제로샷 기계번역

제로샷 번역은 직접 병렬 훈련 데이터 없이 언어 쌍 간에 번역합니다. 이는 다언어 모델의 중간 표현을 통해 달성됩니다.

**메커니즘:**
다언어 모델이 (영어→프랑스어) 및 (영어→독일어)에서 훈련된 경우 명시적 프랑스어-독일어 훈련 데이터 없이도 (프랑스어→독일어) 번역이 가능합니다.

**장점:**
- 희귀 언어 쌍에 대한 번역 가능
- 훈련 데이터 요구 사항 감소
- 고자원 언어 쌍을 활용하여 저자원 쌍 개선

**과제:**
- 직접 병렬 데이터로 훈련된 모델에 비해 품질 저하
- 중간 언어를 통한 오류 전파
- 피벗 언어와 원하는 쌍 간의 언어학적 차이

## 응용 프로그램 및 향후 방향

기계번역은 다음을 강화합니다:
- **실시간 통신**: 라이브 캡션, 채팅 번역
- **콘텐츠 지역화**: 웹사이트, 소프트웨어, 미디어
- **정보 접근**: 뉴스, 연구의 언어 장벽 제거
- **접근성**: 다양한 언어 사용자가 콘텐츠 사용 가능

향후 방향:
- **문서 수준 번역**: 문장이 아닌 전체 문서의 문맥
- **대화형 기계번역**: 사용자가 피드백을 제공하여 번역 개선
- **동시 번역**: 스트리밍 음성의 실시간 번역
- **다중모달 기계번역**: 이미지, 비디오 문맥 통합
- **저자원 언어 지원 개선**: 디지털 격차 해소

## 결론

기계번역은 노동 집약적 규칙 기반 시스템에서 엔드-투-엔드 신경망으로의 급진적인 변환을 겪었습니다. 다언어 사전 훈련을 포함한 현대 트랜스포머는 다양한 언어 쌍에서 놀라운 품질을 달성하며, COMET과 같은 지표는 인간 판단과 더 나은 정렬을 제공합니다. 저자원 언어, 도메인 적응 및 희귀 현상 처리에서 과제가 남아 있지만 궤적은 명확합니다: 신경 접근 방식은 계속 개선되고 번역 기술은 점점 더 접근 가능해지면서 전 세계적으로 언어 간 통신을 민주화하고 있습니다.
