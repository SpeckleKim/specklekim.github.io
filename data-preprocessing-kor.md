# 데이터 전처리와 파이프라인

## 소개

데이터 전처리는 원시의 복잡한 데이터를 ML에 적합한 깨끗한 구조화된 형식으로 변환합니다. "입력이 나쁘면 출력도 나쁘다"는 말은 현실을 반영합니다. 정교한 모델도 품질이 낮은 데이터에서는 실패합니다. 전처리는 종종 데이터 과학자의 시간의 60-80%를 차지하지만 모델 성능과 견고성에 필수적입니다.

## 데이터 정제 기초

### 누락값 처리

누락값(NULL, NaN)은 실제 데이터셋에서 흔합니다:

**전략**:
- **삭제**: 누락값이 있는 행/열 제거 (간단하지만 정보 손실)
- **대체**: 통계값으로 대체 (평균, 중앙값, 최빈값)
- **전방 이월/후방 이월**: 이전/다음 값 사용 (시계열용)
- **고급 대체**: [KNN](/knn-kor.html), 회귀, 행렬 인수분해 (관계 보존)
- **다중 대체**: 여러 완전한 데이셋 생성, 모두 분석, 결과 결합

**고려사항**:
- 완전 임의 누락(MCAR) vs 임의 누락(MAR)
- 도메인 지식: 일부 누락값은 의미가 있음 (센서 고장, 응답 없음)
- 다운스트림 분석에 미치는 영향

### 이상치 감지 및 처리

이상치는 나머지와 편차가 있는 극단값입니다:

**감지 방법**:
- **통계적**: Z-점수 (|z| > 3), IQR (1.5×IQR 밖의 값)
- **거리 기반**: Isolation Forest, Local Outlier Factor
- **모델 기반**: 오토인코더, 가우시안 혼합 모델

**처리**:
- **삭제**: 이상치 제거 (정보를 포함하면 위험)
- **상한선 설정**: 백분위수 값으로 대체 (95, 99)
- **변환**: 로그/Box-Cox로 범위 압축
- **별도 모델**: 이상치를 위한 별도 모델 생성
- **조사**: 근본 원인 이해, 데이터 품질 문제일 수 있음

## 데이터 정규화 및 인코딩

### 정규화

특성을 비교 가능한 범위로 스케일링하면 경사 하강과 [정규화](/regularization-kor.html)가 개선됩니다:

**방법**:
- **최소-최대 스케일링**: X_scaled = (X - min) / (max - min), 범위 [0, 1]
- **Z-점수 정규화**: X_scaled = (X - mean) / std, 범위 약 [-3, 3]
- **견고한 스케일링**: 중앙값과 IQR 사용, 이상치에 저항
- **로그 스케일링**: 왜곡된 분포의 경우, log(1 + X) 적용

**정규화 시기**:
- 항상: 경사 기반 방법 ([신경망](/neural-kor.html), [SVM](/svm-kor.html), 로지스틱 회귀)
- 선택적: 트리 기반 모델 (스케일 불변)
- 중요: 거리 메트릭 ([KNN](/knn-kor.html), K-means)

### 범주형 인코딩

범주형 변수를 수치로 변환:

**원-핫 인코딩**: 범주당 이진 열 생성 (트리 모델, 낮은 카디널리티용)

**레이블 인코딩**: 범주를 정수 0, 1, 2...로 매핑 (트리 모델만)

**타겟 인코딩**: 범주를 목표 변수 평균으로 대체 (유출을 방지하기 위해 [교차 검증](/cross-validation-kor.html) 필요)

**임베딩**: 범주를 위한 조밀한 벡터 학습 ([신경망](/neural-kor.html)용)

**서수 인코딩**: 순서 범주용 (작음, 중간, 큼 → 1, 2, 3)

## 데이터 파이프라인

### Airflow

Apache Airflow는 복잡한 데이터 워크플로를 조율합니다:

```python
from airflow import DAG
from airflow.operators.bash import BashOperator

dag = DAG('data_pipeline', default_args=default_args)
extract = BashOperator(task_id='extract', bash_command='python extract.py')
transform = BashOperator(task_id='transform', bash_command='python transform.py')
extract >> transform
```

기능: 방향 비순환 그래프(DAG), 스케줄링, 모니터링, 재시도, 역채우기.

### Apache Spark

분산 데이터 처리 프레임워크:
- 단일 머신 메모리보다 큰 데이셋 처리
- 구조화된 처리를 위한 RDD 및 DataFrame
- SQL, 머신러닝, 스트리밍 지원
- Hadoop MapReduce보다 10-100배 빠름

### Great Expectations

데이터 품질 보장 프레임워크:
- 기대치 정의: "열 X는 null이 아님", "Y는 [0, 100] 범위"
- 기대치에 대해 데이터 검증
- 문서용 데이터 문서 생성
- 오케스트레이션 도구와 통합

## 데이터 버전 관리

데이터 버전을 코드 버전처럼 추적하면 재현성 보장:

**도구**:
- **DVC (Data Version Control)**: 큰 데이터 파일 추적, 코드와 분리
- **Delta Lake**: 데이터 레이크용 ACID 트랜잭션
- **Git LFS**: Git용 대용량 파일 저장소

**이점**:
- 동일한 데이터로 이전 모델 버전 재현
- 버전 간 변경 사항 이해
- 필요시 이전 데이터로 롤백
- 저장소 충돌 없이 협업

## 특성 엔지니어링

원시 데이터를 의미 있는 특성으로 변환:

**접근**:
- **도메인 지식**: 비즈니스 로직에서 특성 생성
- **통계**: 함수 집계 (합, 평균, 표준편차)
- **시간적**: 시계열용 지연 특성, 이동 윈도우
- **상호 작용**: 특성 곱하기 또는 결합
- **다항식**: 제곱, 세제곱 특성 생성

**모범 사례**:
- 단순하게 시작, 필요시 복잡성 추가
- 특성 중요도 검증
- 유출 방지: 미래 정보 사용 금지
- 재현성을 위해 특성 정의 문서화

## 데이터 품질 메트릭

**완전성**: 누락되지 않은 값의 백분율

**일관성**: 데이터 유형 동의, 형식 표준화

**정확성**: 신뢰 소스 대비 정확성

**적시성**: 데이터가 얼마나 최신인가

**유일성**: 중복 레코드 비율

## 모범 사례

- **자동화**: 스크립트 > 수동 Excel 작업
- **문서화**: 가정, 변환, 임계값 기록
- **버전 관리**: 데이터 및 전처리 코드 변경 추적
- **테스트**: 기대치에 대해 데이터 검증
- **모니터링**: 프로덕션의 데이터 품질 문제 알림
- **반복**: 모델링에서 발견한 품질 문제 → 파이프라인 개선

## 결론

데이터 전처리는 지루하지만 중요한 작업입니다. 견고한 파이프라인, 검증, 버전 관리에 투자하면 모델 신뢰성과 팀 생산성에 배당금을 제공합니다. 80-20 규칙이 적용됩니다: 80% 가치는 깨끗한 잘 엔지니어된 데이터에서, 20%는 멋진 알고리즘에서 나옵니다.
