# 클러스터링 알고리즘: 레이블이 없는 데이터에서 구조 발견

클러스터링은 데이터에서 자연스러운 그룹을 발견하는 근본적인 비지도 학습 기법입니다. 이 가이드는 주요 클러스터링 알고리즘, 그 메커니즘, 평가 방법, 그리고 실무 응용을 다룹니다.

## K-평균: 가장 인기 있는 알고리즘

K-평균은 가장 간단하고 광범위하게 사용되는 클러스터링 알고리즘으로, 비슷한 크기의 구형 클러스터에 이상적입니다.

**알고리즘 단계:**
1. K개 클러스터 중심을 무작위로 초기화 (또는 K-평균++ 사용)
2. 각 점을 가장 가까운 클러스터 중심에 할당
3. 할당된 점들의 평균으로 클러스터 중심 재계산
4. 수렴할 때까지 단계 2-3 반복

**K-평균++ 초기화:**
- 첫 번째 중심을 무작위로 선택
- 나머지 중심의 경우, 가장 가까운 중심으로부터의 거리 제곱에 비례하는 점 선택
- 더 나은 초기화는 수렴을 극적으로 향상
- 국소 최솟값 문제 감소

**최적 K 찾기 (팔꿈치 방법):**
- 클러스터 내 제곱합(WCSS) 대 K를 플롯
- WCSS는 K가 증가할수록 감소 (더 많은 클러스터 = 더 작은 거리)
- 개선이 평평해지는 "팔꿈치" 지점 찾기
- 팔꿈치 너머로는 의미 있는 개선 없이 복잡성만 증가

**장점:**
- 간단하고 빠르며 확장 가능
- 대규모 데이터셋에서도 계산적으로 효율적
- 구현 및 이해가 쉬움
- 구형 클러스터에 잘 작동

**제한 사항:**
- 사전에 K를 지정해야 함
- 초기화에 민감
- 비구형 또는 크기가 다양한 클러스터로 어려움
- 이상치 및 특성 척도 조정의 영향

## DBSCAN: 밀도 기반 클러스터링

DBSCAN은 더 희소한 영역으로 분리된 밀집 영역을 식별하여 임의 모양과 크기의 클러스터를 발견합니다.

**핵심 개념:**
- **Eps (ε):** 두 점 간 이웃으로 간주되는 최대 거리
- **MinPts:** 클러스터를 형성하기 위해 엡실론 이웃에서 필요한 최소 점 개수
- **핵심 점:** 엡실론 내에서 최소 MinPts 이웃을 가진 점
- **경계 점:** 핵심 점의 엡실론 내의 비핵심 점
- **잡음 점:** 어떤 클러스터에도 속하지 않는 점

**알고리즘:**
1. 모든 핵심 점 찾기
2. 엡실론 거리 내의 핵심 점 연결
3. 연결된 핵심 점에서 클러스터 형성
4. 경계 점을 근처 핵심 점 클러스터에 할당
5. 나머지 점을 잡음으로 표시

**주요 장점:**
- 임의 모양의 클러스터 발견
- 클러스터 개수 지정 필요 없음
- 이상치에 견고함
- 잡음 점을 명시적으로 식별
- 모든 거리 메트릭과 작동

**실무 고려사항:**
- 엡실론 및 minPts 파라미터에 민감
- 다양한 밀도의 클러스터로 어려움
- 높은 차원에서 계산 비용이 많음
- 파라미터 튜닝에 도메인 지식 필요

**Eps 및 MinPts 찾기:**
- K-거리 그래프: K번째 최근 이웃까지의 거리 플롯
- 곡선의 "무릎" 찾기
- 최소 클러스터 크기에 대한 도메인 지식 사용
- minPts = 2 × 차원으로 시작

## 계층적 클러스터링: 덴드로그램 구축

계층적 클러스터링은 다양한 규모에서 중첩된 클러스터를 보여주는 나무 같은 구조(덴드로그램)를 만듭니다.

**응집형 (상향식):**
1. 각 점을 자신의 클러스터로 시작
2. 두 개의 가장 가까운 클러스터를 반복적으로 병합
3. 단일 클러스터가 남을 때까지 계속
4. 덴드로그램은 병합 순서 및 거리 표시

**분할형 (하향식):**
1. 모든 점이 한 클러스터에서 시작
2. 클러스터를 반복적으로 분할
3. 덜 일반적이지만 계산적으로 효율적일 수 있음
4. 계층적 데이터 구조에 더 나음

**연결 방법:**
- **단일 연결:** 가장 가까운 점 간 거리 (연쇄 경향)
- **완전 연결:** 가장 먼 점 간 거리 (컴팩트 클러스터 선호)
- **평균 연결:** 모든 쌍 간 평균 거리 (균형 잡힌 접근)
- **Ward 방법:** 클러스터 내 분산 최소화 (가장 일반적)

**덴드로그램 해석:**
- 가로축: 데이터 점 또는 클러스터
- 세로축: 병합 거리
- 연결 높이: 병합된 클러스터 간 거리
- 다양한 높이에서 자르기 = 다양한 클러스터 개수

**장점:**
- 클러스터 관계의 계층적 보기 제공
- 사전에 클러스터 개수 지정 불필요
- 유연함: 모든 높이에서 덴드로그램 자르기 가능
- 결정적 (K-평균처럼 무작위성 없음)

**제한 사항:**
- 계산 비용이 많음: O(n²) 또는 O(n³)
- 내장된 목적 최적화 없음
- 잡음 및 이상치에 민감
- 클러스터가 병합되면 되돌리기 어려움

## 가우시안 혼합 모델 (GMM)

GMM은 데이터가 가우시안 분포의 혼합에서 나온다고 가정하는 확률적 접근입니다.

**핵심 개념:**
- 각 클러스터는 가우시안 분포
- 부드러운 할당: 각 점은 각 클러스터에 속할 확률을 가짐
- EM 알고리즘: 파라미터 추정을 위한 기댓값-최대화
- BIC 또는 AIC를 사용하여 클러스터 개수 추정 가능

**EM 알고리즘:**
- **E-단계:** 각 클러스터에서 각 점의 확률 계산
- **M-단계:** 확률에 기반한 클러스터 파라미터 업데이트
- 수렴할 때까지 반복

**장점:**
- 확률적 프레임워크 (클러스터 멤버십 확률)
- 부드러운 클러스터링으로 불확실성 추정 제공
- 잘 정립된 통계 이론
- 타원형 클러스터 모양 모델링 가능

**제한 사항:**
- 가우시안 분포를 가정 (실제 데이터에 맞지 않을 수 있음)
- 계산 집약적
- 초기화에 민감
- 수렴이 느릴 수 있음

## 평가 지표

**실루엣 점수:**
- 점이 자신의 클러스터와 다른 클러스터에 얼마나 유사한지 측정
- 범위: -1에서 1 (높을수록 좋음)
- 점수 ≥ 0.5는 좋은 클러스터링 나타냄
- 실제 레이블 없이 계산 가능

**Davies-Bouldin 지수:**
- 클러스터와 가장 유사한 이웃 간 평균 유사도
- 낮은 값은 더 나은 클러스터링 나타냄
- 크거나 매우 가까운 클러스터에 페널티

**Calinski-Harabasz 지수:**
- 클러스터 간 대 클러스터 내 분산의 비
- 높은 값은 더 나은 분리 나타냄
- 계산적으로 효율적

**동질성, 완전성, V-측정:**
- 실제 레이블 필요
- 동질성: 각 클러스터는 같은 클래스의 점만 포함
- 완전성: 같은 클래스의 모든 점이 같은 클러스터에
- V-측정: 둘의 조화 평균

## 실무 응용

**고객 분할:**
- 행동 그룹에 K-평균
- 표적화된 마케팅 전략 가능
- 고가치 고객 세그먼트 식별

**문서/주제 클러스터링:**
- 자동으로 유사 문서 그룹화
- 레이블 없이 문서 주제 발견
- 추천 시스템의 기초

**유전자 발현 분석:**
- 유사하게 행동하는 유전자 그룹 식별
- 질병 부타입 발견
- 생물학에 유용한 계층적 클러스터링

**이상 탐지:**
- DBSCAN이 이상치를 자연스럽게 식별
- 어떤 클러스터에서 멀리 떨어진 점은 이상치
- 레이블이 지정된 이상치 불필요

**이미지 분할:**
- 픽셀을 의미론적 영역으로 그룹화
- 색상 양자화에 K-평균
- 객체 탐지에 DBSCAN

## 올바른 알고리즘 선택하기

**K-평균 사용 시:**
- 클러스터가 대략 구형
- 계산 효율성이 중요
- 클러스터 개수를 알거나 추정 가능
- 대규모 및 고차원 데이터셋

**DBSCAN 사용 시:**
- 클러스터가 임의 모양
- 밀도 변동이 존재
- 클러스터 개수 미지수
- 잡음 점을 명시적으로 식별 필요

**계층적 클러스터링 사용 시:**
- 계층적 관계가 의미 있음
- 소규모에서 중간 크기 데이터셋
- 클러스터 세분도 유연성 필요
- 해석 가능성이 중요

**GMM 사용 시:**
- 확률적 프레임워크 원함
- 부드러운 클러스터링 할당 필요
- 타원형 클러스터 모양 예상
- 통계적 엄밀성 중요

## 결론

효과적인 클러스터링은 데이터 특성을 이해하고 적절한 알고리즘을 선택하는 것이 필요합니다. K-평균은 빠르고 간단한 클러스터링을 제공하고, DBSCAN은 임의 모양을 처리하며, 계층적 방법은 관계를 보여주고, GMM은 확률적 통찰력을 제공합니다. 알고리즘 이해, 도메인 지식, 적절한 평가 지표, 탐색적 분석을 결합하여 데이터에서 의미 있는 구조를 발견하세요.
