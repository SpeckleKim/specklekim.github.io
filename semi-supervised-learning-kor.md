# 반지도학습: 레이블 없는 데이터 활용하기

반지도학습은 지도학습과 비지도학습 사이에 있으며, 적은 양의 레이블 있는 데이터와 대량의 레이블 없는 데이터를 결합합니다. 실제로 이것은 가장 관련성 높은 학습 패러다임인데, 레이블 획득이 비용이 크면서 레이블 없는 데이터는 풍부하기 때문입니다.

## 레이블 부족 문제

실제 응용은 중요한 도전에 직면합니다: **레이블 부족**. 의료 영상, 자율주행, 콘텐츠 검토는 비싼 전문가 주석을 필요로 합니다. 한편 우리는 모델에 정보를 줄 수 있는 대량의 레이블 없는 데이터—이미지, 텍스트, 센서 데이터—에 접근합니다.

반지도학습은 이 병목을 해결하기 위해 레이블 없는 데이터를 사용하여 지도학습만으로는 달성할 수 없는 수준 이상으로 모델 성능을 개선합니다. 핵심 가정: 레이블 없는 데이터의 구조가 명시적 레이블 없이도 분류를 돕는 패턴을 드러냅니다.

## 자기학습 (Self-Training)

자기학습은 모델의 예측으로 레이블 없는 데이터에 반복적으로 레이블을 붙입니다.

**과정:**
1. 작은 레이블 있는 데이터셋에서 모델 훈련
2. 모델을 사용하여 레이블 없는 데이터 예측
3. 높은 확신도 예측 선택(확신도 임계값)
4. 유사 레이블 예시를 훈련 세트에 추가
5. 확장된 데이터셋에서 모델 재훈련
6. 수렴 또는 충분한 유사 레이블 획득까지 2-5 반복

**강점:** 단순하고 직관적이며, 모든 기본 분류기와 작동합니다.

**약점:** 확증 편향—초기 모델이 잘못되면 유사 레이블 예시의 오류를 강화합니다. 반복에 따른 오류 누적이 성능을 저하시킬 수 있습니다.

**완화:** 확신도 필터링, 주기적 수동 검증, 앙상블 예측 사용.

## 협력학습 (Co-Training)

협력학습은 데이터의 다른 특성 부분집합이나 뷰에서 훈련된 여러 학습기를 사용합니다.

**과정:**
1. 특성을 조건부 독립 부분집합으로 분할
2. 각 뷰에서 하나씩 두 분류기 훈련
3. 각 분류기가 레이블 없는 데이터에서 높은 확신도 예측 수행
4. 분류기들 간에 이러한 유사 레이블 예시 교환
5. 유사 레이블 데이터로 두 분류기 재훈련
6. 수렴까지 반복

**핵심 아이디어:** 다른 뷰는 다양한 정보를 제공합니다. 분류기 간 불일치는 불확실한 예시를 표시하고, 일치는 확신을 제공합니다.

**예시:** 텍스트 분류의 경우, 한 분류기는 단어 특성을 사용하고 다른 하나는 구문 구조를 사용할 수 있습니다.

**장점:** 단일 모델 편향을 줄이고, 데이터 중복성을 활용합니다.

**한계:** 특성 독립성 가정, 의미 있는 특성 분할 찾기 필요.

## 그래프 기반 방법

그래프 기반 접근은 데이터를 그래프의 노드로 취급하고, 간선은 유사성을 나타냅니다. 레이블이 레이블 있는 노드에서 레이블 없는 노드로 그래프를 통해 전파됩니다.

### 레이블 전파
레이블 전파는 레이블 없는 예시에 가장 영향력 있는 이웃의 클래스를 할당하며, 간선 강도로 가중화됩니다.

**알고리즘:** 레이블 있는 노드 값으로 그래프 초기화, 간선을 통해 반복적으로 레이블 전파, 레이블 수렴 시 중지.

**직관:** 유사한 노드는 유사한 레이블을 가져야 합니다. 전파는 일관성을 강제합니다.

**장점:** 단순하고 수학적으로 우아하며, 데이터 기하학을 존중합니다.

**도전:** 그래프 구성에 민감하고, 큰 그래프에는 계산 비용이 많습니다.

### 조화장과 포텐셜 함수
레이블 전파를 그래프에서 라플라스 방정식 풀기로 모델링—레이블 없는 노드의 레이블은 레이블 있는 노드 레이블의 조화 함수입니다.

**수학적 기초:** 부드러움과 레이블 적합을 동시에 인코딩하는 에너지 함수 최소화.

## 유사 레이블 지정 (Pseudo-Labeling)

유사 레이블 지정은 예측된 레이블을 레이블 없는 예시에 직접 할당하고 결합된 세트에서 모델을 재훈련합니다.

**접근:**
1. 레이블 있는 데이터에서 훈련
2. 모든 레이블 없는 데이터에 레이블 예측
3. 원본 레이블 있는 데이터를 모든 유사 레이블 데이터와 결합
4. 결합된 데이터셋에서 재훈련

**자기학습과의 차이:** 자기학습은 확신도로 필터링하고, 유사 레이블 지정은 모든 예측을 포함합니다.

**이점:** 최대 데이터 활용, 성능 향상 가능.

**위험:** 저확신도 오류 전파, 신중한 임계값 선택 필요.

**현대적 개선:** 모델 불확실성 추정이나 온도 기반 필터링 사용.

## 일관성 정규화 (Consistency Regularization)

일관성 정규화는 데이터 섭동 하에서 예측이 안정적이도록 강제합니다. 레이블 없는 예시를 약간 섭동하면(노이즈 추가, 증강 적용), 모델은 동일한 예측을 해야 합니다.

**수학적 형태:** 레이블 있는 데이터의 지도 손실 + 레이블 없는 데이터의 일관성 손실 최소화:

손실 = L_레이블 + λ * L_일관성

L_일관성은 섭동 하에서 예측 차이를 측정합니다.

**직관:** 레이블 없는 데이터가 의사결정 경계를 제약합니다. 예측은 작은 입력 변화에 견고해야 합니다.

**실제 구현:**
- 레이블 없는 예시에 증강 적용
- 원본과 증강된 버전 모두에서 예측
- 예측 간 발산 최소화(KL 발산, 평균제곱 차)

**강력함:** 다양체 가정을 활용—의사결정 경계는 저밀도 지역에 있어야 합니다.

## MixMatch

MixMatch는 여러 기법을 결합한 최첨단 반지도 알고리즘입니다.

**성분:**
1. **일관성 정규화:** 레이블 없는 예시 증강, 예측 일관성 강제
2. **엔트로피 최소화:** 모델이 확신 있는 예측 하도록 권장
3. **MixUp:** 특성 공간에서 레이블 있는 예시와 레이블 없는 예시 혼합, 목표 선형 보간

**MixUp 연산:** 예시 x와 x'에 대해 파라미터 λ로 혼합:
x_혼합 = λx + (1-λ)x'
y_혼합 = λy + (1-λ)y'

**장점:** 상보적 기법 결합, 우수한 반지도 결과 달성.

**계산 비용:** 여러 증강과 순전파 필요.

## FixMatch

FixMatch는 반지도 학습을 하나의 직관적 아이디어로 단순화합니다: 약한 증강에서의 모델 예측이 강한 증강에서의 유사 레이블과 일치해야 합니다.

**알고리즘:**
1. 약한 증강(뒤집기, 이동)—진정한 레이블 보존
2. 강한 증강(RandAugment, CutOut 등)—진정한 레이블 변경 가능
3. 모델이 약한 증강에서 예측
4. 강한 증강 예측에서 유사 레이블
5. 손실 = 레이블 있는 데이터의 지도 손실 + 예측 간 일관성 손실

**통찰:** 강한 증강이 표면적 단서를 제거하여 모델이 의미론적 특성에 의존하게 강제.

**우아함:** 단순하고 해석 가능하며 매우 효과적.

**확장성:** 효율적이며, 벤치마크 데이터셋에서 복잡한 방법을 능가합니다.

## 응용

**의료 영상:** 제한된 전문가 주석 이미지와 병원 아카이브 결합. 반지도 학습이 적절한 레이블 있는 데이터로 진단 시스템 가능.

**콘텐츠 검제:** 소수 수동 검토 사례가 수백만 자동 검토로 훈련된 모델 안내.

**자율주행:** 제한된 주석 운전 시퀀스가 대량 미가공 센서 데이터와 결합.

**자연어 처리:** 적절한 레이블 데이터가 광대한 웹 텍스트와 결합.

**음성 인식:** 소수 전사된 오디오가 녹음되지만 전사되지 않은 음성과 결합.

## 반지도학습이 중요한 이유

데이터셋이 커지고 주석이 비효율적이 되면서 반지도학습이 필수가 됩니다. 이는 현실적 제약을 반영합니다: 우리는 종종 신호는 있지만 정답이 없습니다. 레이블 없는 데이터 구조를 활용하여 반지도 방법이 이전에는 완전한 주석이 필요했던 성능 향상을 해금합니다.

단순한 자기학습에서 MixMatch, FixMatch 같은 정교한 방법으로의 진화는 분야의 성숙을 보여줍니다. 이들 기법은 이제 실제 레이블 부족을 다루는 실질적 ML 시스템의 기초를 형성합니다.
