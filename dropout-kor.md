# 드롭아웃과 딥러닝 정규화

드롭아웃은 딥러닝에서 가장 간단하면서도 효과적인 정규화 기법입니다. Hinton 등이 도입한 이 기법은 훈련 중 임의로 유닛을 비활성화하여 신경원의 공동 적응을 방지하고, 네트워크가 중복된 표현을 학습하도록 강제합니다.

## 드롭아웃 메커니즘

훈련 중에는 드롭아웃이 일정 비율(p)의 활성화를 확률 p로 무작위로 0으로 설정합니다:

```
y = x * mask,  mask ~ Bernoulli(1-p)
```

핵심 통찰력: 네트워크는 분산된 표현을 학습하도록 강제되어 어떤 단일 신경원도 지나치게 특화되지 않습니다.

## 훈련과 추론

**훈련 단계:**
- 확률 p로 유닛을 임의로 제거
- 반복마다 효과적 네트워크 크기 변함
- 부분 네트워크의 암시적 앙상블 생성
- 신중한 스케일링 필요 (보통 1/(1-p)로 스케일)

**추론 단계:**
- 드롭아웃을 완전히 비활성화하거나
- 역방향 드롭아웃: 훈련 중 (1-p)로 곱하여 테스트 시 재스케일 불필요
- 모든 신경원을 변경되지 않은 가중치로 사용
- 단일 순전파에서 결정론적 예측

역방향 드롭아웃이 현대 표준입니다:
```python
# 훈련
y = (x * mask) / (1 - p)  mask ~ Bernoulli(1-p)

# 추론
y = x  (변경 없음)
```

## 드롭아웃 비율 선택

올바른 드롭아웃 비율 선택은 매우 중요합니다:

- **입력층**: 보통 0.2 (입력 특성의 20% 제거)
- **은닉층**: 보통 0.5 (은닉 유닛의 50% 제거)
- **출력층**: 절대 드롭아웃 사용 금지
- **작업 의존**: 모델 용량과 데이터셋 크기에 따라 증가

높은 드롭아웃 비율:
- 과소적합, 불충분한 학습
- 정규화에도 불구하고 약한 최종 모델

낮은 드롭아웃 비율:
- 불충분한 정규화
- 과적합 지속

## 공간적 드롭아웃

표준 드롭아웃은 각 요소를 독립적으로 처리합니다. 공간적으로 상관된 데이터를 처리하는 합성곱 신경망의 경우, 공간적 드롭아웃은 전체 특성 맵을 제거합니다:

```
형태: (배치, 높이, 너비, 채널)
(높이, 너비) 전체에서 전체 채널 제거
각 채널 내 공간 구조 보존
```

장점:
- 공간 정보 구조 보존
- CNN에 더 효과적인 정규화
- 공간적으로 인접한 특성의 공동 적응 방지

## DropConnect

활성화 대신 가중치를 제거하는 표준 드롭아웃의 대안:

```
y = f(x * (W ⊙ mask))  mask ~ Bernoulli(1-p) (가중치 상)
```

특성:
- 다른 효과적 네트워크 위상 생성
- 드롭아웃과 유사한 정규화 효과
- 가중치에 대한 L2 정규화로 볼 수 있음
- 표준 드롭아웃보다 계산량이 많음

## DropBlock

합성곱 신경망 전용으로 설계되었으며, 개별 유닛이 아닌 연속된 영역을 제거합니다:

```
block_size × block_size 크기의 블록 제거
예약된 제거: 훈련 중 p 증가
일부 작업에서 공간적 드롭아웃보다 더 나은 성능
```

장점:
- 남은 특성을 통한 정보 누수 방지
- 컴퓨터 비전 작업에서 더 나은 성능
- 임의 유닛이 아닌 의미적 정보 블록 제거

## R-Drop (정규화된 드롭아웃)

훈련과 추론 중 모두 드롭아웃을 적용하는 최근 기법:

```
L = L_CE(y₁, y_target) + L_CE(y₂, y_target) + λ * KL(y₁ || y₂)
y₁, y₂는 다른 드롭아웃 마스크로 계산
```

장점:
- 모델 강건성 개선
- 분포 외 샘플에 더 나은 일반화
- 더 안정적인 예측
- 특히 NLP 작업에서 효과적

## 정규화 앙상블 해석

핵심 통찰력: 드롭아웃은 지수적으로 많은 부분 네트워크의 앙상블을 훈련하는 것으로 해석될 수 있습니다:

- 각 순전파는 다른 네트워크 샘플링
- 가능한 부분 네트워크 수: 2^n (n = 유닛 수)
- 최종 예측: 모든 부분 네트워크의 앙상블 평균
- 예측 불확실성: 앙상블 의견 불일치 측정

이 앙상블 관점은 드롭아웃의 효과를 설명합니다:
- 앙상블 평균화를 통한 분산 감소
- 학습된 표현의 다양성 강제
- 입력 변동에 더 강건한 모델

## 실무 고려사항

**드롭아웃 배치:**
- 활성화 함수 후, 다음 층 전
- 보통 입력층 경계에는 배치하지 않음
- 완전 연결 네트워크의 밀집층 후
- CNN의 풀링 후

**다른 기법과의 상호작용:**
- 배치 정규화: 정확한 배치에 대한 논의 있음
- 학습률: 드롭아웃은 약간 더 높은 학습률 필요 가능
- 모델 용량: 더 큰 모델은 드롭아웃의 이점을 더 얻음
- 데이터 증강: 드롭아웃은 다른 정규화를 보완

## 드롭아웃 변형

| 변형 | 대상 | 최적 | 성질 |
|------|------|------|------|
| 표준 | 활성화 | 밀집 네트워크 | 간단, 효과적 |
| 공간적 | 특성 맵 | CNN | 지역성 보존 |
| DropConnect | 가중치 | 모든 아키텍처 | L2 정규화 유사 |
| DropBlock | 영역 | 컴퓨터 비전 | 의미적 블록 |
| R-Drop | 추론 | NLP, 강건 모델 | 앙상블 일관성 |

## 구현 세부사항

올바른 드롭아웃 구현:
```python
# 현대 실무 (역방향 드롭아웃)
if training:
    output = dropout(input, p=dropout_rate) / (1 - dropout_rate)
else:
    output = input
```

일반적 실수:
- 추론에서 드롭아웃 비활성화 잊음
- 스케일링 없이 표준 드롭아웃 사용
- 예측/출력층에 드롭아웃 적용
- 아키텍처 유형에 맞지 않는 드롭아웃 비율

## 드롭아웃과 정규화 강도

드롭아웃은 암시적 정규화 제공:
- 확률론적 정규화 방법으로 작동
- 특정 조건에서 L2 정규화와 동등
- 명시적 L1/L2 페널티 대체 가능
- 명시적 정규화와 함께 잘 작동

## 드롭아웃 사용 시점

드롭아웃이 가장 효과적:
- 모델 용량 대비 훈련 데이터 제한적
- 다른 조치에도 불구하고 과적합 발생
- 수천 개의 매개변수를 가진 모델
- 데이터셋 크기 < 모델 매개변수 * 100

덜 중요:
- 매우 크고 다양한 데이터셋
- 강한 데이터 증강 사용
- 다른 정규화(배치 정규화, L1/L2)
- 이미 용량 제한된 모델

## 결론

드롭아웃은 단순성, 효과, 이론적 우아함으로 인해 주요 정규화 기법으로 남아 있습니다. 앙상블 해석을 이해하면 훈련 중 신경원을 무작위로 비활성화하는 것이 더 나은 일반화로 이어지는 이유에 대한 통찰력을 얻을 수 있습니다. R-Drop과 DropBlock 같은 현대 변형은 기본 메커니즘을 확장하여 특정 아키텍처 요구사항을 처리하므로, 드롭아웃을 현대 딥러닝 과제에 적응 가능하게 만듭니다.

성공적인 드롭아웃의 핵심은 다양한 층에 적절한 비율을 선택하고, 훈련 정확도를 더 나은 일반화로 바꾼다는 것을 이해하는 것입니다—대부분의 딥러닝 응용에 가치 있는 교환입니다.
