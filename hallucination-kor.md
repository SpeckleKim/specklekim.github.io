# LLM 환각 현상

## 환각이란?

대규모 언어 모델에서 **환각**은 모델이 그럴듯해 보이지만 거짓이거나 조작되었거나 근거 없는 정보를 생성하는 현상을 의미합니다. 모델은 일관성 있고 잘 형성된 것처럼 보이지만 현실에 부합하지 않거나, 사실을 위반하거나, 제공된 컨텍스트와 모순되는 텍스트를 생성합니다.

자신의 지식 한계를 아는 인간과 달리 LLM은 자신이 알거나 모르는 것에 대해 명시적인 추론이 부족합니다. 그들은 사실 정확성이 아니라 예측 확률을 최적화하므로 신중한 완화 없이는 환각이 불가피합니다.

### 중요한 이유

환각은 AI 시스템에 대한 신뢰를 훼손합니다. 존재하지 않는 논문을 자신 있게 인용하거나 통계를 발명하거나 거짓 생물학적 세부 정보를 주장하는 모델은 중요한 애플리케이션에 대해 신뢰할 수 없습니다. 환각을 이해하는 것은 책임감 있는 배포에 필수입니다.

## 환각의 유형

연구자들은 환각을 두 가지 주요 유형으로 분류하지만 구분이 항상 명확하지는 않습니다.

### 내재적 환각

모델이 **입력 컨텍스트와 모순되는** 텍스트를 생성합니다. 정보가 주어졌음에도 불구하고 모델은 이를 무시하거나 모순합니다.

**예시:**
- 주어짐: "John은 1985년생 소프트웨어 엔지니어입니다"
- 환각: "John은 1990년생 의사입니다"
- 주어짐: "회사는 50명의 직원을 가지고 있습니다"
- 환각: "회사는 200명의 직원을 가지고 있습니다"

이는 더 명백한 실패입니다—모델은 관련 정보가 있었지만 올바르게 사용하지 않았습니다.

### 외재적 환각

모델이 제공된 입력과 직접 모순하지는 않지만 세계 지식에 대해 **검증할 수 없거나 거짓인** 텍스트를 생성합니다.

**예시:**
- "Dr. Sarah Chen은 2022년 Nature 잡지에 양자 컴퓨팅 논문을 발표했습니다" (사람/논문이 존재하지 않을 수 있음)
- "Freedonia의 수도는 New Libertyville입니다" (Freedonia는 허구)
- "GPT-8은 2024년 3월에 출시되었습니다" (AI 모델에 대한 거짓 주장)

외재적 환각은 모델이 명시적으로 컨텍스트를 위반하지 않기 때문에 감지하기 더 어렵습니다—더 넓은 지식 격차나 오해에서 조작합니다.

## 환각이 발생하는 이유

### 학습 목표 불일치

언어 모델은 **다음 토큰 예측 확률**을 최적화하며 사실 정확성을 최적화하지 않습니다. 모델은 유창한 텍스트를 생성하는 패턴을 학습하지만 유창성 ≠ 진실. 존재하지 않는 엔티티에 대한 일관성 있는 문장은 실제 세계 지식을 요구하지 않고 통계 패턴으로 높은 확률을 달성합니다.

### 지식 격차와 혼동

인터넷 데이터로 학습한 모델은 내재적 불확실성에 직면합니다:
- 학습 데이터의 불완전하거나 충돌하는 정보
- 학습 예제가 제한된 드문 엔티티
- 학습 데이터가 명확히 구분하지 않은 세부적 구분
- 유사하게 들리는 엔티티 또는 개념 간 혼동

### 디코딩 효과

텍스트 생성은 순차 프로세스입니다. 초기 토큰은 나중 토큰을 제약합니다. 모델이 초기 약속을 하면(예: "연구자가 발견했습니다..."), 일관성을 유지하기 위해 일관성을 유지하기 위해 조작된 세부 정보로 따라할 수 있으며, 이러한 세부 정보는 거짓일 수 있습니다.

### 도메인 특정 문제

- **긴 컨텍스트**: 모델은 이전에 제공된 정보를 추적하지 못할 수 있음
- **수치 추론**: 정확한 수 계산은 언어 모델의 강점이 아님
- **드문 사실**: 몇 개의 학습 예제에만 나타나는 정보는 오류 경향
- **추론 체인**: 다단계 추론은 오류를 증폭시킵니다

## 환각 감지

### 표면 수준 감지

일부 환각은 명백합니다:
- **출력 내 모순**: 모델이 X를 말한 후 X 아님을 말함
- **입력과의 사실 불일치**: 명확하게 제공된 사실과 모순
- **내부 논리적 불일치**: 진술이 추론 규칙을 위반

이들은 기본 일관성 검사로 감지 가능합니다.

### 사실성 메트릭

연구는 환각을 측정하는 메트릭을 개발했습니다:

**ROUGE/BLEU 변형**: 생성된 텍스트를 참조 요약과 비교합니다. 높은 유사성은 더 낮은 환각을 시사하지만 좋은 참조 데이터가 존재한다고 가정합니다.

**의미 유사성**: 임베딩 모델을 사용하여 생성된 주장이 참조 지식 기반과 정렬되는지 확인합니다.

**QA 기반 평가**: 생성된 진술에 대해 "신뢰할 수 있는 출처에서 학습한 모델이 이것을 대답할 수 있을까요?"를 묻습니다. 성능이 낮은 모델은 환각을 나타냅니다.

**함의 감지**: 출력이 입력에서 논리적으로 따르나요? 텍스트 함의 모델은 이를 평가할 수 있습니다.

### 인간 평가

금 표준: 인간이 생성된 사실을 검증하도록 합니다. 비싸지만 가장 신뢰할 수 있습니다. 종종 메트릭과 결합되어 평가할 출력을 우선순위화합니다.

## 완화 전략

### 검색 증강 생성 (RAG)

모델의 내부 지식에만 의존하기보다는 **외부 검색으로 생성을 증강합니다**.

1. 쿼리에 대해 지식 기반에서 관련 문서를 검색
2. 이 문서를 모델에 컨텍스트로 제공
3. 모델은 검색된 정보에 근거한 답변 생성
4. 모델은 직접 출처 인용 가능

**장점**: 사실적 근거를 제공하여 외재적 환각을 직접 처리합니다.

**제한**: 검색 품질에 따라 다릅니다. 나쁜 검색 → 나쁜 근거. 여전히 환각을 완전히 제거하지 않습니다.

### 검증 체인

모델이 자체 출력을 검증하도록 합니다:
1. 초기 응답 생성
2. 응답에서 주장 나열
3. 입력이나 지식에 대해 각 주장 검증
4. 필요한 경우 응답 수정

이는 정확성에 대한 명시적 추론을 강제하고 일부 환각을 포착할 수 있지만 모델은 생성과 검증 모두에 일관되게 실패할 수 있습니다.

### 헌법적 AI와 원칙

Anthropic의 접근: 모델을 원칙으로 학습합니다(예: "도움이 되고, 무해하고, 정직하십시오") 그리고 출력이 이 원칙을 위반하는지 추론하도록 합니다. 비평 단계는 생성 전에 출력을 판단하여 환각을 감소시킵니다.

### 고품질 데이터 미세 조정

깨끗하고 잘 주석처리된 데이터로 학습한 모델은 덜 환각합니다. 하지만 확장과 데이터 품질은 영구적인 문제입니다.

### 형식을 통한 근거

생성을 특정 형식으로 제약합니다:
- 모든 주장과 함께 인용 필요
- 검증하기 더 쉬운 구조화된 출력(JSON) 사용
- 모델이 불확실할 때 "모르겠습니다"라고 말하도록 강제

명시적 근거는 환각을 더 어렵게 하고 실수를 더 명백하게 합니다.

## 신실성 메트릭

연구자들은 **신실성** 측정을 위한 메트릭을 제안했습니다—생성된 텍스트가 출처 자료의 사실을 얼마나 잘 반영하는지.

### 원자적 사실 추출

텍스트를 원자적 사실 주장으로 분류한 후 각각 검증합니다:
1. "환자는 당뇨병이 있습니다" → 원자적 사실 포함 "질병 있음: 당뇨병"
2. 각 원자적 사실을 출처에 대해 확인
3. 환각률 = (거짓 사실) / (전체 사실)

이는 전체 텍스트 평가보다 더 세분화됩니다.

### 토큰 수준 속성

고급 접근: 각 생성된 토큰에 속성을 할당합니다—어떤 출처 토큰에 의존하나요? 명확한 속성이 없는 토큰은 환각을 나타낼 수 있습니다.

### 의미 NLI

Natural Language Inference (NLI) 모델을 사용하여 출력의 주장이 입력으로 함의되는지 확인합니다. 출력이 입력과 모순되거나 지원되지 않으면 플래그합니다.

## 과제 및 트레이드오프

### 지식 문제

모델은 유용하기 위해 넓은 지식이 필요하지만 넓은 지식은 환각을 포함합니다. 지식 제약(RAG를 통해)은 도움이 되지만 모델을 덜 유연하게 만듭니다.

### 불확실성 정량화

모델은 자신 있게 환각합니다. 그들은 "나는 확신하고 맞습니다"와 "나는 확신하고 틀렸습니다"를 구분하지 않습니다. 불확실성을 표현하는 것을 배우기는 어렵고 열린 연구 문제입니다.

### 환각 측정

단일 메트릭이 모든 측면을 포착하지 않습니다. 평가는 종종 작업 특정입니다. 요약 메트릭은 대화, 번역, 또는 코드 생성에는 작동하지 않을 수 있습니다.

### 정직성 격차

미세 조정에도 불구하고 모델은 환각을 덜 할 수 있지만 여전히 지식 경계에 대한 진정한 정직성이 부족합니다. 그들은 "모르겠습니다"라고 말하는 패턴을 학습하지만 불확실성을 진정으로 이해하지는 않습니다.

## 실무자를 위한 모범 사례

1. **올바른 도구 선택**: 사실성이 중요한 경우 모델 지식에만 의존하기보다는 신뢰할 수 있는 출처와 함께 RAG를 사용합니다.

2. **기대 설정**: 사용자는 모델이 환각할 수 있다는 것을 이해해야 합니다. 적절한 면책사항이 중요합니다.

3. **고위험 결정 검증**: 의료, 법률, 재무 또는 안전 관련 애플리케이션의 경우 항상 생성된 정보를 신뢰할 수 있는 출처와 비교하여 검증합니다.

4. **불확실성 표시기 사용**: 모델에 신뢰도를 표현하도록 요청합니다. 낮은 신뢰도는 검증이 필요함을 시사합니다.

5. **검색과 결합**: 창의적인 작업의 경우에도 문서의 근거는 환각을 감소시킵니다.

6. **모니터링 및 반복**: 배포에서 환각률을 추적합니다. 피드백을 사용하여 미세 조정하거나 프롬프팅 전략을 조정합니다.

환각은 활발한 연구 영역입니다. 완화 전략이 존재하지만 완전히 제거하는 것은 여전히 열린 문제입니다.
