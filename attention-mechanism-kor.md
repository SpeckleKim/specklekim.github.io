# 어텐션 메커니즘: 디코더가 외면할 수 없는 이유

어텐션 메커니즘은 깊은 학습에서 가장 중요한 혁신 중 하나를 나타내며, [시퀀스-투-시퀀스 모델](/seq2seq-kor.html)에서 현대 [트랜스포머](/llm-kor.html) 시대로 NLP를 변환했습니다. 기본적 문제를 우아하게 해결합니다: 모델이 출력을 생성할 때 입력의 관련 부분에 어떻게 집중할 수 있을까요?

## 동기: 정보 손실

기본 [seq2seq](/seq2seq-kor.html) 모델은 정보를 단일 맥락 벡터로 병목화합니다:

```
입력: "안녕하세요 어떻게 지내세요 날씨는 좋나요"
                    ↓
            [맥락 벡터]  ← 모든 정보가 여기로 압축
                    ↓
출력: "Hi, how are you, is the weather nice?"
```

짧은 문장의 경우, 이것이 충분히 작동합니다. 더 긴 시퀀스의 경우, 중요한 정보가 손실됩니다. 맥락 벡터는 20+ 토큰 의미를 고정 크기 표현으로 압축해야 합니다.

**문제는 다음과 함께 심각해집니다:**
- 긴 문서 또는 기사
- 역참조가 필요한 복잡한 서사
- 많은 엔터티 관계가 있는 기술 콘텐츠
- 광범위한 이력이 있는 대화

어텐션은 디코더가 입력의 다양한 부분에 동적으로 집중할 수 있도록 함으로써 이를 해결합니다.

## 어텐션 개념: 가중 집계

각 디코딩 스텝에서, 최종 인코더 숨겨진 상태만 사용하는 대신, 어텐션은 모든 입력 토큰에 대한 분포를 계산합니다:

```
"여왕"을 디코딩할 때:
입력에 대한 어텐션 분포:
- "The":     0.01
- "quick":   0.02
- "brown":   0.03
- "fox":     0.80  ← 여기 집중!
- "jumped":  0.05
- 기타:      0.09

맥락 = 모든 입력 숨겨진 상태의 가중 합
```

모델은 관련 입력 토큰에 주의하도록 학습하여 효과적으로 다음을 묻습니다: "이 출력 토큰을 위해 입력의 어느 부분을 이해해야 할까?"

## Bahdanau 어텐션 (가산 어텐션)

Bahdanau 등이 제안한 첫 번째 어텐션 메커니즘 중 하나 (2014):

**구성 요소:**
1. 쿼리: 디코더 숨겨진 상태 h^d_t
2. 키: 인코더 숨겨진 상태 h^e_i
3. 정렬 함수: score(h^d_t, h^e_i) = v^T tanh(W_1 h^d_t + W_2 h^e_i)

**계산:**
```
스텝 1: 정렬 점수 계산
        score_i = v^T tanh(W_1 h^d_t + W_2 h^e_i)

스텝 2: 어텐션 가중치로 정규화
        α_i = exp(score_i) / Σ_j exp(score_j)  [softmax]

스텝 3: 맥락 벡터 계산
        c_t = Σ_i α_i * h^e_i  (인코더 상태의 가중 합)

스텝 4: 디코더 상태와 결합
        h'_t = tanh(W_c [c_t; h^d_t])
```

**직관:** 점수 함수는 디코더 상태와 각 인코더 상태 간의 유사성을 측정합니다. 높은 점수 → 높은 어텐션 가중치 → 맥락에 더 강한 기여.

**가산 vs 곱셈:**
- 가산 (Bahdanau): 학습된 가중 행렬과 tanh 비선형성 사용
- 곱셈: 더 간단한 점수 함수 (다음 논의)

## Luong 어텐션 (곱셈 어텐션)

나중에 Luong 등이 제안한 더 간단한 대안:

**Luong 어텐션 점수:** score(h^d_t, h^e_i) = h^d_t^T W h^e_i

단일 학습 행렬 W만 있습니다. 더 간단하고 종종 더 빠릅니다.

**비교:**
| Bahdanau | Luong |
|---|---|
| 가산, 비선형 | 곱셈, 선형 |
| 더 많은 매개변수 | 더 적은 매개변수 |
| 긴 시퀀스에 약간 나음 | 짧은 시퀀스에 약간 나음 |
| 어텐션 스텝당 더 많은 계산 | 더 빠른 어텐션 계산 |

## 스케일된 점-곱 어텐션

현대 표준 ([트랜스포머](/llm-kor.html)에서 사용):

**점수:** score(Q, K) = (Q K^T) / √d_k

Q는 쿼리, K는 키, d_k는 키 차원입니다.

**왜 스케일링?** 큰 점-곱은 softmax를 포화시킬 수 있습니다 (기울기 → 0). √d_k로 나누기는 분산을 [정규화](/regularization-kor.html)합니다.

**전체 계산:**
```
Attention(Q, K, V) = softmax((Q K^T) / √d_k) V
```

간단하지만 효과적입니다. 세 가지 행렬:
- **쿼리 (Q)**: 뭘 찾고 있나요? (디코더에서)
- **키 (K)**: 뭘 말해줄 수 있나요? (인코더에서)
- **값 (V)**: 정보 콘텐츠가 뭔가요? (인코더에서)

세 가지 모두 일반적으로 실제로 동일한 소스에서 파생됩니다.

## 다중 헤드 어텐션

다양한 "헤드"가 다양한 측면에 주의합니다:

```
입력: 숨겨진 상태 h (차원 d_model)

여러 헤드로 분할:
- 헤드 1: 형용사에 집중하도록 학습 ("빠른", "갈색", "게으른")
- 헤드 2: 명사에 집중하도록 학습 ("여우", "개")
- 헤드 3: 동사에 집중하도록 학습 ("뛰었다")
- 등등

출력 연결: [head_1_output; head_2_output; head_3_output]
d_model 차원으로 다시 투영
```

**장점:**
1. **병렬 어텐션**: 여러 헤드가 동시에 주의 (병렬화 가능)
2. **풍부한 표현**: 각 헤드는 다양한 언어 측면 캡처
3. **견고성**: 한 헤드가 실패하면 다른 헤드가 백업 신호 제공
4. **효율성**: 종종 단일 더 큰 헤드보다 더 나은 성능 달성

**구현:**
```
각 헤드 h에 대해:
    Q_h = X W^Q_h
    K_h = X W^K_h
    V_h = X W^V_h
    head_h = Attention(Q_h, K_h, V_h)

MultiHead(Q, K, V) = Concat(head_1, ..., head_h) W^O
```

일반적 아키텍처: 8-12 헤드, 각각 d_model/num_heads 차원.

## 자기 어텐션: 토큰이 토큰에 주의

[자기 어텐션](/self-attention-kor.html)에서, 토큰은 동일 시퀀스의 모든 다른 토큰에 주의합니다:

```
입력 시퀀스: "고양이가 매트 위에 앉았다"

"고양이"에 대해:
- "고양이"에 주의 (대명사)
- "앉았다"에 주의 (동사 관계)
- "매트"에 주의 (간접 목적어)
- "위"에 낮은 주의 (전치사)

Q, K, V는 모두 동일 시퀀스에서 옵니다!
```

[자기 어텐션](/self-attention-kor.html)은 양방향 (토큰이 모든 위치 봄)이고 [트랜스포머](/llm-kor.html)의 기초입니다.

## 교차 어텐션: 한 소스에서의 쿼리, 다른 소스에서의 키/값

교차 어텐션은 시퀀스 간에 주의합니다:

```
인코더 출력: [token_1_state, token_2_state, ..., token_n_state]
디코더 상태: current_position_state

디코더에서의 쿼리 (뭘 원하나?)
인코더에서의 키와 값 (어디서 찾을 수 있나?)

결과: 디코더가 인코더 표현에 주의
```

교차 어텐션은 비대칭이고 입력과 출력 시퀀스가 다른 번역 같은 작업에 중요합니다.

## 어텐션이 NLP를 혁신한 이유

### 1. **정보 접근성**
모든 디코더 위치는 병목을 우회하여 모든 인코더 위치에 직접 접근할 수 있습니다.

### 2. **해석 가능성**
어텐션 가중치는 어떤 입력이 어떤 출력에 영향을 미쳤는지 직접 보여줍니다:

```
영어: "고양이가 매트 위에 앉았다"
↓
프랑스어: "Le chat est sur le tapis"

어텐션 가중치 공개: "chat"이 대부분 "고양이"에 주의, 등
```

이 해석 가능성은 디버깅과 이해를 돕습니다.

### 3. **장거리 의존성**
어텐션은 쉽게 거리가 먼 위치를 연결할 수 있습니다:

```
"은행 임원이 어제 합병을 발표했다."
"임원"을 처리할 때, "은행"(5 위치 뒤)에 주의 가능
"합병"을 처리할 때, "은행"(7 위치 뒤)에 주의 가능
```

RNN은 그러한 긴 의존성에서 고심했습니다.

### 4. **병렬화**
[자기 어텐션](/self-attention-kor.html)은 병렬 처리를 허용합니다 (RNN과 다름, RNN은 순차):

```
RNN: token_1 → token_2 → token_3 (순차, 병렬화 불가)
자기-어텐션: 모든 토큰이 어텐션으로 병렬 처리
```

[트랜스포머](/llm-kor.html)는 병렬화를 통해 거대한 속도 향상을 달성합니다.

### 5. **위치 유연성**
최근 토큰에 편향된 RNN과 다르게, 어텐션은 모든 위치를 동등하게 취급합니다. 정보 콘텐츠가 위치보다 더 중요합니다.

## 실제 어텐션

### 드롭아웃과 정규화
어텐션 가중치는 훈련 데이터에 과적합될 수 있습니다. 어텐션 [드롭아웃](/dropout-kor.html) (어텐션 가중치를 임의로 0으로 설정)이 도움됩니다.

### 잔여 연결
현대 어텐션은 잔여 연결을 포함합니다:
```
출력 = Attention(입력) + 입력
```
이는 기울기 흐름과 훈련 안정성을 돕습니다.

### 계층 정규화
계층 출력 [정규화](/regularization-kor.html)는 훈련을 개선합니다:
```
출력 = LayerNorm(Attention(LayerNorm(입력)) + 입력)
```

## 어텐션 변형

**희소 어텐션**: 매우 긴 시퀀스의 경우, 모든 위치에 주의 계산은 비용이 많이 듭니다. 희소 어텐션은 부분집합에만 주의합니다 (예: 지역 어텐션, 보폭 어텐션).

**상대 위치 편향**: 절대 위치 대신, 위치 간 관계를 인코딩하여 더 긴 시퀀스로의 일반화를 개선합니다.

**버킷 어텐션**: 위치를 버킷으로 그룹화하고, 버킷 내 및 버킷 간에 주의합니다.

**선형 어텐션**: O(n²) 대신 O(n) 복잡성을 위해 커널 방법으로 softmax 어텐션 근사.

## 결론

어텐션 메커니즘은 고정 맥락 벡터의 정보 병목을 제거함으로써 NLP를 위한 깊은 학습을 변환했습니다. 관련 정보에 집중하도록 학습함으로써, 모델은 번역, 요약, 거의 모든 시퀀스 작업의 성능을 극적으로 개선했습니다.

어텐션의 우아함—학습된 가중 평균 계산—은 깊은 함의를 숨깁니다. 모델이 무엇에 집중할지 학습할 수 있음을 보여주며, 관련성의 어려운 문제를 해결합니다. [seq2seq](/seq2seq-kor.html) 모델 어텐션에서 다중 헤드 [자기 어텐션](/self-attention-kor.html)을 가진 [트랜스포머](/llm-kor.html)로의 경로는 이 핵심 아이디어—"모델이 주의하게 하기"—가 너무 강력해서 현대 AI의 중심이 되었음을 보여줍니다. 어텐션을 이해하는 것은 현대 NLP와 [대규모 언어 모델](/llm-kor.html)을 이해하기 위해 필수적입니다.
