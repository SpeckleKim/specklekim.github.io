# 엣지 AI

## 소개

엣지 AI는 머신러닝 추론을 네트워크 가장자리의 장치(스마트폰, 태블릿, IoT 장치, 임베디드 시스템)에 가져갑니다. 데이터를 클라우드 서버로 보내는 대신 모델이 장치에서 로컬로 실행됩니다. 이는 프라이버시 보존 추론, 감소된 지연 시간, 오프라인 기능, 대역폭 효율성을 가능하게 합니다. 모바일 및 IoT 배포가 폭발하면서 엣지 AI가 중요해지고 있습니다.

## 왜 엣지 AI인가?

### 프라이버시 및 데이터 보안

- 데이터가 장치에 남음, 서버로 송신 안 함
- 프라이버시 규정 준수 (GDPR, CCPA)
- 사용자가 개인정보 제어권 유지
- 건강, 금융, 생체 데이터에 특히 중요

### 낮은 지연 시간

- 네트워크 왕복 없음
- 최신 장치에서 밀리초 단위 추론
- 실시간 애플리케이션에 필요 (게임, AR/VR, 자율 주행)

### 오프라인 기능

- 인터넷 연결 없이 작동
- 시골 지역, 비행기, 불안정한 네트워크에 필수
- 배터리 구동 장치를 무한정 작동

### 대역폭 효율성

- 클라우드에 데이터 업로드 불필요
- 데이터 플랜 사용량 감소
- 인프라 비용 절감

## 모델 압축 기술

### 양자화

FP32에서 낮은 정밀도로 수치 정밀도 감소:

**INT8 양자화**: 4배 모델 크기 감소
- 가중치와 활성화를 8비트 정수로 스케일
- 대부분 모델에서 최소 정확도 손실
- 모든 모바일 장치에서 지원

**FP16 양자화**: 2배 크기 감소
- 반정밀도 부동 소수점
- INT8보다 더 많은 정보 보존
- 호환 하드웨어 필요

**이진/삼진 양자화**: 32배 크기 감소
- 가중치를 단일 비트 또는 삼진 (-1, 0, +1)으로
- 상당한 정확도 절충
- 연구 단계, 제한된 프로덕션 사용

### 가지치기

중요하지 않은 가중치 또는 뉴런 제거:

**크기 기반**: 임계값 < 가중치 제거
- 간단한 구현
- 중요한 연결 손상 가능

**복권 티켓 가설**: 희소 서브네트워크가 전체 네트워크처럼 작동
- 네트워크 학습, 중요 가중치 식별, 기타 제거
- 중요 가중치만 재학습

**구조화된 가지치기**: 전체 필터 또는 계층 제거
- 실제 속도 향상 (비정형 = 불규칙한 메모리 액세스)
- 비정형보다 작은 정확도 이득

### 지식 증류

큰 교사 모델에서 작은 학생 모델 학습:

```
교사 (큼) -----(지식)----> 학생 (작음)
              온도 스케일링
              확률 부드럽게
```

- 학생이 교사 행동 모방 학습
- 교사보다 훨씬 작음
- 교사 성능 대부분 유지

### 텐서 분해

가중치 텐서를 더 작은 인수로 분해:
- 순위 감소: 높은 순위 텐서를 낮은 순위로 근사
- 계산 및 메모리 감소
- 다른 기술보다 적음

## 모바일 ML 프레임워크

### TensorFlow Lite

Google의 경량 추론 엔진:
- 컨버터가 모바일용 모델 최적화 ([양자화](/quantization-kor.html), 가지치기)
- Android, iOS 기본 지원
- TensorFlow, PyTorch (ONNX를 통해) 모델 지원
- 전체 TensorFlow보다 3-4배 작음

```python
# 모델 변환
converter = tf.lite.TFLiteConverter.from_saved_model('model/')
tflite_model = converter.convert()

# 장치에 로드 및 실행
interpreter = tf.lite.Interpreter('model.tflite')
interpreter.invoke()
```

### ONNX (Open Neural Network Exchange)

프레임워크 무관 모델 형식:
- PyTorch, TensorFlow 등에서 ONNX로 모델 변환
- 모델 교환을 위한 표준화 형식
- ONNX Runtime이 추론 최적화
- 좋은 생태계 지원

### Core ML (Apple)

Apple의 온디바이스 ML 프레임워크:
- Apple 하드웨어에 대한 자동 최적화
- Neural Engine 지원 (M-시리즈, A-시리즈 칩)
- 프라이버시 우선 (데이터가 장치를 떠나지 않음)
- 컨버터를 통해 TensorFlow, PyTorch에서 변환

### MediaPipe

Google의 멀티미디어 ML 프레임워크:
- 사전 구축 솔루션: 포즈 감지, 손 추적, 객체 감지
- 실시간 모바일 추론 최적화
- 크로스 플랫폼 (Android, iOS, 웹)
- 프로덕션에 매우 유용

## 엣지 AI용 하드웨어

### 모바일 프로세서

**Apple Neural Engine**: M-시리즈, A-시리즈의 16코어 NPU
- ML에 대해 CPU보다 100배 효율적
- INT8 작업에 특화
- 작은 모델의 경우 GPU보다 빠름

**Qualcomm Hexagon NPU**: Snapdragon 프로세서의 NPU
- 정수 및 고정 소수점 계산
- Apple Neural Engine과 경쟁
- Android 생태계 전체에서 이용 가능

**Google Tensor 칩**: Pixel 폰
- 특정 ML 작업을 위한 맞춤형 하드웨어
- TPU 유사 설계로 Google 서비스 최적화
- 초상화 모드, 실시간 번역 같은 AI 작업에 강력

### 임베디드 가속기

**TPU Lite**: 엣지 배포용 더 작은 TPU
- GPU보다 나은 전력 효율
- 클라우드 TPU보다 작은 배치 크기

**NVIDIA Jetson**: [GPU](/gpu-hardware-kor.html) 기반 엣지 컴퓨팅
- 복잡한 모델을 위한 전체 CUDA 지원
- 간단한 추론에는 과도할 수 있음
- 로보틱스, 자율 주행에 좋음

## 엣지 최적화 기술

### 사후 학습 양자화

재학습 없이 이미 학습된 [모델 양자화](/quantization-kor.html):
- 빠르고 간단함
- 정확도 손실은 모델에 따라 다름
- 좋은 기초 접근

### 양자화 인식 학습

학습 중 [양자화](/quantization-kor.html) 시뮬레이션:
- 모델이 낮은 정밀도 처리 학습
- 사후 학습 [양자화](/quantization-kor.html)보다 나은 정확도
- 재학습 필요

### 엣지용 모델 아키텍처 검색

엣지 최적화 아키텍처를 찾는 AutoML 기술:
- MobileNet, ShuffleNet: 모바일용 설계
- EfficientNet: 복합 스케일 (너비, 깊이, 해상도)
- MobileViT: 모바일용 [비전 트랜스포머](/vision-transformers-kor.html)

### 동적 입력 해상도

변수 입력 크기 수용:
- 장치 기능에 적응
- 이미지를 최적 해상도로 크기 조정
- 정확도 vs 속도 균형

## 온디바이스 학습

사용자 데이터로 모델 미세 조정:
- 특정 사용자/컨텍스트에 적응
- 프라이버시 보존 (사용자 데이터 업로드 없음)
- [연합 학습](/privacy-ai-kor.html): 여러 장치에서 학습, 업데이트 집계

**도전**:
- 제한된 계산 자원
- 배터리 소비
- 메모리 제약

**접근**:
- 작은 서브셋에 대한 경사 기반 미세 조정
- 어댑터 모듈: 작은 학습 가능 구성 요소 추가
- 하이퍼네트워크: 네트워크 가중치 동적 생성

## 배포 고려사항

### 모델 크기 관리

- 일반적 예산: 총 앱 크기 20-100 MB
- 모델 자체: 복잡도에 따라 5-50 MB
- 모델 크기 vs 기능 절충

### 메모리 관리

- RAM 제약 (일반적으로 1-4 GB)
- 모델이 한 번 로드되어 요청 간 공유
- 특성 전처리가 메모리에 영향

### 배터리 영향

- ML 추론이 전력 소비
- 배터리 효율을 위해 최적화
- 정확도 vs 전력 소비 균형

### 업데이트 메커니즘

- 모델을 어떻게 업데이트? OTA 업데이트?
- 버전 관리
- 장치에서의 A/B 테스트

## 프로덕션 예

**얼굴 인식 (Apple FaceID)**
- 실시간 얼굴 감지 및 인식
- 전적으로 온디바이스
- 서브 밀리초 지연 시간

**실시간 포즈 감지 (TikTok, Instagram)**
- 17점 신체 스켈레톤 감지
- 스마트폰에서 30-60 FPS
- Snapdragon/Apple 칩에서 실행

**스마트 키보드 예측**
- 다음 단어 예측
- 사용자 동작에서 온디바이스 학습
- 사용에 따라 시간 경과에 따라 개선

## 미래 방향

- **신경형태 하드웨어**: 이벤트 기반 처리, 극저전력
- **포토닉 처리**: 엣지에서 광학 추론
- **양자**: 특정 엣지 작업을 위한 양자 프로세서
- **지속적 학습**: 사용자/환경에 대한 지속적 적응

## 결론

엣지 AI는 클라우드에서 장치로 계산을 제거하여 프라이버시, 반응성, 오프라인 기능을 활성화합니다. 성공은 신중한 모델 최적화, 하드웨어 제약 이해, 견고한 배포 파이프라인이 필요합니다. 모바일 및 IoT가 증가하면서 엣지 AI는 책임감 있고 성능 있는 AI 배포의 중심이 됩니다.
