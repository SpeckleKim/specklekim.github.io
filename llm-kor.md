# 대규모 언어 모델과 트랜스포머: AI의 혁명

## 대규모 언어 모델(LLM)이란 무엇인가?

대규모 언어 모델은 인터넷, 책, 기타 자료들로부터 수집한 방대한 텍스트 데이터로 학습된 수십억 개에서 수조 개의 매개변수를 가진 [신경망](/neural-kor.html)입니다. 이들은 언어의 통계적 패턴을 학습하여 다음 단어를 놀라운 정확도로 예측할 수 있습니다. LLM은 명시적으로 프로그래밍된 규칙이 아니라, [비지도학습](/unsupervised-learning-kor.html)이라 불리는 과정을 통해 문법, 사실, 추론, 심지어 창의적인 글쓰기에 대한 암묵적인 이해를 개발합니다.

LLM은 검색 엔진이나 외운 사실들의 데이터베이스가 아닙니다. 대신, 인간 언어의 기저 구조와 지식을 학습한 정교한 패턴 인식 시스템입니다. LLM에 질문을 하면, 이전 모든 맥락과 학습된 패턴에 기반하여 각 새로운 토큰을 예측하면서 한 번에 하나씩 응답을 생성합니다.

### "대규모"의 의미

"대규모"라는 용어는 매우 중요합니다. 깊은 학습에서 규모 확대는 엄청나게 중요합니다. 70억 개의 매개변수를 가진 모델은 700억 개의 매개변수를 가진 모델과는 질적으로 다르게 작동합니다. 이는 단순히 성능의 차이가 아니라 더 작은 모델에는 없는 "[창발적 능력](/emergent-abilities-kor.html)"을 나타냅니다. 여기에는 복잡한 추론, 코드 작성, 미묘한 대화 수행 능력이 포함됩니다.

## 트랜스포머 아키텍처

2017년 Vaswani 등이 발표한 트랜스포머 아키텍처는 순환 [신경망](/neural-kor.html)(RNN)을 주의(Attention) 메커니즘으로 대체하여 자연어 처리에 혁명을 일으켰습니다.

### 자기-주의 메커니즘

트랜스포머의 핵심에는 자기-주의가 있으며, 이는 모델이 각 단어를 처리할 때 서로 다른 단어의 중요도를 측정할 수 있게 합니다. 예를 들어, "은행 임원이 은행으로 걸어들어갔다"는 문장에서 자기-주의는 맥락에 따라 "은행"의 두 경우가 다른 의미를 가진다는 것을 이해하는 데 도움을 줍니다.

자기-주의는 다음과 같이 작동합니다:
1. 각 단어를 세 개의 벡터로 변환: 질문(Query, Q), 키(Key, K), 값(Value, V)
2. 질문 벡터를 모든 키 벡터와 비교하여 주의 점수 계산
3. 이 점수들을 가중치로 사용하여 값 벡터의 가중합 생성
4. 이 가중합이 해당 위치의 출력이 됨

이 메커니즘은 RNN과 달리 병렬 처리 가능하여, 트랜스포머가 순차적이 아니라 전체 문장을 동시에 처리할 수 있습니다.

### 다중-헤드 주의

트랜스포머는 주의를 한 번만 계산하지 않고, 여러 "헤드"를 병렬로 작동시킵니다. 각 헤드는 다양한 유형의 관계를 학습합니다. 한 헤드는 구문 구조에 초점을 맞추고, 다른 헤드는 의미 관계에, 또 다른 헤드는 장거리 의존성에 초점을 맞춥니다. 이러한 다중-헤드 접근은 더 풍부한 표현을 제공합니다.

### 위치 인코딩

트랜스포머는 모든 토큰을 병렬로 처리하므로, 단어 순서를 이해하는 방법이 필요합니다. [위치 인코딩](/positional-encoding-kor.html)은 삼각함수를 통해 각 단어의 위치에 대한 정보를 추가하여, 모델이 해석하는 방법을 학습할 수 있는 고유한 패턴을 만듭니다.

### 피드포워드 신경망

주의 층 사이에 트랜스포머는 각 위치에 독립적으로 비선형 변환을 적용하는 피드포워드 [신경망](/neural-kor.html)을 포함합니다. 이는 모델링 용량과 비선형성을 추가합니다.

### 디코더 전용 vs 인코더-디코더

원본 트랜스포머는 인코더([BERT](/bert-kor.html) 같은)와 디코더([GPT](/gpt-kor.html) 같은) 모두를 가지고 있었습니다. 인코더는 전체 시퀀스를 한 번에 처리하며 이해 작업에 사용됩니다. 디코더는 한 번에 하나씩 토큰을 생성하며 생성 작업에 사용됩니다. GPT, Claude, LLaMA 같은 현대 LLM은 텍스트를 이해하고 생성할 수 있는 디코더 전용 아키텍처를 사용합니다.

## 언어 모델 발전의 주요 이정표

### 트랜스포머 이전 시대
- **2013년: Word2Vec**: 의미 관계를 포착한 단어 임베딩
- **2014년: Seq2Seq 모델**: [기계 번역](/machine-translation-kor.html)을 위한 주의 메커니즘 도입

### 트랜스포머 혁명 (2017년 이후)
- **2017년: 트랜스포머 아키텍처**: 모든 현대 LLM의 기초
- **2018년: BERT**: 사전학습의 힘을 보여주는 양방향 표현
- **2018년: GPT**: 언어 모델링이 규모에서 후속 작업으로 전이됨을 입증
- **2019년: GPT-2**: 텍스트 생성과 제로샷 학습에서 우려할만한 능력 시연

### 규모 확장 시대
- **2020년: GPT-3**: 1,750억 개 매개변수, 소수 샷 학습 및 [창발적 능력](/emergent-abilities-kor.html) 시연
- **2021년: T5**: 모든 NLP 작업을 텍스트-텍스트로 취급하는 통합 프레임워크
- **2021년: PaLM**: 구글의 5,400억 매개변수 모델
- **2022년: Chinchilla**: 최적 확장 법칙이 이전 가정과 다름을 보여줌
- **2023년: LLaMA**: 메타의 효율적인 오픈소스 모델 (7B-65B)
- **2023년: GPT-4**: [멀티모달](/multimodal-llm-kor.html), 개선된 추론, 향상된 신뢰성
- **2023년: Claude 2**: 헌법적 AI로 향상된 안전성
- **2024년 이후**: Mixture of Experts 모델, 더 긴 [컨텍스트 윈도우](/context-window-kor.html), [멀티모달](/multimodal-llm-kor.html) 능력

## 학습 과정

### 사전학습: 기초 형성

사전학습은 마법이 일어나는 곳입니다. 모델은 다음 토큰 예측(인과 언어 모델링)을 사용하여 수천억 개의 토큰으로 학습됩니다. 목표는 간단합니다: 이전 모든 토큰이 주어졌을 때 다음 토큰을 예측하는 것입니다. 이 단순성에도 불구하고, 모델은 다음을 학습해야 합니다:
- 문법과 구문 이해
- 사실적 지식 저장
- 추론 패턴 학습
- 세계에 대한 모델 개발

이 학습은 계산 집약적이며 전문화된 하드웨어([GPU](/gpu-hardware-kor.html)/TPU)가 필요하고 수주에서 수개월이 걸릴 수 있습니다. 다양한 인터넷 텍스트에 대한 사전학습은 범용 기초를 만듭니다.

### 미세 조정: 특정 작업에 적응

사전학습 후, 모델은 질문 답변, 요약 또는 번역 같은 특정 작업을 위해 작은 엄선된 데이터셋에 대해 미세 조정됩니다. 미세 조정은 사전학습보다 훨씬 적은 계산 자원을 필요로 합니다(수개월 대신 수시간에서 수일).

### 명령 미세 조정: 명령 따르도록 가르치기

현대 LLM은 명령 미세 조정을 거치며, 자연어 명령을 따르는 방법을 학습합니다. (명령, 출력) 쌍의 데이터셋은 모델이 더 유용하고 사용자의 의도를 따르도록 가르칩니다. 이것이 원시 언어 모델을 유용한 어시스턴트로 변환합니다.

### RLHF: 인간 피드백으로부터의 강화학습

RLHF는 모델이 생성하는 것과 인간이 가치 있다고 생각하는 것 사이의 일치를 개선합니다. 과정은 다음을 포함합니다:
1. 출력 간의 인간 선호도에 대해 보상 모델 학습
2. 이 보상 모델을 사용하여 모델 생성 출력 채점
3. [강화학습](/rl-kor.html)으로 LLM 학습하여 보상 신호 최대화

RLHF는 안전성, 도움이 됨, 해 없음을 위해 중요한 것으로 입증되었습니다.

## 토큰화와 임베딩

### 토큰화

텍스트가 LLM에 들어가기 전에, 부분단어 단위인 토큰으로 나뉩니다. 모델은 원시 문자를 처리하지 않고, 대신 50,000에서 100,000 토큰의 어휘를 사용합니다. 일반적인 [토큰화](/tokenization-kor.html) 방법은 다음을 포함합니다:
- **바이트 쌍 인코딩 (BPE)**: 빈번한 바이트 쌍을 반복적으로 병합
- **WordPiece**: BERT에서 사용, 미등록 단어를 위한 특수 토큰 추가
- **SentencePiece**: 언어에 관계없는 접근

[토큰화](/tokenization-kor.html) 선택은 특히 영어가 아닌 언어에 대한 성능에 영향을 미칩니다.

### 임베딩

각 토큰은 고차원 벡터(일반적으로 768에서 12,288 차원)로 변환됩니다. 이 토큰 임베딩은 학습 중에 학습되고 의미 및 구문 정보를 나타내도록 진화합니다. 토큰 임베딩은 모델이 언어와의 유일한 연결입니다. 다른 모든 것은 벡터 공간에서의 패턴 인식입니다.

## 확장 법칙과 창발적 능력

### Chinchilla 확장 법칙

DeepMind의 Hoffman 등 (2022)의 연구는 최적 성능이 다음에 거의 동일한 계산을 필요로 함을 보였습니다:
- 모델 크기 증가 (매개변수)
- 훈련 데이터 증가 (토큰)

이는 모델 크기가 지배적이어야 한다는 이전 가정과 달랐습니다. 의미: 모델은 이전에 최적이라고 생각한 것보다 더 많은 데이터에 대해 훈련되어야 합니다.

### 창발적 능력

모델이 규모를 확대하면서 놀라운 능력이 예기치 않게 나타납니다:
- **문맥 내 학습**: 모델이 프롬프트의 몇 가지 예시에서 작업을 수행하는 방법을 학습
- **생각의 사슬**: 프롬프트되었을 때 모델이 단계적으로 추론 가능
- **코드 생성**: 충분한 규모에서 모델이 기능성 코드 작성
- **명령 따르기**: 모델이 자연어 명령을 이해하고 따름
- **마음의 이론**: 다른 사람의 신념과 의도 이해

이러한 능력은 작은 규모에서 존재하지 않습니다. 규모에 따라 "창발"하며, 모델이 작동하는 방식이 질적으로 변한다는 것을 시사합니다.

## 현재 환경과 응용

### 주요 모델

**오픈소스**: LLaMA-2, Mistral, Falcon
**상용**: [GPT](/gpt-kor.html)-4, Claude 3, Gemini, Copilot

각각은 속도, 비용, 능력, 사용자 정의에서 장단점이 있습니다.

### 응용 분야

- **코딩**: GitHub Copilot, 코드 완성, 버그 탐지
- **콘텐츠 제작**: 기사 작성, 창의적인 소설, 마케팅 카피
- **고객 서비스**: 챗봇, 지원 자동화
- **연구**: 문헌 검토, 가설 생성
- **번역**: 거의 인간 수준의 번역 품질
- **교육**: 개인화된 튜터링, 설명
- **비즈니스 인텔리전스**: 데이터 분석, 인사이트, 요약
- **과학 발견**: 단백질 폴딩, 신약 발견 지원

## 제한 사항과 도전

### 환각(Hallucination)

LLM은 때때로 그럴듯하게 들리는 거짓 정보를 자신감 있게 생성합니다. 이는 거짓말이 아니라, 모델이 진실에 접근할 수 없기 때문입니다. 패턴 매칭일 뿐입니다. 컨텍스트 길이와 훈련 데이터 품질은 환각 율에 영향을 미칩니다.

### 컨텍스트 길이

표준 트랜스포머는 2,000-4,096 토큰의 컨텍스트만 처리할 수 있습니다. 희소 주의 및 반복과 같은 새로운 기법은 이를 연장하지만, 여전히 제한입니다. 더 긴 컨텍스트는 더 나은 추론과 메모리를 가능하게 합니다.

### 진정한 이해의 부족

인상적인 출력에도 불구하고, LLM은 인간의 의미에서 진정으로 "이해"하지 못합니다. 다음이 부족합니다:
- 진정한 세계 모델
- 인과 추론
- 상식
- 물리적 세계에서의 접지

더 정확하게는 "확률적 장황"으로 묘사되며, 정교한 패턴을 학습했습니다.

### 편향과 공정성

인터넷 데이터로 훈련된 모델은 인간의 편향을 상속받습니다. 출력은 고정관념을 반영할 수 있고, 부당함을 시연하거나, 해로운 아이디어를 영구화합니다. 편향을 완화하려면 신중한 데이터셋 큐레이션, 미세 조정, 지속적인 모니터링이 필요합니다.

### 에너지 소비

대규모 모델 훈련은 막대한 에너지를 필요로 하여 환경 문제를 제기합니다. 단일 대규모 모델 훈련 실행은 연간 소도시만큼의 전기를 소비할 수 있습니다. 규모에서의 추론도 상당한 자원을 필요로 합니다.

### 안전과 정렬

강력한 AI 시스템을 인간 가치와 정렬하는 것은 개방 문제입니다. RLHF는 도움이 되지만, 다음에 대한 질문이 남습니다:
- 적대적 견고성
- 조작과 기만
- 보상 해킹
- 가치 명시

## 미래 방향

### 더 긴 컨텍스트와 메모리

미래 모델은 더 긴 컨텍스트(잠재적으로 수백만 토큰)를 처리하고 대화 전반에 걸쳐 정보를 유지할 수 있는 메모리 메커니즘을 구현할 것입니다.

### Mixture of Experts

모든 입력에 모든 매개변수를 사용하는 대신, Mixture of Experts ([MoE](/moe-kor.html)) 모델은 입력을 전문화된 부분 네트워크로 라우팅합니다. 이는 더 효율적으로 수조 매개변수로 확장할 수 있게 합니다.

### 멀티모달 모델

텍스트, 이미지, 오디오, 비디오를 통합 모델에 결합하는 것이 표준이 되고 있습니다. 미래 모델은 LLM이 텍스트를 처리하는 것처럼 모든 데이터 유형을 원활하게 처리할 수 있을 것입니다.

### 추론과 계획

현재 모델은 패턴 매칭에 탁월하지만 새로운 추론에는 어려움을 겪습니다. 향후 작업은 외부 도구, 지식 기반, 명시적 추론 메커니즘 통합에 초점을 맞출 것입니다.

### 해석 가능성과 설명 가능성

LLM이 결정을 내리는 방법을 이해하는 것은 신뢰와 안전을 위해 중요합니다. 기계적 해석 가능성 연구는 개별 뉴런과 회로가 특정 행동에 어떻게 기여하는지 밝혀내고 있습니다.

### 인간-AI 협력

완전히 자율적인 AI보다는, 미래 시스템은 인간-루프 협력을 강조할 수 있으며, AI가 인간을 대체하기보다는 인간의 능력을 증강합니다.

## 결론

대규모 언어 모델은 AI의 근본적인 돌파구를 나타내며, 단순한 목표(다음-토큰 예측)를 충분한 데이터와 계산으로 확장하면 놀랍도록 능력 있는 시스템이 생성될 수 있음을 보여줍니다. 트랜스포머 아키텍처는 기초를 제공하지만, 규모, 데이터 품질, 학습 후 기법의 결합이 우리가 지능적인 대화를 나눌 수 있는 모델을 만듭니다.

그러나 우리는 여전히 초기 단계에 있습니다. 현재 LLM은 진정한 이해 없이 정교한 패턴 매칭입니다. 앞으로 나아갈 길은 더 나은 아키텍처, 개선된 훈련 기법, 견고한 안전 조치, 이러한 시스템이 어떻게 작동하는지에 대한 더 깊은 이해를 포함합니다.

LLM의 진정한 영향은 개별 모델 능력이 아니라 사회에 어떻게 통합되는지에 의해 측정될 것입니다. 인간의 창의성을 증강하고, 과학적 발견을 가속화하고, 우리가 함께 달성할 수 있는 것을 확장합니다.
