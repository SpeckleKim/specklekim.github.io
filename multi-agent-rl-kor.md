# 다중 에이전트 강화학습: 단일 에이전트를 넘어서

## 소개

다중 에이전트 강화학습(MARL)은 단일 에이전트 RL을 여러 상호작용 에이전트가 있는 환경으로 확장합니다. 에이전트는 협력적(공유 목표), 경합적(반대 목표), 혼합될 수 있습니다. MARL은 도전을 제시합니다: 비정상 환경, 조정 어려움, 확장성 문제. 응용은 로봇 집단, 교통 관리, 게임 플레이를 포함합니다.

## 기본적 차이점

**단일 에이전트 RL:** 환경의 응답은 단일 에이전트의 행동에만 의존합니다.

**다중 에이전트 RL:** 환경의 응답은 모든 에이전트의 행동에 의존하며, 각 에이전트는 불완전한 상태 정보를 관찰합니다.

**주요 도전:**
- 환경이 비정상이 됩니다 (다른 에이전트의 정책이 변경)
- 신용 할당: 여러 에이전트로 성공/실패를 누가 야기했는가?
- 확장성: 지수적 상태/행동 공간 증가
- 수렴: 여러 에이전트가 동시에 적응할 때 보장 없음

## 게임 이론적 기초

**정상형 게임:**
- 동시 행동 선택
- 보상 행렬이 결과 결정
- 내시 균형: 단측 편차로 이익이 없는 에이전트

**광범위형 게임:**
- 순차 행동 선택
- 게임 트리가 결정 구조 표현
- 부분게임 완전 균형이 더 적절

**영합 vs 일반합:**
- 영합: 한 에이전트의 이득이 다른 에이전트의 손실 (포커, 체스)
- 일반합: 상호 이익/손실 가능 (교통, 협력)

## 내시 균형

기본 게임 이론적 해 개념:

```
σ*이 내시 균형이면:
E[u_i(σ*_i, σ*_{-i})] ≥ E[u_i(σ'_i, σ*_{-i})] 모든 σ'_i에 대해
```

각 에이전트의 전략은 다른 에이전트 전략의 최선 대응입니다.

**다중 에이전트 학습 목표:** 내시 균형 정책 프로필 찾기.

**도전:**
- 여러 균형 존재 가능
- 모든 균형이 바람직하지는 않음
- 균형으로의 수렴이 불확실

## 협력 vs 경합

**완전 협력:**
- 공유 보상: 모든 에이전트가 동일 목표 최대화
- 커뮤니케이션 필수
- 중앙집중식 학습 유리

**완전 경합:**
- 영합 게임
- 반대 목표
- 자기 플레이 학습 효과적

**혼합 설정:**
- 팀 경쟁 (여러 팀)
- 혼합 동기 게임 (일부 공유, 일부 반대)
- 복잡한 전략적 추론 필요

## 중앙집중식 학습, 분산화된 실행 (CTDE)

**주요 프레임워크:** 결합 상태/행동으로 학습하고 로컬 관찰로 실행합니다.

**학습 단계:**
- 중앙 비평가가 모든 에이전트, 모든 행동 관찰
- 학습은 전체 정보 사용

**실행 단계:**
- 각 에이전트가 독립적으로 행동
- 로컬 관찰만 사용
- 학습 정책은 커뮤니케이션 없이 작동해야

**장점:**
- 전역 정보로부터의 학습 안정성
- 중앙 조정자 없는 실행 확장성
- 전역 비평가를 통한 신용 할당 해결

**표준 접근: MADDPG**
- DDPG의 다중 에이전트 확장
- 학습 중 중앙집중식 비평가
- 실행을 위한 분산화된 행위자

## 커뮤니케이션과 조정

**암시적 커뮤니케이션:** 신흥 행동을 통해 학습.

**명시적 커뮤니케이션:** 에이전트가 메시지 보냄; 커뮤니케이션 프로토콜 학습.

**조정 메커니즘:**
- 핸드셰이크 프로토콜
- 공유 목표/목표
- 조정을 향한 보상 형성

**CommNet, QMIX:** 에이전트 간 커뮤니케이션을 가능하게 하는 아키텍처.

## MAPPO (다중 에이전트 PPO)

다중 에이전트 설정으로의 PPO 확장:

**주요 요소:**
1. **공유 정책 네트워크:** 모든 에이전트가 정책 매개변수 공유
2. **개별 가치 네트워크:** 각 에이전트가 별도 가치 함수 가짐
3. **중앙집중식 가치 함수:** 안정성을 위한 중앙 비평가 (선택)
4. **절단된 대리:** PPO 절단을 다중 에이전트 경우에 적용

**학습:**
```
각 에피소드에 대해:
  모든 에이전트가 전이 수집
  로컬 가치 함수를 사용한 우위 계산
  공유 정책과 개별 가치 업데이트
  선택적으로 중앙 비평가 업데이트
```

**특성:**
- 단순하고 실무적 알고리즘
- 많은 협력 도메인에 효과적
- 많은 에이전트로 확장 가능
- 정책 경사 장점 유지

## 비정상 환경에서의 학습

**핵심 문제:** 다른 에이전트의 정책이 변경되면 환경이 비정상으로 나타납니다.

**해결책:**

**정책 지연:** 상대방 모델을 일시적으로 고정 유지.
- 고정 정책에 대해 학습
- 주기적으로 상대방 모델 업데이트
- 비정상성 효과 감소

**상대방 모델링:** 다른 에이전트의 행동을 명시적으로 추적.
- 상대방 정책의 모델 학습
- 그에 따라 자신의 정책 적응
- 계산 비용 높음

**자기 플레이:** 자신 정책의 사본에 대해 학습.
- 영합 게임에 효과적
- 자연스러운 교육 과정: 점점 더 강한 상대
- 초인적 성능 가능 (AlphaGo, AlphaStar)

## 자기 플레이 학습

**과정:**
```
각 반복에 대해:
  현재 정책 v_t가 이전 버전에 대해 플레이
  v_t를 이기도록 새 정책 v_{t+1} 학습
  모든 세대의 경험 누적
  v_{t+1}이 새 최선이 되고 더 약하면 롤백 가능
```

**장점:**
- 자연스러운 교육 과정: 상대가 학습자와 함께 강해짐
- 탐험: 이전 정책이 탐험 다양성 제공
- 복잡성 관리: 점진적 난이도 증가

**성공 사례:**
- AlphaGo가 바둑 세계 챔피언 격파
- AlphaStar가 StarCraft 초인적 성능 도달
- 다중 에이전트 환경의 신흥 행동

## 신흥 행동

**놀라운 현상:** 명시적 프로그래밍 없이 단순 보상 함수에서 복잡하고 조정된 행동이 신흥합니다.

**예:**
- 사전 프로그래밍된 행동 없는 포식자-피식자 동학
- 명시적 조정 규칙 없는 대형 제어
- 다중 에이전트 시스템의 언어 같은 커뮤니케이션
- 팀의 작업 전문화

**신흥을 가능하게 하는 요인:**
- 충분한 에이전트 인구
- 지속적인 학습
- 적절한 보상 구조
- 환경적 복잡성

## 확장성 도전

**지수적 증가:**
- 상태 공간: n 에이전트에 대해 O(|S|^n)
- 행동 공간: n 에이전트에 대해 O(|A|^n)
- 복잡성이 에이전트 수와 함께 극적으로 증가

**해결책:**

**평균장 근사:** 단일 에이전트가 다른 에이전트의 평균과 상호작용한다고 가정.
```
가치 ≈ f(자신_상태, 다른_것의_평균_장)
```
차원 감소, 많은 에이전트 학습 가능.

**그래프 신경망:** 에이전트 상호작용을 그래프 구조로 사용.
- 에이전트 간 표현 공유
- 순열 불변성 활용
- 큰 에이전트 인구로 확장

**계층적 학습:** 다양한 조정 규모.
- 저수준: 개별 에이전트 제어
- 고수준: 팀/집단 조정

## 응용

**로봇 집단:** 조정된 동작, 집단 의사결정.

**교통 제어:** 도로 상의 자율 차량 조정; 혼잡 감소.

**게임 플레이:** Dota 2 (OpenAI Five), StarCraft (AlphaStar), 체스 변형.

**공급망:** 여러 창고, 유통업자 조정.

**스펙트럼 공유:** 무선 장치 공존 학습.

## 벤치마크 및 환경

**SMAC (StarCraft 다중 에이전트 도전):** StarCraft의 협력 전투 시나리오.

**OpenAI Gym 환경:** 고전 환경의 다중 에이전트 변형.

**OpenSpiel:** 보드 게임, 게임 이론적 환경.

**맞춤 시뮬레이터:** 특정 도메인용 (로봇공학, 교통 등).

## 이론적 고려사항

**수렴 보장:** 다중 에이전트 설정에서는 제한적.
- 단일 에이전트 수렴 증명 적용 안 됨
- 순환 행동 가능
- 비수렴 동학이 일반적

**커뮤니케이션 복잡성:** 몇 개의 메시지 필요?
- 정보 이론적 한계
- 효율성과 조정 간 트레이드오프

**경직성 결과:** 일부 다중 에이전트 문제 NP 어려움.

## 현대 동향

**주의 메커니즘:** 커뮤니케이션/조정을 위한 관련 에이전트에 집중.

**트랜스포머 아키텍처:** 가변 에이전트 인구 처리.

**커뮤니케이션 프로토콜 학습:** 에이전트가 최적 메시징 전략 발견.

**가치 분해:** 결합 가치를 개별 기여로 분해 (QMIX, QTRAN).

## 결론

다중 에이전트 RL은 단일 에이전트 학습을 복잡하고 상호작용하는 환경으로 확장합니다. 협력 및 경합 설정은 다른 접근법을 요구합니다. CTDE 프레임워크는 확장 가능한 학습과 실행을 가능하게 합니다. 자기 플레이는 영합 게임에서 초인적 성능을 주도합니다. 신흥 행동은 단순 목표에서의 복잡한 조정을 보여줍니다. 다중 에이전트 시스템이 확산됨에 따라(자율 차량, 드론 집단, 게임 AI), MARL은 실무 응용에서 점점 더 중요해집니다.
