# 시퀀스-투-시퀀스 모델: 신경망 번역의 기초

시퀀스-투-시퀀스(seq2seq) 모델은 많은 현대 NLP 애플리케이션의 기초입니다. 기계 번역에서 요약, 대화 시스템에 이르기까지, seq2seq 아키텍처는 널리 보편화되었습니다. 이 모델들은 가변 길이 입력 시퀀스를 받아 가변 길이 출력 시퀀스를 생성하므로 유연하고 강력합니다.

## Seq2Seq 문제

전통적 기계학습은 고정 크기 입력과 출력을 가정합니다. 언어 문제는 이를 따르지 않습니다:

```
영어: "안녕하세요, 어떻게 지내세요?" (5 토큰)
한국어: "Hi, how are you?" (5 토큰)
일본어: "こんにちは、お元気ですか?" (토큰화 후 7 토큰)
```

**핵심 도전:**
- 가변 길이 입력 및 출력
- 일대일 토큰 정렬 없음
- 언어 간 복잡한 재배열
- 장거리 의존성 포착

Seq2seq 모델은 명시적 언어 규칙이 아닌 암묵적 정렬을 학습함으로써 이를 처리합니다.

## 인코더-디코더 아키텍처

seq2seq 프레임워크는 처리를 두 가지 구성 요소로 분리합니다:

### 인코더: 입력 처리

인코더는 입력 시퀀스를 처리하고 컴팩트한 표현(맥락 벡터)을 생성합니다:

```
입력: "안녕하세요, 어떻게 지내세요?"
↓ (LSTM/GRU 처리)
맥락 벡터: [2.3, -1.1, 0.8, ..., 1.2]
```

**인코더의 책임:**
1. 시퀀셜 또는 병렬로 토큰 처리
2. 맥락을 캡처하는 숨겨진 상태 유지
3. 전체 입력을 나타내는 단일 벡터 생성
4. 입력과 출력 도메인 간의 간격 해결

인코더는 전체 시퀀스 의미를 단일 벡터로 압축해야 합니다. 긴 시퀀스의 경우, 정보 손실은 피할 수 없으므로 주의(나중에 논의)에 대한 동기를 부여합니다.

### 디코더: 출력 생성

디코더는 맥락 벡터를 받아서 출력 토큰을 순차적으로 생성합니다:

```
맥락 벡터: [2.3, -1.1, 0.8, ..., 1.2]
↓ (디코더 초기화)
숨겨진 상태: 맥락 벡터
↓ (토큰별 생성)
출력: "Hi, how are you?"
```

**디코더의 책임:**
1. 인코더 맥락에서 초기화
2. 순차적으로 토큰 생성
3. 생성 전체에 걸쳐 숨겨진 상태 유지
4. 적절히 생성 중지 ([EOS] 토큰)

디코더는 본질적으로 입력 표현에 조건화된 언어 모델입니다.

## RNN 기반 Seq2Seq

대부분의 seq2seq 모델은 재귀 아키텍처(LSTM, GRU)를 사용합니다:

**인코더:**
```
입력: x₁, x₂, ..., xₙ
LSTM: h₁ → h₂ → ... → hₙ
출력: hₙ (맥락 벡터)
```

**디코더:**
```
초기화: h₀' = hₙ (인코더 맥락)
생성: y₁ ← p(y₁|h₀', 맥락)
      h₁' = LSTM(y₁, h₀')
      y₂ ← p(y₂|h₁', 맥락)
      ... [EOS]까지 반복
```

**핵심 결정: 양방향 인코더**

양방향 RNN은 입력을 좌우로 그리고 우좌로 처리하여 표현을 연결합니다:

```
순방향: → 안녕하세요 어떻게 지내세요 →
역방향: ← 세요 내 지 게 어 세요 하 안 ←
연결: 각 토큰에 대해 [순방향; 역방향]
```

양방향 인코더는 일반적으로 단방향을 능가하므로, 양쪽 방향의 맥락이 표현을 알립니다.

## 교사 강제 (Teacher Forcing)

훈련 중에 디코더는 어떻게 생성하는 법을 배울까요?

```
순진한 접근법:
입력: x₁, x₂, ... (예: 영어 문장)
목표: y₁, y₂, y₃ (예: 한국어 문장)
디코더 스텝 1: y₁ 예측
               틀리면 잘못된 y₁을 기반으로 y₂ 예측
               예측이 오류를 증폭
```

**교사 강제 해결책:**

훈련 중 정답 목표 사용:

```
훈련 시:
스텝 1: 맥락이 주어지면 y₁ 예측
스텝 2: 맥락 + 정답 y₁이 주어지면 y₂ 예측
스텝 3: 맥락 + 정답 y₁, y₂가 주어지면 y₃ 예측
```

**불일치 문제:**
- 훈련: 디코더는 정답 토큰을 봄
- 추론: 디코더는 자신의 예측을 봄 (잘못될 수 있음)
- 분포 이동: 훈련과 테스트 조건이 다름

이 노출 편향은 교사 강제로 훈련한 모델이 테스트 시에 더 많은 오류를 만들도록 합니다. 해결책에는 다음이 포함됩니다:
- **예약된 샘플링**: 훈련 중 정답에서 예측으로 점진적 전환
- **빔 검색**: 탐욕 디코딩이 아닌 여러 가설 사용
- **데이터 증강**: 때때로 손상된 입력으로 훈련

## 주의 메커니즘 (간단한 소개)

기본 seq2seq의 핵심 문제: 전체 시퀀스를 하나의 벡터로 압축하면 정보 손실이 발생합니다.

```
영어: "안녕하세요, 어떻게 지내세요, 날씨는 좋나요?"
한국어: "Hi, how are you, is the weather nice?"

고정 맥락 벡터는 정렬 정보를 잃음.
어떤 출력 토큰이 어떤 입력 토큰과 정렬되나요?
```

주의는 입력 토큰에 대한 가중 분포를 계산합니다:

```
한국어 "날씨"(weather)를 생성할 때:
주의는 영어 "weather"에 집중
한국어 "어떻게"(how)를 생성할 때:
주의는 영어 "how"에 집중
```

최종 인코더 상태만 사용하는 대신, 주의는 디코더가 관련성으로 가중치화된 입력 토큰을 "뒤돌아보게" 합니다. 이는 주의 메커니즘 기사에서 완전히 탐구됩니다.

## 빔 검색 디코딩

테스트 시에, 탐욕 디코딩(항상 가장 높은 확률 토큰 선택)은 최적이 아닙니다:

```
탐욕은 "안녕하세요 다시..."를 선택할 수 있음
더 나은 해결책: "안녕 다시..."
하지만 탐욕은 스텝 1에서 "안녕"으로 잠금
```

빔 검색은 여러 가설을 유지합니다:

```
스텝 1: 상위 k=3 가설 유지
        "안녕" (확률=0.4), "안녕하세요" (확률=0.3), "Hi" (확률=0.2)
스텝 2: 각각 확장, 전체 상위 k 유지
        "안녕 어떻게" (0.4*0.5), "안녕하세요 어떻게" (0.3*0.6)
스텝 3: [EOS]까지 계속
```

빔 크기 트레이드오프:
- k=1: 탐욕, 가장 빠름, 낮은 품질
- k=3-5: 품질과 속도의 좋은 균형
- k>5: 약간 더 나은 품질, 더 느림

## Seq2Seq의 응용

### 기계 번역

seq2seq의 원래 동기. 신경망 기계 번역(NMT)은 구문 기반 통계 접근법을 대폭 능가했습니다.

```
입력 (영어): "The meeting is scheduled for tomorrow."
출력 (한국어): "회의는 내일로 예정되어 있습니다."
```

### 추상적 요약

요약을 시퀀스-투-시퀀스 작업으로:

```
입력: 전체 기사 (수백 토큰)
출력: 요약 (50-100 토큰)
```

추상적 (새 텍스트 생성) vs 추출적 (문장 선택). Seq2seq는 추상적 접근법을 가능하게 합니다.

### 질의 응답

읽기 이해를 seq2seq로:

```
입력: "존은 가게에 갔습니다. 그는 우유를 샀습니다. 그는 떠났습니다."
질문: "존이 뭘 샀나요?"
출력: "우유."
```

또는 맥락과 질문을 입력으로 하는 더 복잡한 seq2seq QA.

### 대화와 챗봇

대화 AI를 seq2seq로:

```
입력: "안녕하세요, 어떻게 지내세요?"
출력: "잘 지내고 있습니다, 물어봐 주셔서 감사합니다!"
```

인코더는 대화 이력 또는 맥락을 처리하고, 디코더는 응답을 생성합니다.

## Seq2Seq 확장

현대 변형은 기본 seq2seq를 개선합니다:

1. **양방향 디코딩**: 미래 맥락을 캡처하기 위해 역방향 생성
2. **범위 메커니즘**: 어떤 입력 토큰이 번역되었는지 추적
3. **다중 작업 학습**: 번역과 관련 작업 결합
4. **역번역**: 합성 훈련 데이터 생성
5. **계층별 주의**: 다양한 디코더 계층이 서로 다른 입력 수준에 주의

## 기본 Seq2Seq의 한계

1. **정보 병목**: 단일 벡터가 전체 입력 표현
2. **긴 시퀀스 성능**: RNN은 매우 긴 시퀀스에서 고심
3. **경직된 정렬**: 많은-대-많은 또는 복잡한 재배열 처리 불가
4. **훈련 비효율성**: 입력 수준에서 병렬화 불가 (순차 RNN 처리)
5. **맥락 취약성**: 입력의 작은 변화는 극적으로 출력 변경 가능

이런 한계들이 주의 메커니즘과 결국 트랜스포머 아키텍처로 이어졌습니다.

## 역사적 영향

Seq2seq 모델 (주의 포함)은 2014-2018년 NLP를 지배했습니다. 이들은 달성했습니다:
- 기계 번역에서 획기적 성능
- 요약에서 최신 결과
- 질의 응답을 위한 강력한 기준선
- 현대 대화 시스템의 기초

아키텍처는 유연함을 증명했으므로 트랜스포머 기반 변형에 영감을 주었고 현대 NLP 아키텍처 이해에 개념적으로 기초가 됩니다.

## 결론

시퀀스-투-시퀀스 모델은 기본적 문제를 해결했습니다: 신경망을 사용하여 가변 길이 입력을 가변 길이 출력으로 처리하는 방법. 인코더-디코더 프레임워크는 일부 작업에서 트랜스포머의 도전을 받고 있지만, 현대 생성 NLP를 이해하는 데 개념적으로 중앙에 있습니다. 교사 강제, 빔 검색, 주의 메커니즘은 seq2seq 연구에서 나타나는 중요한 기법입니다. 특정 RNN 기반 seq2seq 아키텍처가 2024년에 덜 일반적이더라도, 현대 모델은 이 기초 위에 구축됩니다. Seq2seq를 이해하는 것은 현대 NLP와 생성 AI를 파악하는 데 필수적입니다.
