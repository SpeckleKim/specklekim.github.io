# 지식 증류

## 소개

지식 증류(Knowledge Distillation)는 더 크고 정확한 모델(교사)을 모방하도록 더 작고 빠른 모델(학생)이 학습하는 모델 압축 기법입니다. 처음부터 학습하는 대신, 학생 네트워크는 교사의 학습된 표현과 결정 경계를 학습하여 모델 크기, 추론 시간, 메모리 요구 사항을 크게 줄이면서 유사한 성능을 달성합니다.

## 교사-학생 프레임워크

기본 아이디어는 비대칭적 지식 전달입니다:

```
교사 네트워크 (크고 정확함)
        ↓ (지식 전달)
학생 네트워크 (작고 빠름)
```

**교사 모델**:
- 높은 정확도로 학습된 대규모 네트워크
- 풍부한 학습된 표현 보유
- 학습 중에만 사용, 추론 시 폐기
- 더 강력한 지식을 위해 앙상블이 될 수 있음

**학생 모델**:
- 더 작은 네트워크 (10-100배 축소)
- 원본 데이터와 교사 지도 모두로부터 학습
- 프로덕션에서 추론을 위해 배포
- 더 낮은 계산 비용으로 교사와 같은 성능 달성

## 소프트 타겟과 온도 스케일링

**하드 타겟**: 전통적인 학습은 원-핫 인코딩 레이블(각 클래스에 0 또는 1)을 사용합니다.

**소프트 타겟**: 지식 증류는 교사로부터의 확률 분포를 사용합니다:

```
소프트 타겟 = softmax(z_교사 / T)
```

여기서 T는 온도 매개변수입니다. 소프트 타겟은 상대적 클래스 유사성에 대한 정보를 포함합니다—교사가 부정확한 예측에 대해 어떻게 "생각하는지".

**온도 매개변수**:
- T = 1: 표준 softmax (하드 타겟)
- T > 1: 확률 분포를 부드럽게 하여 클래스 간 관계 강조
- 전형적인 범위: T = 3-20 (작업 복잡도에 따라)
- 높은 T는 모든 클래스 확률을 더 유사하게 만들어 세밀한 관계를 드러냅니다

**직관**: "답은 '개'"를 가르치는 대신, 증류는 "답은 0.8 확률의 '개', 0.15의 '늑대', 0.03의 '고양이'"를 가르킵니다—늑대성이 고양이성보다 개성과 더 유사함을 드러냅니다.

## 증류 손실

총 학습 손실은 다음을 결합합니다:

```
L_총합 = α × L_CE(y, σ(z_학생)) + (1-α) × L_KL(σ(z_교사/T), σ(z_학생/T))
```

**구성 요소**:
- 첫 번째 항: 하드 레이블의 교차 엔트로피 (표준 작업 손실)
- 두 번째 항: 교사와 학생 분포 간의 Kullback-Leibler 발산
- α: 두 목표를 균형 맞추는 가중치 (일반적으로 0.5-0.9)
- 온도 T: 두 분포에 사용되는 동일한 값

## 증류 변형

**반응 기반 증류**:
- 학생이 교사의 최종 출력 분포와 일치
- 가장 간단한 형태, 분류에 잘 작동
- 동일한 출력 공간에서의 정렬 필요

**특성 기반 증류**:
- 학생이 교사의 중간 특성 표현과 일치
- 내부 표현의 더 깊은 적응 가능
- 다른 출력 공간이 있는 아키텍처 처리
- 더 유연하지만 정렬 계층 필요

**관계 기반 증류**:
- 학생이 교사가 학습한 데이터 포인트 간의 관계와 일치
- 예: 교사가 샘플 A와 B에 대해 유사한 출력을 예측하면, 학생도 그렇게 해야 함
- 높은 수준의 의미적 관계 캡처
- 메트릭 학습 및 순위 지정 작업에 유용

## 실제 모델 압축 예제

**DistilBERT**:
- BERT-base에서 40% 크기 감소
- 언어 이해 능력의 97% 유지
- 60% 더 빠른 추론
- 증류와 계층 가지치기로 달성

**TinyBERT**:
- 여러 증류 계층 (포함된 학습)
- BERT보다 7-13배 작고, 9-15배 빠름
- 여전히 성능의 약 99% 유지
- 반응 및 특성 기반 증류 모두 사용

**MobileNets**:
- 모바일/엣지 장치용 경량 아키텍처
- 교사: ResNet-101
- 학생: MobileNet
- ResNet 정확도의 95%를 모델 크기의 1/100으로 달성

## 지식 증류의 장점

1. **모델 압축**: 50-90% 크기 감소
2. **속도**: 5-10배 더 빠른 추론
3. **엣지 배포**: 모바일 장치, 임베디드 시스템에 적합
4. **더 낮은 메모리**: RAM 및 캐시 요구 사항 감소
5. **정확도 유지**: 학생이 교사 성능과 일치
6. **앙상블 지식**: 교사 앙상블에서 증류 가능
7. **온라인 학습**: 교사-학생 프레임워크는 지속적인 적응을 가능하게 함

## 엣지 배포 응용

**모바일 장치**:
- 실시간 객체 감지 (YOLO 교사에서 학생)
- 기기 내 NLP (모바일 앱용 DistilBERT)
- 감소된 지연 시간의 얼굴 인식

**IoT 장치**:
- 압축된 모델이 제한된 메모리(킬로바이트에서 메가바이트)에 적합
- 배터리 제약 장치는 더 빠른 추론의 이점
- 엣지 컴퓨팅 요구사항이 있는 센서

**자율 시스템**:
- 차량은 제한된 하드웨어로 실시간 추론이 필요
- 증류된 모델은 낮은 지연 시간의 의사 결정을 가능하게 함
- 극단적 압축을 위해 양자화와 결합

## 고급 증류 기법

**다중 작업 증류**:
- 학생이 교사로부터 여러 관련 작업을 학습
- 공유 표현은 전이를 개선
- 단일 작업 증류보다 더 나은 일반화

**자기 증류**:
- 동일 네트워크의 초기 계층이 후기 계층을 가르침
- 추가 교사 없이 최종 모델 개선
- 반복적으로 적용 가능 (교사 = n 계층, 학생 = m < n 계층)

**교차 모달 증류**:
- 모달리티 간의 지식 전달 (비전에서 언어 또는 그 반대)
- 예: 이미지 설명이 이미지 이해를 안내
- 모델이 상호 보완적 정보 출처를 활용하도록 함

**주의 전달**:
- 학생이 교사로부터 주의 맵 전달
- 교사가 추론 중 어디에 집중하는지 캡처
- 해석 가능성과 성능 개선

## 학습 전략 및 하이퍼매개변수

**학습률**: 종종 표준 학습보다 낮음 (0.0001-0.001)

**온도 선택**:
- T=4로 시작, 성능에 따라 조정
- 더 어려운 (더 유사한) 작업에 대해 더 높은 T
- 검증 세트에서 성능 모니터링

**가중치 α**:
- 일반적으로 0.5-0.9로 증류 손실을 선호
- 높은 값은 교사 행동 매칭 강조
- 낮은 값은 작업별 학습 보존

**조기 종료**: 학생-교사 손실 일치 모니터링

**배치 크기**: 더 큰 배치(128-512)는 종종 안정적인 증류에 유용

## 한계 및 과제

- **교사 품질 의존성**: 학생은 교사 성능을 초과할 수 없음
- **하이퍼매개변수 민감성**: 온도와 가중치 α는 조정 필요
- **항상 유용하지 않음**: 때때로 표준 학습이 마찬가지로 잘 작동
- **추가 학습 시간**: 교사와 학생 모두 학습 필요
- **지식 특수성**: 증류된 지식은 작업별

## 지식 증류를 사용할 시기

**이상적인 시나리오**:
- 자원이 제한된 장치에 배포
- 실시간 추론 필요 (지연 시간 임계)
- 제한된 전력 예산이 있는 엣지 컴퓨팅
- 수백만 사용자를 제공하는 프로덕션 시스템 (비용 감소)

**필요하지 않을 때**:
- 계산 자원이 풍부함
- 모델 크기와 속도가 제약이 아님
- 정확도가 효율성보다 더 중요

## 결론

지식 증류는 모델 정확도와 계산 효율성 간의 간격을 메웁니다. 큰 학습된 모델의 지식을 활용하여 더 작은 학생을 학습함으로써, 표준 깊은 학습이 이전에 불가능했던 장치와 시스템에서 최첨단 성능을 가능하게 합니다. 모델이 더 커지고 배포 시나리오가 더 다양해질수록, 지식 증류는 실용적인 AI 시스템에 점점 더 필수적이 되었습니다.
