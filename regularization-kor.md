# 정규화 기법 (L1, L2, 드롭아웃)

정규화는 분산을 감소시키고 과적합을 방지하기 위한 주요 기법입니다. 학습 목적 함수에 제약 또는 벌칙을 추가하여 작동하며, 모델이 더 잘 일반화되는 더 단순한 해를 찾도록 강제합니다.

## 정규화가 필수적인 이유

정규화 없이 최적화 알고리즘은 훈련 오류를 모든 비용으로든 최소화하려고 합니다. 작은 데이터셋이나 높은 용량의 모델의 경우, 이는 참 패턴을 학습하기보다는 노이즈를 암기합니다. 정규화는 단순한 모델로의 편향을 도입하여 약간의 훈련 정확도를 훨씬 더 나은 일반화로 교환합니다.

정규화된 목적 함수는 다음과 같이 됩니다:
```
손실_전체 = 손실_데이터 + λ * 손실_정규화
```

하이퍼파라미터 λ (람다)는 정규화 강도를 제어합니다. 더 높은 λ는 단순성을 향한 더 강한 제약을 시행합니다.

## L2 정규화 (능선 회귀)

L2 정규화는 모델 가중치의 제곱에 비례하는 벌칙을 추가합니다:
```
손실_L2 = Σ(실제 - 예측)² + λ Σ(w²)
```

**작동 방식:**
- 최적화 중에 알고리즘은 큰 가중치 값을 피합니다
- 특성 전체에 분산된 작은 가중치를 장려합니다
- 가중치가 정확히 0이 되지는 않습니다 (단지 축소됨)

**사용 시기:**
- 모든 특성이 관련이 있다고 믿을 때
- 상관된 많은 특성의 영향을 균형 잡고 싶을 때
- 거의 항상 안전한 범용 정규화

**모델에 미치는 영향:**
- 모든 가중치가 비례적으로 감소
- [신경망](/neural-kor.html)의 더 부드러운 결정 경계
- 계산상 효율적 (선형 모델에는 닫힌 형태 해결책이 존재)

[신경망](/neural-kor.html)에서 L2는 훈련 중에 가중치를 0으로 감소시키기 때문에 **가중치 감쇠**라고도 불립니다.

## L1 정규화 (라소)

L1 정규화는 가중치의 절댓값에 비례하는 벌칙을 추가합니다:
```
손실_L1 = Σ(실제 - 예측)² + λ Σ(|w|)
```

**작동 방식:**
- 원점 주위에 "다이아몬드" 제약 영역 생성
- 해가 제약 영역의 축/경계에 있도록 강제
- 이 기하학은 자연스럽게 많은 가중치를 정확히 0으로 만듭니다

**사용 시기:**
- 특성 선택을 원할 때 (희소 모델)
- 특성의 부분 집합만 중요하다고 생각할 때
- 해석 가능성이 중요할 때

**모델에 미치는 영향:**
- 일부 가중치가 정확히 0이 됨 (자동 특성 선택)
- 0이 아닌 가중치는 클 수 있음 (L2보다 덜 축소됨)
- 관련 없는 특성을 제거하는 데 더 공격적
- L2보다 계산상 더 비용이 많이 듦

L1 정규화는 많은 가중치가 0인 **희소 모델**을 자연스럽게 생성합니다. 이는 모델을 필수 특성으로 축소합니다.

## L1 vs L2: 기하학적 직관

2개의 파라미터 (w1, w2)가 있는 가중치 공간을 생각해봅시다:

**L2 제약:** w1² + w2² ≤ t (원)
- 등고선이 원의 어느 지점에서든 접합니다
- 최적 해는 축에 거의 없음 (가중치는 0이 아님 유지)

**L1 제약:** |w1| + |w2| ≤ t (다이아몬드)
- 다이아몬드의 모서리가 축에 있음
- 등고선이 모서리에서 접할 가능성 (한 가중치 = 0)

이 기하학적 차이는 L1이 희소성을 생성하고 L2는 생성하지 않는 이유를 설명합니다.

## 탄성 넷: L1과 L2 결합

탄성 넷은 두 가지 벌칙을 결합합니다:
```
손실 = Σ(실제 - 예측)² + λ₁ Σ(|w|) + λ₂ Σ(w²)
```

또는 혼합 파라미터 α (0 ≤ α ≤ 1)를 사용하여:
```
손실 = 손실_데이터 + λ[(1-α) Σ(w²)/2 + α Σ(|w|)]
```

**장점:**
- α=0은 순수 L2 회복; α=1은 순수 L1 회복
- 특성 선택(L1)과 가중치 축소(L2) 결합
- 특성이 고도로 상관될 때 L1보다 더 안정적
- 상관된 특성에 대해 유사한 계수를 장려 (그룹화 효과)

**사용 시기:**
- L1과 L2 사이에서 일부 희소성을 원하지만 L1의 불안정성을 두려워할 때
- 고차원 데이터와 상관된 특성의 경우
- L1과 L2 사이에서 불확실할 때 기본 선택

## 드롭아웃: 신경망용 정규화

[드롭아웃](/dropout-kor.html)은 [신경망](/neural-kor.html)에만 해당하는 확률론적 정규화 기법입니다:

**훈련 중:**
- 각 층에서 무작위로 뉴런의 일부 p를 "드롭" (0으로 설정)
- 생존한 뉴런을 통해서만 가중치 업데이트
- 매 반복마다 다른 무작위 부분 집합이 드롭됨

**추론 중:**
- 모든 뉴런을 사용하되 활성화를 (1-p)로 스케일
- 이는 예상 값이 훈련 동작과 일치하도록 보장

**작동 원리:**
- 뉴런의 공동 적응 방지
- 네트워크가 중복 표현을 학습하도록 강제
- 많은 얇은 네트워크의 평균화와 동등
- 모델의 암묵적 [앙상블](/ensemble-methods-kor.html) 생성

**일반적인 드롭아웃 비율:**
- 입력층: 0.1-0.2
- 숨겨진 층: 0.3-0.5
- 출력층: 보통 [드롭아웃](/dropout-kor.html) 없음

**사용 시기:**
- 모든 현대 딥러닝의 표준 관행
- 제한된 데이터의 경우 특히 중요
- 네트워크가 매우 깊을 때 가장 효과적

[드롭아웃](/dropout-kor.html)은 놀랍도록 효과적이고 간단합니다. 거의 모든 [신경망](/neural-kor.html)에 추가합니다.

## 조기 종료: 타이밍을 통한 정규화

조기 종료는 훈련 중 검증 오류를 모니터링하고 증가하기 시작할 때 중지합니다:

**알고리즘:**
1. 데이터를 훈련/검증으로 분할
2. 몇 에포크마다 검증 손실 모니터링
3. N 에포크 동안 검증 손실이 개선되지 않으면 중지
4. 최종 모델이 아닌 최상의 모델(가장 낮은 검증 손실) 사용

**작동 원리:**
- 과적합으로 이어지는 긴 훈련 방지
- 훈련이 패턴을 학습했지만 노이즈를 암기하지 않은 최적점 찾기

**장점:**
- 구현이 간단함
- 모든 모델과 함께 작동
- 해석 가능한 중지 기준 제공
- 계산상 효율적 (추가 비용 없음)

**단점:**
- 별도의 검증 세트 필요 (훈련 데이터 감소)
- 무작위성 도입 (다른 실행은 다른 에포크에서 중지될 수 있음)

**모범 사례:** 조기 종료를 다른 정규화 기법과 결합합니다.

## 데이터 증강을 정규화로

[데이터 증강](/data-augmentation-kor.html)은 변환을 적용하여 인위적으로 데이터셋 크기를 증가시킵니다:

**컴퓨터 비전:**
- 회전, 뒤집기, 자르기, 밝기 변경
- Mixup (두 이미지 혼합)
- CutMix (이미지 간에 패치 붙여넣기)

**자연어 처리:**
- 의역, 역번역
- 동의어로 단어 교체
- 무작위 삽입/삭제

**정규화인 이유:**
- 각 증강 버전은 약간 다름
- 네트워크는 정확한 픽셀/단어를 암기하기보다는 불변성을 학습해야 함
- 더 많은 데이터로 훈련하는 것과 같은 효과, 분산 감소

**장점:**
- 효과적인 데이터셋 크기 증가
- 종종 L1/L2보다 일반화를 더 향상
- 도메인 지식 통합 가능 (어떤 변환이 유효한지)

## 배치 정규화를 정규화로

[배치 정규화](/batch-normalization-kor.html)는 층 입력을 0 평균과 단위 분산을 갖도록 정규화합니다:

```
x_정규화 = (x - 배치_평균) / sqrt(배치_분산 + ε)
```

**정규화 효과:**
- 미니배치 통계의 노이즈가 확률론적 교란 추가
- 내부 공변량 시프트 감소
- 암묵적 정규화 효과 (더 부드러운 손실 경관)
- 더 큰 배치 크기는 정규화 효과 감소

**이점:**
- 훈련 속도 향상 (더 높은 [학습률](/learning-rate-scheduling-kor.html) 허용)
- 깊은 네트워크의 그래디언트 흐름 개선
- 정규화로 작동 ([드롭아웃](/dropout-kor.html) 필요성 감소)

**참고:** [배치 정규화](/batch-normalization-kor.html)의 정규화 효과는 배치 통계가 안정화되는 큰 배치 크기로 감소합니다.

## 실무 정규화 전략

1. **L2로 시작:** 안전한 기본값으로 거의 항상 안전
2. **신경망에 드롭아웃 추가:** 표준 관행
3. **조기 종료 사용:** 항상 검증 성능 모니터링
4. **데이터 증강 고려:** 특히 데이터가 제한적일 때
5. **해석 가능성을 위해 L1 또는 탄성 넷으로 전환:** 특성 선택이 필요할 때
6. **교차검증으로 λ 튜닝:** 정규화 강도는 하이퍼파라미터

## 일반적인 λ 값

- **선형 모델의 L2:** 0.0001 ~ 0.1
- **신경망의 L2:** 1e-6 ~ 1e-3
- **드롭아웃 비율 p:** 숨겨진 층의 경우 0.3 ~ 0.5
- **조기 종료 인내심:** 개선 없음 10-20 에포크

## 일반적인 실수

- **정규화 없음:** 모델이 정규화 없이 과적합
- **너무 많은 정규화:** 모델이 과소적합 (높은 편향)
- **테스트 세트에서 λ 튜닝:** 테스트 추정을 무효화
- **정규화 잊기:** 정규화는 특성이 비슷하게 스케일됨을 가정
- **테스트 시간에 드롭아웃:** 활성화를 (1-p)로 스케일하거나 역 [드롭아웃](/dropout-kor.html) 사용해야 함

정규화의 목표는 완벽한 훈련 정확도가 아니라 좋은 일반화입니다. 일반화가 잘되는 약간 덜 훈련된 모델이 일반화되지 않는 완벽하게 훈련된 모델을 이깁니다.
