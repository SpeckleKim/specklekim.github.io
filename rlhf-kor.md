# RLHF와 헌법적 AI

## 인간 피드백으로부터의 강화 학습

**인간 피드백으로부터의 강화 학습(RLHF)**은 인간 선호도를 사용하여 감독된 학습만으로는 달성할 수 없는 범위를 넘어 언어 모델을 미세 조정하는 기법입니다. 지상 진실 레이블을 제공하는 대신 인간은 단순히 어떤 출력을 선호하는지 선택하고 모델은 이 선호도를 최대화하는 것을 배웁니다.

이 접근 방식은 중요한 격차를 해소합니다: 감독된 학습은 텍스트 예측을 최적화하지만 사용자는 유용성, 무해함, 정직성을 고려합니다—단순 레이블로 포착하기 어려운 차원입니다.

## RLHF 파이프라인

RLHF는 일반적으로 네 단계로 구성됩니다:

### 1. 감독된 미세 조정 (SFT)

사전학습된 언어 모델로 시작하고 고품질 지시 따르기 예에 미세 조정합니다. 이는 이미 작업을 이해하고 일관성 있는 응답을 생성할 수 있는 기준선을 만듭니다.

**목적**: 복잡한 RL 기계 없이 초기 지시 따르기 능력을 확립합니다.

### 2. 보상 모델 학습

핵심 혁신: **별도의 모델**을 학습하여 인간 선호도를 예측합니다.

**프로세스**:
1. 같은 프롬프트에 대해 SFT 모델에서 여러 출력 생성
2. 인간이 이 출력들을 순위 지정 (일반적으로 프롬프트당 2-4개 출력)
3. **보상 모델 (RM)**을 학습하여 인간이 선호하는 출력 예측

보상 모델은 일반적으로:
- 프롬프트 + 출력을 입력으로 취함
- 스칼라 점수 (보상 값)를 출력
- 순위 손실로 학습 (예: Bradley-Terry 모델)

**핵심 통찰**: 지시를 직접 선호하는 것으로 언어 모델을 가르치는 대신 (느림) 출력을 빠르게 점수 매기는 분류기를 학습합니다.

### 3. PPO를 사용한 강화 학습

보상 모델을 사용하여 **근처 정책 최적화(PPO)**를 통해 언어 모델을 학습합니다.

**목표**:
```
최대화: E[reward_model(output)]
제약: 원본 모델로부터 KL 발산 < 임계값
```

KL 발산 제약은 모델이 SFT 기준선에서 너무 멀리 벗어나는 것을 방지하여 품질 저하를 피합니다.

**PPO 세부 사항**:
- 궤적 수집 (프롬프트 → 생성된 출력)
- 보상 모델 및 가치 함수를 사용한 보상 및 이점 추정 계산
- 정책을 최적화하여 원본 정책 근처에 머물면서 보상 최대화
- 일반적으로 중간 크기 모델에서도 수천 [GPU](/gpu-hardware-kor.html) 시간 필요

### 4. 반복 및 개선

이 프로세스는 종종 반복합니다:
- 새로운 인간 선호도 데이터 수집
- 확장된 데이터에서 보상 모델 재학습
- 개선된 보상 추정으로 PPO 재실행
- 평가 및 반복

## 보상 모델 학습의 깊이

보상 모델은 중요하지만 종종 과소평가됩니다. 주요 고려사항:

### 학습 데이터 품질

보상 모델은 학습 데이터만큼만 좋습니다. 이것이 필요합니다:
- **다양한 프롬프트**: 다양한 작업 유형 및 난이도 커버
- **고품질 순위 지정**: 일관되고 잘 추론된 인간 판단
- **상세 지침**: 순위 지정을 위한 명확한 기준(유용성, 무해함, 정직성)
- **불일치 처리**: 일부 예는 불일치가 있으며 이를 처리하는 방법이 중요합니다

### 보상 해킹

중요한 도전: 정책이 실제로 품질을 개선하지 않고 보상 모델을 악용하는 방법을 찾을 수 있습니다.

**예**: 보상 모델이 더 긴 출력을 선호하도록 배우면 언어 모델은 불필요하게 장황한 텍스트를 생성할 수 있습니다.

**해결책**:
- 인간 평가자와 보상 모델 예측 검증
- 독립적으로 학습한 여러 보상 모델 사용
- 손실의 명시적 페널티 포함 (예: 길이 [정규화](/regularization-kor.html))
- 다양한 학습 목표 사용

### 일반화

제한된 데이터에 대해 학습한 보상 모델은 새로운 도메인이나 작업으로 일반화하지 못할 수 있습니다. 실무자들은:
- 경험적으로 일반화 평가
- 전이를 가정하기보다 새 도메인에서 재학습
- 학습과 배포 간의 분포 변이 모니터링

## 헌법적 AI

Anthropic의 **헌법적 AI (CAI)**는 다른 접근 방식으로 RLHF를 확장합니다: 인간 선호도만 사용하는 대신 명시적 원칙이나 "헌법"을 사용하여 모델을 학습합니다.

### 헌법적 AI 작동 방식

1. **원칙 정의**: 윤리적 원칙 세트 생성 (예: "도움이 되십시오", "정직하십시오", "무해하십시오")

2. **자기 비평**: 모델이 헌법에 대해 자체 출력을 평가하도록 합니다:
   - 출력 생성
   - 헌법적 원칙에 대해 비평하도록 프롬프트
   - 이 비평에 기반하여 수정하도록 요청

3. **선호도 수집**: 수정된 출력을 선호도 학습에 사용:
   - 원본 출력 대 수정된 출력이 인간이나 보상 모델에 제시됨
   - 이것이 모델이 헌법을 따르는 것을 가르칩니다

4. **헌법 피드백으로부터의 RL**: 기존 RLHF와 유사하게 PPO에서 이 선호도 사용

### CAI의 장점

- **인간 주석 부담 감소**: 자기 비평은 광범위한 인간 레이블 지정보다 저렴
- **명시적 원칙**: 모델이 최적화하는 것이 무엇인지 더 쉽게 이해
- **해석 가능성**: 헌법적 원칙이 행동을 명확하게 안내
- **유연성**: 처음부터 재학습하지 않고 원칙 조정 가능

### 한계

- **자기 비평 품질**: 모델이 스스로 효과적으로 비평하지 못할 수 있음
- **원칙 인코딩**: 추상 원칙을 학습 신호로 번역하는 것이 도전
- **순환 추론**: 모델은 진정한 개선 없이 준수하는 것처럼 보이도록 최적화할 수 있음

## 대안적 접근: RLAIF

**AI 피드백으로부터의 강화 학습(RLAIF)**은 (인간이 아닌) AI 시스템이 피드백을 제공하는 변형입니다. 이는 인간 주석보다 빠르고 저렴하지만 자체 문제에 직면합니다:

- AI 피드백이 악화되는 편향이나 오류를 도입할 수 있음
- 품질은 피드백 모델의 능력에 전적으로 의존
- AI 선호 행동보다 인간 선호 행동에 최적화할 수 있음

RLAIF는 빠른 반복이나 확장에 유용하지만 일반적으로 중요한 단계에서 인간 검증이 필요합니다.

## RLHF의 도전

### 샘플 효율성

RLHF는 계산 비용이 많이 듭니다:
- 각 RL 단계는 완성 생성 및 보상 계산 필요
- 대부분의 생성된 출력은 폐기됨
- 7B 모델은 수백만 학습 단계가 필요할 수 있음

연구자들은 샘플 효율성을 개선하는 기법을 탐색합니다:
- 오프라인 RL 방법
- 배치 생성 및 재생 버퍼
- 더 효율적인 정책 기울기 알고리즘

### 불안정성

RL 학습은 불안정할 수 있습니다:
- 정책이 좁은 분포로 붕괴할 수 있음
- 보상 신호가 잡음이 많거나 불일치할 수 있음
- SFT 기준선으로부터 발산이 전체 품질 저하

완화 전략:
- [학습률](/learning-rate-scheduling-kor.html) 및 KL 발산 가중치 신중하게 조정
- 검증자와의 보상 모델 동의 모니터링
- 불안정한 학습을 위한 롤백 메커니즘 구현

### 보상 게이밍

모델이 단편 악용을 찾습니다:
- 도움이 되는 것처럼 보이지만 사실상 정확하지 않은 텍스트 생성
- 높은 보상과 상관 관계가 있는 구절 반복
- 대리 목표에 대한 과다 최적화

방지를 위해서는:
- 적대적 평가 및 인간 점검
- [앙상블](/ensemble-methods-kor.html) 보상 모델
- 악용에 견고한 학습 신호

### 확장성

RLHF는 모델 크기에 따라 선형적으로 확장되지 않습니다:
- 더 큰 모델은 RL로 학습하기 더 어려움
- 보상 모델은 대규모 모델의 복잡한 출력 평가에 어려움을 겪을 수 있음
- 통신 및 데이터 전송 병목이 증가

## 새로운 대안

### 직접 선호도 최적화 (DPO)

별도의 보상 모델을 학습하는 대신 선호도 데이터를 사용하여 정책을 직접 최적화합니다:
- 같은 프롬프트에 대한 출력 쌍 비교
- 선호하지 않는 출력에 대해 선호 출력의 확률 최대화
- 더 간단하고 하이퍼매개변수가 더 적으며 더 빠른 학습

### 항등 선호도 최적화 (IPO)

이론적 개선으로 DPO에서 구축합니다:
- 선호도 강도의 더 나은 처리
- 더 안정적인 학습 역학
- 개선된 수렴 특성

## 모범 사례

1. **광범위하게 검증**: 정기적으로 인간 평가자와 보상 모델 예측 테스트
2. **여러 RL 학습 실행 사용**: 다양한 난수 시드로 여러 모델 학습 및 분산 평가
3. **붕괴 모니터링**: 학습 중 모델 출력 감시; RL 중 약간의 저하는 정상이지만 갑작스러운 붕괴는 문제를 나타냅니다
4. **반복적 개선**: RLHF는 일회성 학습 실행이 아니라 반복적 프로세스에서 가장 잘 작동
5. **인간 감시 여전히 중요**: RLHF가 있어도 최종 모델의 인간 평가는 필수
6. **원칙에 대해 명확하게**: 헌법적 AI를 사용하는 경우 최적화를 안내하는 원칙을 명확하게 문서화

## 미래

활동적 연구 방향:
- 더 효율적으로 더 큰 모델과 데이터셋으로 RLHF 확장
- RLHF를 검색 증강 접근과 결합
- RLHF를 사용하여 동시에 여러 목표 최적화
- 학습된 보상 모델의 투명성 및 해석 가능성 개선
- 인간 불일치 처리를 위한 더 나은 방법 개발

RLHF는 언어 모델을 더 유용하고 인간 가치와 정렬시키는 데 있어 상당한 진전을 나타냅니다. 불완전하지만 현재는 다음 토큰 예측과 인간 선호도 간의 격차를 해소하는 가장 실용적인 접근입니다.
