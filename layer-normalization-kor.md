# 레이어 정규화: 트랜스포머 아키텍처의 기초

레이어 [정규화](/regularization-kor.html)(Layer Normalization, LayerNorm)는 현대 딥러닝에서 가장 중요한 요소 중 하나가 되었으며, 특히 [트랜스포머](/llm-kor.html) 모델에서 핵심적인 역할을 합니다. [배치 정규화](/batch-normalization-kor.html)가 배치 차원에서 정규화하는 것과 달리, 레이어 정규화는 각 샘플에 대해 독립적으로 특성 차원에서 정규화합니다.

## 배치 정규화의 문제점

[배치 정규화](/batch-normalization-kor.html)는 배치 차원 전체에서 통계(평균과 분산)를 계산하며, 여러 시나리오에서 문제가 발생합니다:

- **변수 배치 크기**: 추론 시에는 보통 하나의 예제를 처리하므로 배치 통계 가정이 깨집니다
- **순환신경망과 수열 모델**: [배치 정규화](/batch-normalization-kor.html)는 가변 길이의 수열에서 제대로 작동하지 않습니다
- **NLP 응용**: 다양한 수열 길이는 배치 통계를 신뢰하기 어렵게 만듭니다
- **GPU 메모리 제약**: 안정적인 통계를 위해 더 큰 배치가 필요합니다

이러한 한계들은 다른 [정규화](/regularization-kor.html) 방식의 개발을 촉발했습니다.

## 레이어 정규화의 작동 원리

LayerNorm은 각 샘플에 대해 독립적으로 특성 차원(d) 전체에서 평균과 분산을 계산합니다:

```
μ_i = (1/d) * Σ(x_ij)  샘플 i의 모든 특성 j에 대해
σ_i² = (1/d) * Σ(x_ij - μ_i)²

y_ij = γ * (x_ij - μ_i) / √(σ_i² + ε) + β
```

주요 특성:
- 각 샘플에 독립적으로 작동
- 배치 차원이 아닌 특성 차원에서 [정규화](/regularization-kor.html)
- 학습 가능한 매개변수 γ(스케일)와 β(시프트) 포함
- 학습과 추론 시간에 동일한 계산
- 배치 크기나 다른 예제에 대한 의존성 없음

## 트랜스포머가 레이어 정규화를 사용하는 이유

[트랜스포머](/llm-kor.html)는 다음의 설득력 있는 이유들로 LayerNorm을 기본 [정규화](/regularization-kor.html) 방식으로 채택했습니다:

1. **배치 크기 독립성**: 배치 크기와 관계없이 동일하게 작동하며, 가변 길이 수열에 필수적
2. **훈련 안정성**: 배치 통계 변동성 없이 일관된 기울기 흐름 제공
3. **수열 모델 호환성**: [어텐션 메커니즘](/attention-mechanism-kor.html)과 가변 길이 입력에서 완벽하게 작동
4. **Post-LN vs Pre-LN**: 훈련 동역학을 개선하기 위한 다양한 아키텍처 변형 가능
5. **분산 훈련**: 동기화 오버헤드 없이 분산 설정에서 더 간단하게 구현

## 변형과 대안들

**RMSNorm** (제곱평균제곱근 [정규화](/regularization-kor.html)):
- 중심화 단계를 제거하고 분산만 사용하여 [정규화](/regularization-kor.html)
- 더 간단한 계산과 약간의 효율성 개선
- LLaMA 같은 최근 모델에 채택됨
- 공식: y = x / √(mean(x²) + ε) * γ

**그룹 정규화**:
- 채널을 그룹으로 나누고 각 그룹 내에서 [정규화](/regularization-kor.html)
- [배치 정규화](/batch-normalization-kor.html)의 배치 크기 의존성 해결
- 배치 크기가 제한적일 때 유용
- LayerNorm과 InstanceNorm의 중간적 성질

**인스턴스 정규화**:
- 각 샘플에 대해 채널별로 독립적으로 [정규화](/regularization-kor.html)
- 스타일 전이와 [이미지 생성](/image-generation-kor.html)에서 자주 사용
- 콘텐츠를 보존하면서 스타일 정보 제거
- 판별 작업에서는 너무 공격적일 수 있음

## 실무에서의 레이어 정규화

LayerNorm을 구현할 때:

1. **수치 안정성**: 분산에 작은 엡실론(ε ≈ 1e-6)을 더해 0으로 나누기 방지
2. **초기화**: γ = 1, β = 0으로 초기화 (항등 함수)
3. **배치**: Pre-LN (부분층 전)과 Post-LN (부분층 후) 중 선택에 따라 최적화 영향
4. **통합**: 보통 [트랜스포머](/llm-kor.html) 블록에서 [어텐션](/attention-mechanism-kor.html)과 피드포워드 층 후에 적용

## Pre-LN과 Post-LN

**Post-LN** (원본 [트랜스포머](/llm-kor.html)):
- 잔여 연결 후에 적용
- 깊은 네트워크에서 [기울기 폭발](/vanishing-gradients-kor.html) 가능성
- 더 신중한 [학습률](/learning-rate-scheduling-kor.html) 조정 필요
- 적절한 워밍업으로 더 나은 성능

**Pre-LN**:
- 부분층 전에 적용
- 더 안정적인 기울기, 훈련 용이
- 폭발 없이 더 깊은 네트워크 가능
- 최근 아키텍처의 표준

## 다른 정규화와의 비교

| 방법 | 독립성 | 속도 | 배치 민감도 | 사용 사례 |
|------|--------|------|-----------|----------|
| [배치 정규화](/batch-normalization-kor.html) | 배치 의존 | 빠름 | 높음 | [CNN](/cnn-deep-dive-kor.html), 대형 배치 |
| 레이어 [정규화](/regularization-kor.html) | 샘플 의존 | 느림 | 없음 | [트랜스포머](/llm-kor.html), RNN |
| 그룹 [정규화](/regularization-kor.html) | 그룹 의존 | 중간 | 낮음 | 작은 배치, 이미지 |
| RMSNorm | 샘플 의존 | 가장 빠름 | 없음 | 대형 언어 모델 |
| 인스턴스 [정규화](/regularization-kor.html) | 완전 독립 | 중간 | 없음 | 스타일 전이 |

## 수학적 직관

LayerNorm은 다음과 같은 방식으로 훈련을 안정화합니다:

1. 신경원 입력의 분포 변화(내부 공변량 시프트) 감소
2. 최적화를 쉽게 하기 위한 손실 경관 평활화
3. [정규화](/regularization-kor.html)를 통한 정규화 효과 제공
4. 발산 없이 더 높은 [학습률](/learning-rate-scheduling-kor.html) 가능

## 일반적인 구현 문제

- **엡실론 값**: 너무 작으면 불안정, 너무 크면 [정규화](/regularization-kor.html) 효과 제거
- **기울기 흐름**: Pre-LN이 일반적으로 Post-LN보다 기울기 성질이 더 좋음
- **학습 가능한 매개변수**: γ와 β는 중요하며 고정하지 말 것
- **혼합 정밀도 훈련**: float16 연산에서는 더 큰 엡실론 필요할 수 있음

## 결론

레이어 [정규화](/regularization-kor.html)는 현대 아키텍처, 특히 [트랜스포머](/llm-kor.html)에서 선택의 정규화 방법으로 입증되었습니다. 배치 크기 독립성과 안정성 특성은 순차 및 가변 길이 데이터에서 [배치 정규화](/batch-normalization-kor.html)보다 우수합니다. LayerNorm의 메커니즘—배치가 아닌 특성 차원에서 정규화—을 이해하면 이것이 이러한 영역에서 왜 그렇게 잘 작동하는지에 대한 통찰력을 얻을 수 있습니다.

RMSNorm과 다른 변형으로의 진화는 효율성과 성능을 계속 개선하고 있지만, LayerNorm은 여전히 [트랜스포머](/llm-kor.html) 혁명을 가능하게 한 기초적인 기술로 남아 있습니다.
