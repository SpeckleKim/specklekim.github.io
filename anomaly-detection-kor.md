# 이상 탐지: 데이터의 비정상 패턴 식별

이상 탐지는 비정상 사건, 부정 행위, 시스템 장애를 식별하는 데 중요합니다. 이 가이드는 이상의 유형, 탐지 방법, 알고리즘, 그리고 실제 응용을 다룹니다.

## 이상의 유형

**포인트 이상:**
- 정상 행동에서 크게 벗어나는 개별 데이터 점
- 가장 일반적인 유형
- 예: 비정상적 거래액, 센서 판독값 스파이크
- 개별 점을 전체 분포와 비교하여 탐지

**문맥적 이상:**
- 특정 문맥에서 비정상이지만 전역적으로 정상인 점
- 문맥은 시간, 공간 또는 다른 속성으로 정의
- 예: 자정의 높은 온도, 새벽 3시의 비싼 구매
- 문맥별 정상 행동 이해 필요

**집단 이상:**
- 개별적으로 정상이지만 집단적으로 비정상인 데이터 점의 부분집합
- 관계 및 패턴 검사로 탐지
- 예: 작은 거래 시퀀스(자금 세탁), 조율된 네트워크 트래픽
- 자동으로 식별하기 가장 어려움

## 통계적 방법

**Z-점수 방법:**
- 데이터가 정규 분포를 따른다고 가정
- Z-점수 계산: (x - 평균) / 표준 편차
- |z-점수| > 3인 점은 이상 (신뢰도 99.7%)
- 간단하고 해석 가능하지만 정규성 가정

**고립 포레스트:**
- 고차원 데이터에 효과적
- 무작위로 특성 및 분할 값 선택
- 더 적은 분할로 이상 고립 (이상은 고립됨)
- 계산 복잡도: 트리당 O(n log n)

**고립 포레스트 작동 원리:**
1. 특성을 무작위로 선택
2. 최소/최대 사이의 분할 값을 무작위로 선택
3. 데이터를 재귀적으로 분할
4. 이상은 고립하기 위해 더 적은 분할 필요
5. 이상 점수: 고립까지의 평균 경로 길이

**장점:**
- 거리 메트릭 불필요
- 혼합 데이터 유형 처리
- 효율적이고 확장 가능
- 부분표본 크기에 선형 시간 복잡도

## One-Class SVM

One-Class SVM은 대부분의 데이터를 포함하는 최소 초구(hypersphere)를 찾습니다. 초구 외부의 점은 이상입니다.

**핵심 아이디어:**
- 정상 데이터 주변의 결정 경계 학습
- 경계에서 멀리 떨어진 점은 이상
- 비선형 커널(RBF)은 복잡한 분포 처리
- Nu 파라미터는 이상치 비율 제어 (보통 0.01-0.1)

**장점:**
- 비선형 경계 처리
- 중간 차원에서 효과적
- 잘 정립된 통계 이론
- 해석 가능한 결정 경계

**제한 사항:**
- 특성 척도 조정 및 커널 선택에 민감
- 계산 비용이 많음: O(n²)에서 O(n³)
- 매우 높은 차원에서 성능 저하
- 파라미터 튜닝에 검증 데이터 필요

## 국소 이상인수 (LOF)

LOF는 이웃과의 국소 밀도를 비교하여 밀도 기반 이상을 식별합니다.

**핵심 개념:**
- 정상 점은 이웃과 유사한 밀도
- 이상은 훨씬 낮은 밀도
- LOF 점수: 이웃 밀도 대 자신의 밀도 비
- LOF ≈ 1은 정상; LOF >> 1은 이상 나타냄

**알고리즘:**
1. 각 점의 도달 가능성 거리 계산
2. 국소 도달 가능성 밀도(LRD) 계산
3. 각 점의 LOF 계산
4. LOF > 임계값은 이상

**장점:**
- 밀도 기반 이상 효과적으로 탐지
- 다양한 밀도 영역 처리
- 분포 가정 없음
- 국소 패턴 캡처

**제한 사항:**
- 계산 비용이 많음: O(n²)
- K-최근 이웃 파라미터에 민감
- 높은 차원에서 해석 어려움
- 임계값 설정 검증 필요

## 오토인코더 기반 탐지

오토인코더는 정상 데이터의 압축된 표현을 학습하여 낮은 오류로 재구성합니다. 이상은 재구성이 잘못됩니다.

**구조:**
- 인코더: 입력을 잠재 표현으로 압축
- 디코더: 잠재 코드에서 원본 입력 재구성
- 병목: 좁은 중간 층으로 유용한 표현 강제
- 정상 데이터는 잘 압축/해제; 이상은 그렇지 않음

**훈련 프로세스:**
1. 정상 데이터만으로 훈련
2. 재구성 오류 최소화
3. 오토인코더는 정상 데이터 패턴 학습
4. 높은 재구성 오류의 테스트 데이터는 이상

**장점:**
- 복잡한, 고차원 패턴 처리
- 수동 [특성 공학](/feature-engineering-kor.html) 없이 자동 학습
- 순차 데이터 처리 가능 (LSTM 오토인코더)
- 다양한 데이터 유형을 위한 유연한 구조

**제한 사항:**
- 훈련을 위해 대량의 정상 데이터 필요
- 일부 이상을 재구성하도록 학습 가능
- 구조 튜닝 어려움
- "블랙 박스" 해석 문제

## 실무 응용

**사기 탐지:**
- 신용 카드 거래: 비정상적 금액, 상점, 타이밍 탐지
- 여러 신호 결합: 위치, 구매 이력, 사용자 행동
- 종종 여러 알고리즘을 결합한 [앙상블](/ensemble-methods-kor.html) 사용
- 실시간 탐지는 고객 경험에 중요

**제조 품질 관리:**
- 생산 중 불량 제품 탐지
- 센서 데이터 이상은 장비 문제 나타냄
- 예방적 유지 보수로 비용 절감
- 고립 포레스트 및 LOF는 센서 데이터에 효과적

**사이버 보안 및 네트워크 침입:**
- 비정상적 네트워크 트래픽 패턴 탐지
- 손상된 시스템 또는 무단 접근 식별
- 집단 이상은 종종 조율된 공격 드러냄
- 도메인 전문가 지식과 결합

**의료 모니터링:**
- 환자 생체 신호의 비정상적 판독값 모니터링
- 문맥적 이상은 중요 (환자의 기준선에 대해 비정상)
- 조기 질병 탐지를 위한 ECG/EEG 이상 탐지
- 오토인코더 모델은 시계열 데이터에 효과적

**센서 네트워크 모니터링:**
- 산업용 IoT 시스템은 지속적 모니터링 필요
- 시간 종속성을 가진 여러 센서
- 장애 발생 전에 장비 장애 탐지
- 고차원 데이터는 효율적 알고리즘 필요

## 평가 지표

**정밀도:**
- 탐지된 이상 중 참 이상의 비율
- 거짓 긍정 감소
- 거짓 양성이 비용이 많을 때 중요

**재현율:**
- 모든 참 이상 중 탐지된 이상의 비율
- 실제 이상을 놓치지 않음
- 이상을 놓치는 것이 비용이 많을 때 중요

**ROC-AUC:**
- 참 양성 비율 대 거짓 양성 비율 플롯
- 곡선 아래 영역은 전반적 성능 측정
- 임계값 독립적 평가
- 불균형 데이터셋에 좋음

**F1-점수:**
- 정밀도 및 재현율의 조화 평균
- 두 지표 균형
- 비교를 위한 단일 숫자

## 이상 탐지의 도전

**레이블 불균형:**
- 이상은 드물고 (데이터의 0.1-1%)
- 불균형 데이터셋은 일반적 지표를 혼동
- 정확도 대신 정밀-재현 곡선 사용
- 합성 오버샘플링 또는 비용 민감 학습 도움

**개념 표류:**
- 정상 행동은 시간에 따라 변함
- 사전훈련된 모델은 구식됨
- 주기적 재훈련 필요
- 온라인 학습 접근은 지속적으로 적응

**차원의 저주:**
- 높은 차원에서 거리 메트릭은 신뢰성 없음
- 많은 알고리즘이 가늠하지 못함
- 특성 선택 또는 차원성 감소 도움
- 고립 포레스트 및 오토인코더는 더 탄력적

**레이블이 없는 데이터:**
- 참 이상은 드물고 레이블 지정 어려움
- 도메인 전문가는 비용이 많을 수 있음
- 반지도 접근은 제한된 레이블 사용
- 레이블 없는 이상의 검증 전략 중요

## 올바른 접근 선택하기

**통계적 방법 사용 시:**
- 데이터가 저차원
- 정상 행동이 잘 이해됨
- 해석 가능성이 중요
- 간단하고 빠른 기준선 필요

**고립 포레스트 사용 시:**
- 고차원 데이터
- 실시간 탐지 필요
- 계산 효율성이 중요
- 혼합 데이터 유형 존재

**One-Class SVM 사용 시:**
- 중간 차원성
- 복잡한 결정 경계 필요
- 계산 리소스 사용 가능
- 도메인 특정 커널 유용

**LOF 사용 시:**
- 밀도 기반 이상이 중요
- 다양한 밀도 영역
- 중간 크기 데이터셋
- 국소 문맥이 중요

**오토인코더 사용 시:**
- 매우 고차원 데이터
- 정상 행동의 복잡한 패턴
- 풍부한 정상 훈련 데이터 사용 가능
- 순차/시계열 데이터

## 결론

이상 탐지는 데이터 특성, 이상의 유형, 탐지 요구사항을 이해하는 것이 필요합니다. 보편적 솔루션은 없으며, 최선의 접근은 데이터 차원성, 계산 제약, 응용 요구사항에 따라 다릅니다. [앙상블](/ensemble-methods-kor.html) 접근을 통해 여러 방법을 결합하고, 현실적 시나리오에서 철저하게 검증하며, 정상 행동이 진화함에 따라 성능을 지속적으로 모니터링하세요.
