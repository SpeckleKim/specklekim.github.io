# 옵티마이저: 기울기 기반 학습 알고리즘

옵티마이저는 [손실 함수](/loss-functions-kor.html)를 최소화하기 위해 기울기를 기반으로 모델 매개변수를 업데이트합니다. 올바른 옵티마이저 선택은 수렴 속도, 안정성, 일반화에 영향을 미칩니다. 특성을 이해하면 문제에 맞는 알고리즘을 선택할 수 있습니다.

## 확률적 경사 하강법 (SGD)

SGD는 기초적인 옵티마이저로 기울기의 반대 방향으로 매개변수를 업데이트합니다:

`W_새 = W_기존 - [학습률](/learning-rate-scheduling-kor.html) * dL/dW`

**특징**: 간단함, 견고함, 신중한 튜닝으로 효과적.

**장점**: 해석 가능한 업데이트. 일반화가 적응형 방법보다 종종 더 좋음. 보편적 기준.

**단점**: 느린 수렴. 신중한 [학습률](/learning-rate-scheduling-kor.html) 튜닝 필요. 학습률에 높은 민감도.

**사용 사례**: 고전적 기준. 충분한 데이터 있는 문제.

## 모멘텀을 가진 SGD

모멘텀은 시간에 따라 기울기 정보를 누적하여 수렴을 가속화합니다:

`v = β*v + (1-β)*dL/dW` (속도 누적)
`W_새 = W_기존 - [학습률](/learning-rate-scheduling-kor.html) * v`

**특징**: 노이즈 많은 기울기를 부드럽게 함. 일관된 방향으로 가속.

**장점**: 바닐라 SGD보다 빠른 수렴. 모멘텀=0.9일 때 더 좋음.

**단점**: 추가 하이퍼파라미터 β. 여전히 [학습률](/learning-rate-scheduling-kor.html) 튜닝 필요.

**사용 사례**: 대부분의 문제. 고전 딥러닝의 표준.

## Nesterov 가속 기울기

Nesterov는 미리 보기로 모멘텀을 개선합니다:

`W_미리보기 = W_기존 - [학습률](/learning-rate-scheduling-kor.html) * v`
`v = β*v + (1-β)*dL/dW|_{W_미리보기}`

**특징**: 미래를 보는 모멘텀. 더 좋은 수렴 특성.

**장점**: 표준 모멘텀보다 빠름. 이론적 보장.

**단점**: 구현이 약간 복잡. 모멘텀보다 실질적 개선 미미.

**사용 사례**: 모멘텀이 부족할 때. 일부 [컴퓨터 비전](/cv-kor.html) 작업.

## Adagrad

Adagrad는 기울기 이력에 따라 매개변수별 [학습률](/learning-rate-scheduling-kor.html)을 적응시킵니다:

`g_누적 = g_누적 + (dL/dW)²`
`W_새 = W_기존 - ([학습률](/learning-rate-scheduling-kor.html) / √g_누적) * dL/dW`

**특징**: 시간에 따라 [학습률](/learning-rate-scheduling-kor.html)이 감소. 매개변수별 학습률.

**장점**: 희소 기울기에서 잘 작동. 튜닝 필요 적음.

**단점**: [학습률](/learning-rate-scheduling-kor.html)이 단조적으로 감소. 너무 작아질 수 있음.

**사용 사례**: 희소 데이터. 희소 단어 임베딩이 있는 NLP.

## RMSprop

RMSprop은 제곱 기울기의 지수 이동 평균을 사용하여 Adagrad의 감소를 해결합니다:

`v = β*v + (1-β)*(dL/dW)²`
`W_새 = W_기존 - ([학습률](/learning-rate-scheduling-kor.html) / √v) * dL/dW`

**특징**: 단조적 감소 없는 적응형 [학습률](/learning-rate-scheduling-kor.html).

**장점**: 실무에서 잘 작동. 비정상 문제 처리.

**단점**: 튜닝할 하이퍼파라미터 β. 이론적 근거 약함.

**사용 사례**: 순환 [신경망](/neural-kor.html). 범용 적응형 최적화.

## Adam (적응형 모멘트 추정)

Adam은 모멘텀과 RMSprop을 결합하여 1차 및 2차 모멘트 추정을 유지합니다:

`m = β₁*m + (1-β₁)*dL/dW` (1차 모멘트)
`v = β₂*v + (1-β₂)*(dL/dW)²` (2차 모멘트)
`m_보정 = m / (1-β₁^t)` (편향 보정)
`v_보정 = v / (1-β₂^t)` (편향 보정)
`W_새 = W_기존 - [학습률](/learning-rate-scheduling-kor.html) * m_보정 / (√v_보정 + ε)`

**특징**: 모멘텀과 RMSprop의 장점 결합. 편향 보정.

**장점**: 빠른 수렴. 다양한 문제에서 작동. 최소 튜닝 필요.

**단점**: SGD보다 일반화가 나을 수 있음. 높은 메모리 사용.

**사용 사례**: 많은 작업의 기본 옵티마이저. 딥러닝에 좋음.

## AdamW (분리된 가중치 감쇠가 있는 Adam)

AdamW는 가중치 감쇠를 기울기 기반 업데이트로부터 분리합니다:

`W_새 = W_기존 - [학습률](/learning-rate-scheduling-kor.html) * m_보정 / (√v_보정 + ε) - 학습률 * λ * W_기존`

**특징**: 적절한 L2 [정규화](/regularization-kor.html). Adam + 가중치 감쇠 문제 해결.

**장점**: Adam보다 더 나은 일반화. 현대 딥러닝의 표준.

**단점**: 하이퍼파라미터 λ(가중치 감쇠). 약간 더 복잡.

**사용 사례**: 현대 깊은 네트워크. 최첨단 모델. Adam보다 권장.

## LAMB (배치 훈련을 위한 계층 적응형 모멘트 옵티마이저)

LAMB는 계층별 [학습률](/learning-rate-scheduling-kor.html) 적응으로 AdamW를 확장합니다:

`업데이트 = m_보정 / (√v_보정 + ε)`
`신뢰 비율 = ||W|| / ||업데이트||` (계층별 적응)
`W_새 = W_기존 - [학습률](/learning-rate-scheduling-kor.html) * 신뢰 비율 * 업데이트`

**특징**: 계층별로 업데이트 [정규화](/regularization-kor.html). 대규모 배치 훈련에 더 좋음.

**장점**: 대규모 배치 훈련 가능. 거대한 데이터셋으로 확장.

**단점**: 복잡함. 계층별 통계의 오버헤드.

**사용 사례**: 대규모 [분산 훈련](/distributed-training-kor.html). 대규모 자연어 처리.

## 학습률과 수렴

[학습률](/learning-rate-scheduling-kor.html)은 모든 옵티마이저에 걸쳐 중요합니다:
- **너무 높음**: 발산, 진동, 불안정성
- **너무 낮음**: 느린 수렴, 국소 최솟값에 갇힘
- **최적점**: 빠른 수렴과 안정성

**학습률 스케줄**은 시간에 따라 [학습률](/learning-rate-scheduling-kor.html)을 감소시킵니다:
- **스텝 감쇠**: 주기적으로 인수만큼 감소
- **지수 감쇠**: 지수 감소
- **코사인 어닐링**: 부드러운 코사인 기반 감소
- **워밍업**: 초기에 증가한 후 감소

대부분의 현대 프레임워크는 자동으로 지원합니다.

## 옵티마이저 선택

**일반 딥러닝**: AdamW로 시작. 속도와 안정성의 균형.

**수렴이 느릴 때**: SGD와 모멘텀 시도. 더 나은 일반화.

**대규모 배치 훈련**: LAMB 사용. 안정성 유지.

**희소 기울기 또는 NLP**: RMSprop 또는 Adam 잘 작동.

**의심스러울 때**: AdamW는 안전한 실무 기본값. 분야 전반에서 작동.

## 요약

현대 최적화는 단순한 SGD에서 Adam과 AdamW 같은 적응형 방법으로 진화했습니다. 각 옵티마이저는 수렴 속도, 안정성, 일반화 사이의 균형을 맞춥니다. 대부분의 현대 딥러닝에서 AdamW는 탁월한 균형을 제공합니다. 다양한 옵티마이저의 특성을 이해하면 문제 특성과 계산 제약에 기반하여 정보에 입각한 선택을 할 수 있습니다.
