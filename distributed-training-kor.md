# 분산 학습

## 소개

분산 학습(Distributed Training)은 여러 GPU, TPU 또는 머신에 걸쳐 신경망 학습을 병렬화하여 학습 시간을 줄이고 단일 기기 메모리를 초과하는 모델의 학습을 가능하게 합니다. 모델이 수백만에서 수십억 개의 매개변수로 성장함에 따라, 분산 학습은 실용적인 깊은 학습에 필수적이 되었습니다. 이 가이드는 분산 학습을 가능하게 하는 기본 전략, 통신 패턴, 현대 시스템을 다룹니다.

## 기본적인 학습 병렬화 전략

학습을 분산하는 세 가지 주요 접근 방식이 있으며, 종종 결합됩니다:

**데이터 병렬화**:
- 각 기기에서 모델 복제
- 각 기기는 다른 데이터 배치 처리
- 기기 전체에서 그래디언트 집계 (AllReduce)
- 가장 간단한 접근, ~8개 GPU까지 잘 확장
- 단일 GPU에 맞는 모델 크기로 제한

**모델 병렬화**:
- 모델을 기기 전체에 분할
- 각 기기는 모델 매개변수의 부분집합 보유
- 활성화/그래디언트는 기기 간에 통과
- 단일 GPU 메모리를 초과하는 큰 모델 학습 가능
- 더 높은 통신 오버헤드, 더 복잡함

**파이프라인 병렬화**:
- 모델을 깊이 차원을 따라 스테이지로 분할
- 각 기기는 하나의 스테이지 처리
- 스테이지는 배치를 효율성을 위해 파이프라인화
- 버블 타임 감소 (기기 유휴 대기)
- 비동기 실행으로 큰 모델 가능

## 데이터 병렬화 상세

**기본 접근**:
1. 각 N개 기기에 모델 복사
2. 데이터 분배: 각 기기는 batch_size/N 샘플 받음
3. 각 기기는 독립적으로 전진 패스와 그래디언트 계산
4. 모든 기기에 걸쳐 그래디언트 집계 (AllReduce)
5. 모든 기기는 집계 그래디언트로 모델 업데이트

**동기 vs 비동기**:

*동기*:
- 모든 기기가 업데이트 전에 반복 완료
- 느린 기기가 전체 속도 결정
- 일관된 수렴 특성
- 프로덕션에서 표준

*비동기*:
- 기기는 최신 매개변수로 독립적으로 업데이트
- 전체적으로 더 빠른 처리량
- 잠재적 오래됨: 일부 기기는 오래된 매개변수 사용
- 신중한 튜닝 없이 발산할 수 있음

**스케일링 효율성**:
```
속도향상 = min(N, 효율_인수 × N)
```

효율_인수는 다음을 나타냅니다:
- 통신 오버헤드
- 로드 불균형
- 동기화 비용

선형 스케일링 (효율 = 1)이 이상적이지만 8개 기기를 초과하면 거의 달성되지 않습니다.

## 모델 병렬화 전략

**텐서 병렬화**:
- 가중치 행렬을 기기 전체에 차원을 따라 분할
- 예: 선형 계층 Y = XW의 경우, W를 수직(열) 또는 수평(행)으로 분할
- 매 행렬 곱셈마다 통신 필요
- 세밀한 병렬화

**시퀀스 병렬화** (Transformer용):
- 시퀀스 차원을 따라 분할
- 다른 기기는 다른 토큰 처리
- 시퀀스 길이에 비례하여 기기당 메모리 감소
- 긴 시퀀스에 효과적

**파이프라인 병렬화 구현**:
```
기기 0: 계층 1-6
기기 1: 계층 7-12
기기 2: 계층 13-18

배치 분할: [미니배치 1, 미니배치 2, 미니배치 3, ...]

타임라인:
기기 0: F(1) → F(2) → F(3) → B(1) → B(2) → B(3)
기기 1: _ → F(1) → F(2) → F(3) → B(1) → B(2)
기기 2: _ → _ → F(1) → F(2) → F(3) → B(1)

(F=전진, B=역진, 숫자는 미니배치를 나타냄)
```

이상적: 초기 기기가 역전파할 때 이후 기기가 이미 다음 미니배치 처리 중.

## 현대적 분산 학습 프레임워크

**완전히 분산된 데이터 병렬화(FSDP)** (PyTorch):
- 모델을 샤드로 분할, 각 샤드는 한 기기에
- 전진 패스: 샤드 매개변수 수집, 계산, 사용 후 폐기
- 역진 패스: 샤드 매개변수 다시 수집, 그래디언트 계산, 폐기
- 샤드당 옵티마이저 상태 (복제 안 함)
- 메모리를 3N에서 3+N/K로 감소 (K = 기기 수)

**DeepSpeed ZeRO** (Microsoft):
- **Stage 1**: 옵티마이저 상태 샤딩 (4배 감소)
- **Stage 2**: 추가로 그래디언트 샤딩 (2배 감소)
- **Stage 3**: 추가로 매개변수 샤딩 (K배 감소)
- 결합: 8개 기기로 약 16배 메모리 감소
- CPU 오프로딩 포함: 추가 감소를 위해 CPU RAM으로 유출

**Megatron-LM** (NVIDIA):
- 파이프라인과 텐서 병렬화 결합
- 파이프라인 병렬화: 스테이지는 다른 기기에
- 텐서 병렬화: 각 스테이지 내에서 행렬 분할
- 큰 언어 모델 학습에 매우 효율적

## 통신 패턴

**AllReduce 연산**:
```
이전:
기기 0: [1, 2]
기기 1: [3, 4]
기기 2: [5, 6]

AllReduce(Sum) 이후:
기기 0: [9, 12]
기기 1: [9, 12]
기기 2: [9, 12]
```

모든 기기는 집계 결과로 끝납니다. 사용:
- 기기 전체에서 그래디언트 평균화
- 모델 상태 동기화

**Ring AllReduce** (큰 클러스터에 효율적):
- 기기는 논리적 링으로 배열
- 각 기기는 다음에 전송, 이전에서 수신
- 여러 라운드: 각 라운드는 다른 데이터 부분 전송
- 복잡도: N개 기기에 대해 ~2(N-1) 데이터 전송
- 대역폭 효율적: 네트워크 링크 포화

```
Ring AllReduce 스텝 (4개 기기):
라운드 1: 0→1→2→3→0
라운드 2: 3→0→1→2→3
라운드 3: 2→3→0→1→2
```

**트리 감소**:
- 이진 트리 같은 계층 구조
- 감소 단계: 루트까지 위로 결합
- 브로드캐스트 단계: 루트에서 아래로 분배
- 작은 데이터의 경우 Ring보다 낮은 지연, 큰 데이터의 경우 높음

**파라미터 서버**:
- 중앙 서버가 모델 매개변수 보유
- 워커는 매개변수 가져오기, 그래디언트 계산, 업데이트 전송
- 서버가 업데이트 적용
- 확장성 떨어짐, 최신 시스템에서 덜 사용됨

## 통신 효율성 기법

**그래디언트 압축**:
- 그래디언트를 낮은 정밀도(8비트)로 양자화
- 스파시피케이션: 가장 큰 그래디언트만 전송
- 트레이드오프: 줄어든 통신 vs. 느린 수렴
- Top-k 스파시피케이션: 가장 큰 그래디언트 k%만 전송

**그래디언트 누적**:
- 여러 반복에 걸쳐 그래디언트 누적
- 매 반복이 아닌 주기적인 AllReduce
- 통신 빈도 감소
- 동기화 오버헤드 감소

**배치 크기 스케일링**:
- 기기를 두 배로 할 때, 배치 크기도 두 배
- 계산/통신 비율 유지
- 학습률 조정 필요 (학습률 스케일링)
- 선형 스케일링 규칙: lr_새 = lr_구 × (batch_size_새 / batch_size_구)

## 실제 고려 사항

**로드 균형**:
- 불균등한 데이터 분배는 기기 유휴 시간 야기
- 해결책: 데이터 셔플링, 동적 작업 할당
- 동기 학습은 불균형의 영향을 더 받음

**수렴**:
- 더 큰 배치 크기는 수렴을 해칠 수 있음
- LARS 옵티마이저: Layer-wise Adaptive Rate Scaling
- 그래디언트/가중치 비율에 따라 계층당 학습률 조정
- 안정적인 수렴으로 큰 배치 학습 가능

**결함 허용**:
- 모델 및 옵티마이저 상태의 주기적인 체크포인팅
- 기기 장애에서의 복구는 마지막 체크포인트에서 재개 필요
- 체크포인트 빈도: 저장소와 복구 시간 간의 트레이드오프

## 스케일링 효율성 분석

**약한 스케일링**:
- 기기당 총 계산을 고정, 기기 증가
- 문제 크기를 비례적으로 증가
- 이상적: 기기당 고정 문제에 대해 일정한 시간
- 예: 8개 GPU에서 256 배치 크기 vs 64개 GPU에서 2048 배치 크기

**강한 스케일링**:
- 문제 크기를 고정, 기기 증가
- 더 많은 기기로 시간이 감소 (이상적으로 선형)
- 통신이 지배적이 되어 제한됨
- 실제 제한: 순수 데이터 병렬화의 경우 모델당 ~8-16개 기기

**속도향상 공식**:
```
시간 = 계산시간/N_기기 + 통신시간
효율 = 속도향상/N_기기 = T(N=1) / (N × T(N))
```

## 대표 프레임워크 및 도구

**PyTorch Distributed**:
- `torch.nn.parallel.DistributedDataParallel`: 데이터 병렬화
- `torch.distributed`: 저수준 통신 (AllReduce, Send/Recv)
- FSDP for 완전 모델 샤딩

**TensorFlow 분산 전략**:
- `tf.distribute.MirroredStrategy`: 단일 머신에서 데이터 병렬화
- `tf.distribute.MultiWorkerMirroredStrategy`: 다중 머신
- 내장된 그래디언트 집계 및 동기화

**Horovod** (Uber):
- 프레임워크 불가지론적 분산 학습
- 통신 효율성을 위한 Ring-allreduce
- PyTorch, TensorFlow, MXNet과 작동

## 도전 과제 및 한계

- **동기화 오버헤드**: 더 많은 기기로 증가
- **네트워크 대역폭**: 큰 클러스터의 병목
- **복잡성**: 분산 시스템 디버깅은 더 어려움
- **로드 불균형**: 낙오자가 전체 처리량 제한
- **수렴**: 더 큰 배치는 신중한 튜닝 없이 최종 정확도 해칠 수 있음

## 결론

분산 학습은 여러 기기에 걸쳐 계산을 병렬화하여 대규모 모델과 데이터셋의 학습을 가능하게 합니다. FSDP, DeepSpeed, Megatron과 같은 최신 프레임워크는 여러 병렬화 전략을 결합하여 GPU 클러스터에서 거의 선형 스케일링을 달성합니다. 데이터, 모델, 파이프라인 병렬화 간의 트레이드오프, 통신 패턴, 효율성 기법을 이해하는 것은 규모에서 최첨단 모델을 학습하는 실무자에게 필수적입니다.
