# GPT와 자기회귀 모델

## 자기회귀 생성이란?

자기회귀 모델은 한 번에 하나의 토큰을 생성하며, 각 새로운 토큰은 이전에 생성된 모든 토큰을 기반으로 예측됩니다. 이 순차적 접근 방식은 인간이 자연스럽게 언어를 생성하는 방식을 모방합니다—문맥을 바탕으로 다음 단어를 예측합니다. 확률 분포는 P(x) = ∏ P(x_t | x_{<t})로 인수분해되며, 이는 모델이 전체 이력이 주어졌을 때 다음 토큰을 예측하도록 학습한다는 의미입니다.

이 접근 방식은 마스크된 언어 모델링([BERT](/bert-kor.html)) 또는 비자기회귀 생성(모든 토큰을 동시에 예측)과 같은 다른 생성 패러다임과 대조됩니다. 자기회귀 프레임워크는 언어 모델링을 통해 자연스러운 훈련 목표를 제공합니다: 이전 수열이 주어졌을 때 다음 토큰을 예측합니다.

## GPT 계열의 진화

### GPT-1 (2018)
OpenAI의 첫 번째 생성형 사전 훈련 [트랜스포머](/llm-kor.html)는 대규모 텍스트 코퍼스에서 감독되지 않은 사전 훈련 후 작업별 미세 조정이 다양한 NLP 작업에서 강력한 성능을 달성할 수 있음을 입증했습니다. 1억 1천 7백만 개의 파라미터를 가진 GPT-1은 분류, 함의, 유사성 및 질의응답 작업에서 언어 모델을 최소한의 아키텍처 변경으로 효과적으로 적용할 수 있음을 보여주었습니다.

### GPT-2 (2019)
GPT-2는 15억 개의 파라미터로 확장했으며 중요한 통찰력을 소개했습니다: [대규모 언어 모델](/llm-kor.html)은 명시적인 작업별 미세 조정 없이도 인상적인 "제로샷" [전이 학습](/transfer-learning-kor.html)을 수행할 수 있습니다. 모델은 적절하게 프롬프트될 때 번역, 질의응답, 요약과 같은 진정한 신흥 능력을 시연했습니다. GPT-2는 규모를 언어 모델링의 기본 원리로 사용할 수 있는 잠재력을 보여주었습니다.

### GPT-3 (2020)
1,750억 개의 파라미터를 가진 GPT-3은 모델 규모와 능력에서 비약적인 도약을 이루었습니다. 프롬프트에 제공된 소수의 예제만으로 새로운 작업을 학습하는 능력인 강력한 소수샷 학습을 시연했습니다. GPT-3은 컨텍스트 내 학습을 주요 상호작용 패러다임으로 도입하여 작업별 미세 조정의 필요성을 줄였습니다.

### GPT-4 (2023)
규모 확장 패러다임을 바탕으로 GPT-4는 추론 능력을 개선하고 환각을 줄이며 엣지 케이스를 더 잘 처리했습니다. [멀티모달](/multimodal-llm-kor.html) 이해(텍스트 및 이미지)를 시연했으며 수학 및 코딩을 포함한 복잡한 추론 작업에서 개선된 성능을 보여주었습니다.

## 디코더 전용 아키텍처

GPT 모델은 다음으로 구성된 디코더 전용 [트랜스포머](/llm-kor.html) 아키텍처를 사용합니다:

- **토큰 임베딩**: 각 토큰이 고차원 벡터로 매핑됨
- **위치 인코딩**: 토큰 위치에 대한 정보가 임베딩에 추가됨
- **트랜스포머 블록**: 멀티헤드 자기 주의 및 피드포워드 네트워크의 누적 계층
- **인과 마스킹**: 주의가 이전 토큰으로만 제한되어 모델이 "미리 보기"하는 것을 방지
- **계층 정규화 및 잔차 연결**: 깊은 네트워크의 훈련을 안정화

디코더 전용 설계는 입력 수열을 완전히 처리한 후 출력을 생성하는 인코더-디코더 모델([seq2seq](/seq2seq-kor.html))과 다릅니다. 디코더 전용 모델에서 생성은 순전히 순차적이고 자기회귀적입니다.

## 컨텍스트 내 학습

컨텍스트 내 학습은 프롬프트에 제공된 예제에서 명시적인 훈련 업데이트 없이 새로운 작업을 학습하는 언어 모델의 능력을 말합니다. 프롬프트는 다음을 포함할 수 있습니다:

```
감정 분류 예제:
"좋은 영화!" → 긍정
"끔찍한 경험" → 부정
"놀라운 서비스" → 긍정

"나는 그것을 싫어했어" →
```

모델은 문맥에서 작업 정의를 학습하고 새로운 인스턴스에 적절한 응답을 생성합니다. 이 메커니즘은 사전 훈련 중에 학습된 모델의 내부 표현에 의존하는 것으로 보이며, 이는 많은 암시적 작업 해결 전략을 인코딩합니다.

## 소수샷 및 제로샷 프롬프팅

**소수샷 프롬프팅**은 프롬프트 내에서 작업의 여러 예제를 제공하여 모델이 패턴을 유추하고 새로운 인스턴스에 적용할 수 있도록 합니다. 재훈련 없이 빠른 작업 적응을 위해 강력합니다.

**제로샷 프롬프팅**은 예제를 제공하지 않으며 작업 설명만 제공합니다. 모델은 사전 훈련 중에 획득한 지식에만 의존해야 합니다. 제로샷 성능은 일반적으로 낮지만 언어 모델에 인코딩된 지식의 범위를 시연합니다.

**원샷 프롬프팅**은 작업 형식을 설정하기 위해 단일 예제를 제공하면서 중간 지점을 차지합니다.

## 신흥 능력

[대규모 언어 모델](/llm-kor.html)은 특정 임계값 이상으로 규모가 증가할 때 갑자기 나타나는 능력인 "신흥 능력"을 나타냅니다. 예제는 다음을 포함합니다:

- **산술 추론**: 소규모 모델은 단순 수학에 실패하고 대규모 모델은 다단계 문제를 해결함
- **코드 생성**: 모델은 충분한 규모에서 작동하는 코드를 갑자기 생성함
- **체인-오브-생각 추론**: 모델은 "단계적으로 생각하라"고 요청받을 때 내부 추론 단계를 개발함
- **번역**: 명시적인 번역 훈련 데이터 없이 모델은 언어 간에 번역함

이러한 능력은 종종 놀라우며 예측 불가능하여 규모 자체만으로 질적으로 새로운 능력을 유도할 수 있음을 시사합니다.

## 규모 확장의 영향

연구자들은 모델 크기, 훈련 데이터 크기 및 계산을 하류 성능에 관련시키는 일관된 규모 확장 법칙을 확인했습니다. 규모 확장 법칙은 대략 Loss ∝ N^(-α)를 따르며, 여기서 N은 모델 크기이고 α ≈ 0.07입니다. 이는 더 큰 모델과 더 많은 데이터가 일관되게 더 낮은 손실을 달성함을 시사합니다.

규모 확장으로부터의 주요 통찰력:

- **더 크면 더 좋음**: 모델 크기, 데이터 및 계산을 확장하면 성능이 개선됨
- **절충이 존재함**: 계산, 파라미터 및 데이터는 거래될 수 있음(최적화에서 일부 유연성)
- **멱법 행동**: 개선이 규모에서 계속됨; 명확한 고점 없음(기계 학습과 달리)
- **신흥 능력**: 규모 확장이 더 작은 모델이 수행할 수 없는 새로운 능력을 활성화함

이 이해는 점점 더 큰 모델을 향한 경쟁을 주도했으나, 수익 감소와 증가하는 자원 요구 사항이 있습니다.

## 결론

GPT 모델은 자연어 처리에서 패러다임 전환을 나타내며, 처음부터 훈련된 작업별 모델에서 컨텍스트 내 학습을 통해 다양한 작업을 처리할 수 있는 범용 기반 모델로 이동합니다. 디코더 전용 아키텍처와 대규모 규모의 조합은 놀랍도록 효과적임을 증명했으며, 모델이 더 커짐에 따라 개선이 계속 출현합니다. 자기회귀 생성, 규모 확장 법칙 및 신흥 능력을 이해하는 것은 현대 [대규모 언어 모델](/llm-kor.html)과 작업하기 위한 필수입니다.
