# 개체명 인식

## 개체명 인식이란?

개체명 인식(Named Entity Recognition, NER)은 텍스트에서 명명된 개체를 식별하고 분류하는 작업입니다. 명명된 개체는 고유명사로 표시될 수 있는 실제 객체(사람, 조직, 위치, 날짜 등)입니다. NER은 비정형 텍스트에서 구조화된 정보를 자동으로 추출하여 개체 경계를 식별하고 의미론적 유형을 할당합니다.

예를 들어 문장 "Apple Inc.는 Steve Jobs에 의해 Cupertino에서 1976년 4월 1일에 설립되었습니다."에서 NER은 다음을 식별할 것입니다:
- "Apple Inc." → ORGANIZATION
- "Steve Jobs" → PERSON
- "Cupertino" → LOCATION
- "1976년 4월 1일" → DATE

## 일반적인 개체 유형

표준 NER 시스템은 일반적으로 여러 개체 범주를 인식합니다:

- **PERSON**: 개인 이름(John Smith, Marie Curie)
- **ORGANIZATION**: 회사, 기관 또는 그룹 이름(Microsoft, UN, Harvard University)
- **LOCATION**: 도시, 국가, 지역을 포함한 지리적 위치(Paris, Brazil, Silicon Valley)
- **DATE**: 시간 표현(2023년 1월 15일, 지난주, 내일)
- **TIME**: 특정 시간(오후 3시 45분, 정오, 자정)
- **MONEY**: 금전 가치 및 통화(€50, $120만)
- **PERCENT**: 백분율(42%, 0.5%)
- **FACILITY**: 건물, 공항, 고속도로(Golden Gate Bridge, JFK Airport)
- **PRODUCT**: 제품 이름(iPhone 15, Windows 11)
- **EVENT**: 명명된 사건(올림픽 게임, 제2차 세계대전)

도메인별 응용 프로그램을 위해 사용자 정의 개체 유형을 정의할 수 있습니다.

## BIO 및 BIOES 태깅 체계

NER은 일반적으로 각 토큰이 태그를 수신하는 수열 태깅 문제로 공식화됩니다. 가장 일반적인 태깅 체계는 BIO(Begin-Inside-Outside)입니다:

- **B-X**: X 유형 개체의 시작
- **I-X**: X 유형 개체 내부(계속)
- **O**: 어떤 개체 외부도 아님

예를 들어:
```
Apple    → B-ORG
Inc.     → I-ORG
was      → O
founded  → O
by       → O
Steve    → B-PER
Jobs     → I-PER
```

BIOES 체계는 다음을 추가합니다:
- **E-X**: X 유형 개체의 끝
- **S-X**: X 유형의 단일 토큰 개체

BIOES는 더 정보가 많으며 개체 경계를 명시적으로 캡처하지만 더 많은 훈련 데이터가 필요합니다. BIO는 더 간단하고 널리 사용됩니다.

## 조건부 확률장(Conditional Random Fields, CRF)

조건부 확률장은 수열 레이블링에 사용되는 확률 그래픽 모델입니다. 각 토큰의 레이블을 독립적으로 예측하는 독립 분류기와 달리 CRF는 다음과 같은 제약을 캡처하는 레이블 의존성을 모델링합니다:
- 특정 태그 수열이 유효합니다(B-PER → I-PER은 유효; B-PER → I-ORG는 가능성이 낮음)
- 레이블은 인접 토큰 및 해당 레이블에 따라 달라집니다

선형 체인 CRF 모델: P(y|x) ∝ exp(Σ_i λ_i f_i(y_{t-1}, y_t, x, t))

여기서 f_i는 발출(토큰→레이블) 및 전이(레이블→레이블)를 캡처할 수 있는 특성 함수입니다. CRF는 특히 명확한 레이블 의존성이 있는 데이터셋에서 강력한 성능을 달성합니다.

## BiLSTM-CRF 아키텍처

인기 있고 효과적인 접근 방식은 BiLSTM과 CRF를 결합합니다:

1. **임베딩 계층**: 토큰을 단어 임베딩에 매핑(사전 훈련되거나 학습)
2. **BiLSTM**: 양방향 LSTM이 토큰을 앞뒤로 처리하여 문맥 캡처
3. **CRF 계층**: 전이로 레이블 수열을 모델링하여 유효한 태그 수열 적용

BiLSTM은 풍부한 상황화된 표현을 학습하는 반면 CRF 계층은 출력 수열의 구조적 유효성을 보장합니다. 이 조합은 일관되게 더 간단한 모델을 능가하며 NER의 표준 기준이 되었습니다.

## 개체명 인식을 위한 BERT

BERT는 양방향 트랜스포머에서 상황화된 토큰 표현을 제공합니다. NER의 경우 접근 방식은 다음과 같습니다:

1. **토큰화**: 텍스트를 BERT 워드피스 토큰으로 분할
2. **표현 획득**: 사전 훈련된 BERT 계층을 통해 전달
3. **토큰 분류**: BERT 토큰 표현 위에 분류 계층 추가
4. **미세 조정**: NER 작업 레이블에서 훈련

BERT의 상황화된 임베딩은 정적 임베딩보다 단어 경계, 다의성 및 장거리 의존성을 더 잘 처리합니다. 미세 조정된 BERT 모델은 최첨단 NER 성능을 달성하여 종종 표준 벤치마크에서 90% 이상의 F1 점수를 초과합니다.

## NER을 위한 spaCy

spaCy는 효율적인 NER 기능을 갖춘 산업용 NLP 라이브러리입니다. 다음을 제공합니다:

- **사전 훈련된 모델**: 영어, 독일어, 프랑스어, 포르투갈어 등을 위한 즉시 사용 가능한 모델
- **빠른 성능**: 최적화된 C 구현은 대용량 문서 처리를 가능하게 함
- **쉬운 통합**: 개체 추출을 위한 간단한 Python API
- **사용자 정의 모델**: 도메인별 데이터에서 사용자 정의 NER 모델 훈련 지원
- **개체 연결**: 인식된 개체를 외부 지식 베이스에 연결

사용 예시:
```python
import spacy
nlp = spacy.load("en_core_web_sm")
doc = nlp("Apple Inc.는 Steve Jobs에 의해 설립되었습니다.")
for ent in doc.ents:
    print(f"{ent.text} → {ent.label_}")
```

## NER의 응용 프로그램

### 정보 추출
NER은 비정형 문서에서 구조화된 정보를 추출하기 위한 기초입니다. 예를 들어 뉴스 기사에서 언급된 모든 회사, 사람 및 위치를 추출하면 자동 문서 구성 및 검색을 가능하게 합니다.

### 지식 그래프
NER은 실제 객체 간의 관계를 구조화된 표현인 지식 그래프를 만드는 데 연결될 수 있는 개체를 식별합니다. Google Knowledge Graph, Wikidata 및 유사한 시스템은 NER을 기초 구성 요소로 사용합니다.

### 질의응답
NER 시스템은 사실 질문에 대한 후보 답변을 추출합니다. "Einstein은 언제 태어났나?"라는 질문을 받았을 때 QA 시스템은 NER을 사용하여 DATE 개체를 찾아 답변 정확도를 개선할 수 있습니다.

### 관계 추출
개체 인식 후 관계 추출은 개체 간의 관계를 결정합니다. NER은 개체 언급을 식별하고 후속 처리는 "(Einstein, born_in, Germany)"와 같은 관계를 결정합니다.

### 문서 분류 및 검색
문서에서 언급된 개체는 분류 및 검색 시스템의 특성으로 사용될 수 있습니다. "Apple"과 "Microsoft"를 언급하는 문서는 "Obama"와 "Trump"를 언급하는 문서와 다르게 그룹화될 수 있습니다.

## NER의 과제

- **개체 경계 탐지**: 개체가 시작되고 끝나는 위치를 결정하는 것은 자명하지 않습니다("New York"은 하나 또는 두 개의 개체?)
- **모호성**: 개체는 여러 범주에 속할 수 있습니다(Stanford → ORGANIZATION 또는 LOCATION)
- **도메인 적응**: 뉴스에서 훈련된 모델은 의료 또는 과학 텍스트에서 실패합니다
- **희귀 개체**: 빈도가 낮은 개체 유형은 인식하기 어렵습니다
- **신흥 개체**: 새로운 개체가 계속 나타나며 모델은 일반화해야 합니다

## 결론

개체명 인식은 텍스트에서 구조화된 정보 추출을 가능하게 하는 기초 NLP 작업입니다. 초기 규칙 기반 및 통계 접근 방식에서 현대 신경 아키텍처를 통해 NER은 깊은 상황화된 표현을 활용하도록 진화했습니다. 현대적 BERT 기반 및 트랜스포머 모델은 우수한 성능을 달성하는 반면 spaCy와 같은 실용적 도구는 효율적이고 프로덕션 준비가 된 솔루션을 제공합니다. NER 원리 및 도구를 이해하는 것은 정보 추출, 지식 그래프 구성 및 수많은 하류 NLP 응용 프로그램에 필수적입니다.
