# 직접 선호 최적화 (DPO)

직접 선호 최적화(Direct Preference Optimization, DPO)는 별도의 보상 모델 없이 인간의 선호도를 학습하는 현대적인 정렬(alignment) 기법입니다. 복잡한 다단계 파이프라인이 필요한 RLHF와 달리, DPO는 선호도 데이터를 사용하여 모델을 직접 최적화합니다.

## RLHF의 병목 현상

전통적인 인간 피드백 강화학습(RLHF)은 세 가지 단계로 구성됩니다: 지도 미세조정, 보상 모델 학습, RL 학습. 이 다단계 접근 방식은 여러 도전과제를 야기합니다:

- **계산 오버헤드**: 별도 모델 학습과 RL 알고리즘 실행으로 인한 막대한 비용과 시간
- **보상 모델 불안정성**: 학습된 보상 모델이 실제 인간 선호도를 정확히 반영하지 못할 수 있음
- **오차 누적**: 초기 단계의 실수가 이후 학습 단계로 전파됨
- **하이퍼파라미터 튜닝**: 각 단계마다 신중한 설정이 필요

이러한 한계로 인해 더 효율적으로 선호도를 직접 최적화하는 방법이 모색되었습니다.

## DPO 공식화

DPO는 정렬 문제를 수학적으로 재정의합니다. 핵심 통찰은 명시적인 보상 모델 학습 없이 언어 모델에서 직접 암묵적 보상 모델을 추출할 수 있다는 것입니다.

선호도 최적화 목적함수는 최대가능도 추정 프레임워크에서 유도됩니다. 같은 프롬프트 x에 대한 응답 쌍(y_w = 선호하는 응답, y_l = 덜 선호하는 응답)이 주어졌을 때:

```
L_DPO = -log σ(β(log π(y_w|x) - log π_ref(y_w|x)) - β(log π(y_l|x) - log π_ref(y_l|x)))
```

여기서:
- π는 학습 중인 모델
- π_ref는 참조 모델(초기 지도 미세조정 모델)
- β는 선호도 강제 정도를 조절하는 온도 매개변수
- σ는 시그모이드 함수

이 공식은 선호도 신호를 언어 모델의 목적함수에 직접 통합합니다.

## 암묵적 보상 모델

DPO의 우아한 측면 중 하나는 보상 모델이 암묵적으로 나타난다는 것입니다. 모델과 참조 모델 간의 로그 확률 차이가 암묵적 보상 신호가 됩니다:

```
r(x, y) ≈ β log(π(y|x) / π_ref(y|x))
```

이 암묵적 보상 모델은 별도의 학습 데이터나 아키텍처가 필요 없으므로 복잡성을 줄이면서 정렬 효과는 유지합니다.

## 학습 프로세스

DPO 학습 절차는 간단합니다:

1. **데이터 준비**: 인간 주석자 또는 기존 주석에서 선호도 쌍 수집
2. **참조 모델**: 지도 미세조정 모델(SFT 모델)로 초기화
3. **순전파**: 선호하는 응답과 비선호 응답 모두에 대한 로그 확률 계산
4. **손실 계산**: 선호도 공식을 사용하여 DPO 손실 계산
5. **그래디언트 업데이트**: 선호하는 응답의 확률을 높이도록 모델 가중치 업데이트

학습은 표준 언어 모델 미세조정과 유사하게 배치 처리되어 기존 인프라 및 분산 학습 설정과 호환됩니다.

## DPO 변형 및 확장

DPO의 성공은 여러 변형을 영감했습니다:

**IPO (Identity Policy Optimization)**: 참조 모델의 동작을 더 잘 유지하면서 선호도를 최적화하는 수정된 손실 함수로, KL 발산을 줄입니다.

**KTO (Kahneman-Tversky Optimization)**: 전망 이론 원칙을 통합하여 인간의 인지 편향에 기반해 성공(선호)과 손실(비선호)을 비대칭으로 처리합니다.

**ORPO (Odds Ratio Preference Optimization)**: 오즈비 프레이밍을 사용하여 선호도 학습을 단순화하며 향상된 학습 안정성과 정렬 품질을 보여줍니다.

**CPO (Contrastive Preference Optimization)**: 대조 학습으로 DPO를 확장하여 선호도 쌍을 더 잘 구별합니다.

## RLHF와의 비교

### DPO의 장점:
- **단순성**: 3단계 RLHF 파이프라인 vs. 1단계 학습
- **안정성**: 별도 보상 모델이 없어 학습 불안정 요소 감소
- **표본 효율성**: 더 적은 선호도 예제로 더 나은 결과 달성
- **계산 비용**: 더 낮은 메모리 및 계산 요구사항

### RLHF의 장점:
- **모듈성**: 다양한 목표를 위해 보상 모델 교체 가능
- **해석 가능성**: 명시적 보상 모델이 직접적인 피드백 신호 제공
- **유연성**: RL 프레임워크로 탐색에 대한 세밀한 제어 가능

실제로 DPO는 단순함에도 불구하고 RLHF보다 더 좋은 성능을 보이곤 합니다. 선호도의 직접 최적화가 중간 보상 모델 학습보다 더 효과적인 것으로 보입니다.

## 실질적 장점

**인프라 복잡성 감소**: DPO는 별도의 보상 모델 학습 인프라를 제거하여 더 작은 팀과 조직에 접근 가능하게 합니다.

**빠른 반복**: 단일 학습 루프로 다양한 선호도 데이터셋 실험이 더 빠르고 효율적입니다.

**더 나은 확장성**: DPO는 더 긴 시퀀스와 더 큰 모델에 더 효율적으로 확장되며, 여러 모델을 동시에 추적할 필요가 없습니다.

**실증적 성능**: DPO로 학습된 모델은 선호도 벤치마크 및 인간 평가에서 RLHF 기준 대비 지속적으로 높은 점수를 달성합니다.

## 구현 고려사항

DPO 구현 시:

- **β 매개변수 튜닝**: 온도 매개변수가 동작에 크게 영향; β가 높을수록 선호도를 더 엄격하게 강제
- **참조 모델 품질**: 더 나은 SFT 모델은 더 나은 DPO 학습으로 이어짐
- **선호도 데이터 품질**: 인간 주석 품질이 학습 효과에 직접 영향
- **KL 발산 모니터링**: 참조 모델로부터의 발산을 추적하여 치명적인 변화를 방지

DPO는 현대적 언어 모델 정렬을 위해 선호도 최적화를 실용적이고 효율적으로 만드는 중요한 진전을 나타냅니다. 그 단순성과 효과는 많은 실무자들이 선호하는 접근법이 되었습니다.
