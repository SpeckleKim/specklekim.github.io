# 잔차 연결 (스킵 연결)

## 소개

잔차 연결(Residual Connections), 또는 스킵 연결(Skip Connections)은 매우 깊은 신경망의 학습을 가능하게 하는 근본적인 아키텍처 혁신입니다. 2015년 He et al.이 ResNet 아키텍처에서 도입한 스킵 연결은 정보가 중간 계층을 우회하여 통과하도록 하며, 역전파 과정에서 그래디언트가 흐르는 방식을 근본적으로 변화시킵니다.

## 동기: 그래디언트 소실 문제

스킵 연결 이전에는 매우 깊은 신경망을 학습시키기 어려웠습니다. 네트워크가 깊어질수록 역전파 중에 계산되는 그래디언트는 점점 더 작아져서 그래디언트 소실 문제(Vanishing Gradient Problem)가 발생했습니다. 입력층 근처의 계층은 무시할 수 있을 정도의 그래디언트 업데이트만 받아 깊은 계층의 효과적인 학습을 방해했습니다.

핵심 통찰은 입력 x에서 출력으로의 직접 매핑 F(x)를 학습하는 대신, 네트워크가 잔차 매핑 R(x) = F(x) - x를 학습할 수 있다는 것입니다. 이러한 재구성은 네트워크가 입력을 얼마나 변경할지 학습하는 것을 가능하게 하며, 처음부터 완전한 변환을 학습하는 것보다 쉽습니다.

## 항등 매핑

잔차 연결의 핵심 메커니즘은 항등 매핑을 추가하는 것입니다:

```
y = F(x) + x
```

여기서 F(x)는 하나 이상의 계층으로 학습되는 변환(잔차 분기)을 나타냅니다. F(x) ≈ 0이면 출력 y ≈ x가 되어, 정보가 변하지 않은 상태로 통과합니다. 이는 역전파 과정에서 그래디언트가 직접 흐를 수 있는 경로를 제공합니다.

## 스킵 연결이 더 깊은 네트워크를 가능하게 하는 이유

**개선된 그래디언트 흐름**: 역전파 중 그래디언트는 스킵 연결을 통해 직접 흐를 수 있으므로(dy/dx = 1 + dF/dx) 그래디언트 소실을 방지합니다. dF/dx가 작더라도, 그래디언트는 직접적인 경로를 가집니다.

**쉬운 최적화**: 네트워크는 완전한 매핑이 아닌 잔차를 명시적으로 학습합니다. 변경할 내용을 학습하는 것이 처음부터 전체 변환을 학습하는 것보다 종종 더 쉽습니다.

**암묵적 정규화**: 스킵 연결은 일종의 암묵적 정규화를 제공하여 내부 공변량 이동을 방지하고 학습을 안정화합니다.

## ResNet 아키텍처

ResNet은 50, 101, 또는 심지어 152개 계층을 가진 네트워크를 효과적으로 학습할 수 있음을 보여주어 깊은 학습을 혁신했습니다. 기본 구성 요소는:

**잔차 블록**:
- 2~3개의 합성곱 계층
- 선택적 배치 정규화
- 입력을 출력에 더하는 스킵 연결
- ReLU 활성화

더 깊은 병목 변형은 계산을 줄이면서 표현 능력을 유지하여 ResNet-50, ResNet-101, ResNet-152를 대규모 응용에 실용적으로 만듭니다.

## 변형 및 확장

**DenseNet**: 스킵 연결을 더하는 대신, DenseNet은 모든 이전 계층의 특성 맵을 연결합니다:

```
y = [x, F₁(x), F₂([x, F₁(x)]), ...]
```

이는 정보 흐름을 위한 많은 짧은 경로를 만들고 더 효율적인 특성 전파를 가능하게 하여, 더 적은 매개변수로 최첨단 정확도를 달성합니다.

**Highway Networks**: ResNet 이전에 도입된 Highway Networks는 게이팅 메커니즘을 사용합니다:

```
y = T(x) ⊙ F(x) + (1 - T(x)) ⊙ x
```

여기서 T(x)는 잔차 또는 항등 정보를 얼마나 사용할지 결정하는 학습된 게이트입니다.

**Transformer의 스킵 연결**: Transformer는 전체적으로 잔차 연결을 사용합니다:

```
Output = LayerNorm(Input + Attention(Input))
Output = LayerNorm(Output + FeedForward(Output))
```

이들은 GPT-3(96개 계층) 및 BERT 변형과 같은 매우 깊은 Transformer 모델의 학습을 가능하게 합니다.

## U-Net 아키텍처

의미 분할에서 널리 사용되는 U-Net은 스킵 연결을 다르게 사용합니다. 인코더 경로(다운샘플링)의 특성은 대응하는 디코더 경로(업샘플링) 특성과 연결됩니다:

```
decoder_output = concatenate(encoder_features, upsampled_features)
```

이는 공간 정보와 세밀한 세부 사항을 보존하여 병목 계층에도 불구하고 정확한 픽셀 단위 예측을 가능하게 합니다.

## 수학적 분석

**그래디언트 관점**: 잔차 연결이 있는 블록을 고려하면:

```
∂L/∂x = ∂L/∂y · (I + ∂F/∂x)
```

항등 항 I은 ∂F/∂x가 작더라도 그래디언트가 완전히 감소되지 않도록 보장합니다. 야코비안은 깊은 네트워크 전체에서 잘 조건화된 상태를 유지합니다.

**이론적 통찰**: 스킵 연결은 효과적으로 서로 다른 깊이의 네트워크의 암묵적 앙상블을 만듭니다. n개의 잔차 블록을 가진 ResNet은 2ⁿ개의 더 짧은 네트워크 경로의 암묵적 앙상블로 볼 수 있습니다.

## 실제 고려 사항

**차원 일치**: 입력과 출력 차원이 다를 때 프로젝션이 필요합니다:
- 선형 프로젝션 (1×1 합성곱)
- 0 패딩
- 차원을 유지하기 위한 신중한 아키텍처 설계

**배치 정규화 순서**: Pre-activation 잔차 블록(BN → ReLU → Conv → Add)은 종종 post-activation (Conv → BN → ReLU → Add)보다 더 잘 작동하여 정보 흐름을 개선합니다.

**학습률 및 초기화**: 스킵 연결은 신중한 초기화가 필요합니다. He 초기화는 특히 잔차 네트워크의 증가된 그래디언트 흐름을 처리합니다.

## 장점 및 영향

- **매우 깊은 네트워크 가능**: 19개 계층(VGG)에서 152개 이상(ResNet)으로
- **개선된 그래디언트 흐름**: 학습 중 더 나은 수렴
- **더 나은 일반화**: 스킵 연결은 정규화기 역할
- **계산 효율성**: 학습하기 쉽고 더 짧은 시간에 더 나은 정확도 달성
- **광범위한 적용 가능성**: 이제 CNN, Transformer 및 기타 아키텍처의 표준

## 한계 및 대안

스킵 연결이 보편적으로 유용하지만, 모든 깊은 학습 문제를 해결하지는 않습니다. 일부 네트워크는 대안 설계의 이점을 봅니다:

- **넓은 네트워크**: 때때로 좁고 깊은 네트워크보다 넓고 얕은 네트워크가 더 좋은 성능
- **신중한 초기화**: 최적 성능을 위해 여전히 적절한 기술이 필요
- **문맥 종속성**: 일부 도메인에서 짧은 연결이 덜 도움이 될 수 있음

## 결론

잔차 연결은 이전에 가능했던 것보다 수 자리 더 깊은 네트워크의 학습을 가능하게 함으로써 깊은 학습을 근본적으로 변화시켰습니다. 완전한 매핑을 학습하는 대신 잔차를 학습하는 단순하면서도 우아한 아이디어는 오늘날 사실상 모든 최첨단 아키텍처에서 나타나는 현대 신경망 설계의 초석이 되었습니다.
