# 위치 인코딩

[셀프 어텐션](/self-attention-kor.html)은 순열 불변성(permutation invariance)을 가집니다: 토큰 순서와 관계없이 입력 시퀀스를 동일하게 처리합니다. 이 속성은 병렬화를 가능하게 하지만, 모델이 위치 정보를 명시적으로 받아야 함을 의미합니다. 위치 인코딩은 위치 정보를 시퀀스 표현에 통합하여 이 기본 과제를 해결합니다.

## 왜 위치가 중요한가?

동일 토큰을 가진 두 문장을 생각해봅시다:
- "Dog bites man"
- "Man bites dog"

위치 정보 없이, 모델은 이 문장들을 구분할 수 없습니다. [셀프 어텐션](/self-attention-kor.html)만으로는 내용에서만 작동하고 구조를 인식하지 못합니다. 위치 인코딩은 모델이 순서 의존적 관계를 학습하도록 보장합니다.

핵심 통찰:
1. **위치 정보는 필수**: 언어 이해뿐만 아니라
2. **다양한 해결책 존재**: 다양한 특성을 가진 여러 인코딩 방식
3. **트레이드오프 존재**: 계산 효율성과 표현력 사이

## 삼각함수 위치 인코딩

원본 [트랜스포머](/llm-kor.html)는 삼각함수를 사용합니다:

```
PE(pos, 2i) = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
```

여기서:
- pos: 시퀀스의 위치 (0, 1, 2, ...)
- i: 차원 인덱스 (0, 1, ..., d_model/2 - 1)
- d_model: 임베딩 차원 (512, 768, 등)

### 삼각함수 인코딩 뒤의 직관

1. **고유한 위치 표현**: 각 위치가 고유한 인코딩 벡터를 가짐
2. **상대 위치 인식**: 위치 간 선형 관계
3. **긴 시퀀스로의 일반화**: 주파수가 학습 길이를 초과하여 외삽
4. **해석 가능한 구조**: 다양한 주파수가 다양한 거리 척도 포착

### 예시 (d_model=4)
```
pos=0: [sin(0),  cos(0),  sin(0),     cos(0)    ] = [0,     1,      0,      1      ]
pos=1: [sin(1),  cos(1),  sin(0.01),  cos(0.01) ] ≈ [0.841, 0.540,  0.01,   1.000  ]
pos=2: [sin(2),  cos(2),  sin(0.02),  cos(0.02) ] ≈ [0.909, -0.416, 0.02,   0.9998 ]
```

낮은 차원(2i+1)은 천천히 변하고(장주기파), 높은 차원은 빠르게 변합니다(단주기파).

## 학습된 위치 임베딩

대체 접근법은 위치 임베딩을 직접 학습합니다:

```
pos_embed[i] ∈ ℝ^d_model   (학습 가능한 매개변수)
```

장점:
- 작업을 위한 직접 최적화
- 복잡한 위치 패턴 포착 가능
- 주파수 엔지니어링 필요 없음

단점:
- 고정 어휘: 학습 중 max_seq_len까지의 위치만 학습
- 외삽 불가: 테스트 시 더 긴 시퀀스에서 실패
- 메모리 오버헤드: 추가 임베딩 행렬

## 상대 위치 편향 (ALiBi - Attention with Linear Biases)

Press et al.이 도입한 ALiBi는 위치 의존적 편향을 [어텐션](/attention-mechanism-kor.html) 점수에 직접 추가합니다:

```
Attention(Q, K, V) = softmax(Q*K^T / sqrt(d_k) + bias_matrix) * V
```

여기서 bias_matrix[i,j] = -α * |i - j| (학습된 또는 고정된 α).

핵심 특성:
1. **학습 가능한 임베딩 없음**: 위치가 [어텐션](/attention-mechanism-kor.html) 편향으로 인코딩됨
2. **단순 선형 감쇠**: 거리에 따라 [어텐션](/attention-mechanism-kor.html)이 선형으로 감소
3. **긴 시퀀스로 일반화**: 편향 공식은 모든 길이에 작동
4. **희소하고 효율적**: 계산할 추가 임베딩 없음
5. **해석 가능**: 거리와 [어텐션](/attention-mechanism-kor.html) 간 명확한 관계

ALiBi는 현대 LLM에서 인기 증가하고 있습니다:
- 긴 시퀀스로 자연스럽게 외삽
- 메모리 사용 감소
- 위치 어휘 필요 없음

## 회전 위치 임베딩 (RoPE)

Su et al.이 제안한 RoPE는 2D 회전 행렬을 사용합니다:

```
[cos(θ)  -sin(θ)]   [q_i]
[sin(θ)   cos(θ)] * [q_j]

여기서 θ = base^(-2i/d_model) * pos
```

핵심 특성:
1. **기하학적 해석**: 복소 평면의 회전으로서의 위치
2. **상대 위치 인코딩**: [어텐션](/attention-mechanism-kor.html)이 자연스럽게 상대 거리 포착
3. **좋은 일반화**: 보간 기법이 긴 시퀀스로 확장
4. **효율적**: [어텐션](/attention-mechanism-kor.html) 전에 Q와 K에 직접 적용
5. **합성적**: 내적 관계 유지

RoPE는 이제 LLaMA와 같은 모델에서 광범위하게 채택됩니다:
- 뛰어난 길이 외삽 특성
- 수학적으로 우아함
- 효율적 [GPU](/gpu-hardware-kor.html) 구현
- 강력한 실증적 성능

## 인코딩 방법 비교

| 방법 | 매개변수 | 외삽 | 복잡도 | 해석 가능성 |
|------|---------|------|--------|-----------|
| 삼각함수 | 0 | 좋음 | O(1) | 높음 |
| 학습됨 | O(max_seq_len) | 나쁨 | O(1) | 낮음 |
| ALiBi | 0 | 훌륭함 | O(n) | 중간 |
| RoPE | 0 | 좋음* | O(n) | 높음 |

*보간 또는 외삽 기법 사용 시

## 실전의 위치 인코딩

### 학습 중
- 모델은 일반적으로 고정 문맥 길이 학습 (512, 2048, 4096 토큰)
- 위치 인코딩은 [0, context_length)에 대해 생성됨
- 기울기 업데이트는 문맥 창에만 영향

### 추론 중
- 사전학습된 모델의 고정 위치 인코딩
- 시퀀스가 학습 길이를 초과하면:
  - 삼각함수: 외삽 (주파수 패턴 계속)
  - 학습됨: 실패 또는 미세조정 필요
  - ALiBi/RoPE: 자연스럽게 외삽

### 더 긴 문맥으로 확장
현대 기법은 더 긴 시퀀스를 가능하게 합니다:
- **보간**: 위치 버킷 크기 증가
- **외삽**: 패턴이 유지된다는 확신
- **저계수 적응**: 더 긴 길이를 위해 미세조정
- **결합 학습**: 가변 길이로 학습

## 교차 어텐션의 위치 인코딩

인코더-디코더 모델에서:
- 인코더는 자신의 위치 인코딩을 가짐
- 디코더는 자신의 위치 인코딩을 가짐
- 교차 [어텐션](/attention-mechanism-kor.html)은 둘 다 사용 (또는 모두 사용하지 않음)

다양한 전략:
- 양쪽 위치: 둘 다에서 절대 위치
- 디코더만: 디코더 위치만 중요
- 상대적: 인코더/디코더 간 상대 위치 계산

## 함정 및 모범 사례

1. **길이 불일치**: 위치 인코딩이 실제 시퀀스 길이를 포함하는지 확인
2. **외삽 실패**: 학습보다 긴 시퀀스에서 모델 동작 테스트
3. **주파수 편향**: 삼각함수는 특정 주파수 범위를 과소 활용 가능
4. **드롭아웃과의 상호작용**: 위치 인코딩은 일반적으로 드롭되지 않음
5. **멀티스케일 이해**: 견고성을 위해 여러 위치 인코딩 결합

## 결론

위치 인코딩은 순열 불변성 [셀프 어텐션](/self-attention-kor.html)을 순서 인식 메커니즘으로 변환합니다. 삼각함수에서 학습된 임베딩, 현대 RoPE까지, 분야는 계속 더 나은 위치 표현을 탐구합니다. ALiBi와 RoPE는 최근 진전을 나타냅니다: 학습 가능한 위치 임베딩을 제거하면서 외삽 기능을 유지 또는 개선합니다.

적절한 위치 인코딩 선택은 모델 능력, 특히 학습 문맥보다 긴 시퀀스를 다룰 때 중요합니다. 단순성, 효율성, 표현력 간 트레이드오프를 이해하는 것은 정보 있는 설계 결정을 지시합니다.
