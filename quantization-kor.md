# 모델 양자화

모델 양자화는 모델 가중치와 활성화의 수치 정밀도를 낮추어 모델 크기와 계산 요구사항을 대폭 감소시키면서 성능을 유지하는 기법입니다. 이는 자원이 제한된 장치에서 대규모 언어 모델을 배포하는 데 필수적입니다.

## 양자화의 필요성

현대적 언어 모델은 수십억 개에서 수조 개의 매개변수를 포함합니다. 70억 개의 매개변수를 가진 모델을 32비트 부동소수점(FP32)으로 저장하면 약 28GB의 메모리가 필요합니다. 이는 배포를 비싸고 느리며 많은 애플리케이션에서 실질적이지 않게 만듭니다. 양자화는 이러한 근본적인 도전을 해결합니다.

메모리 절감을 넘어 양자화는 다음을 제공합니다:
- **대역폭 감소**: 더 작은 모델은 더 적은 데이터 전송 필요
- **더 빠른 계산**: 더 낮은 정밀도 연산은 특수 하드웨어에서 더 빠르게 실행
- **낮은 에너지 소비**: 추론이 더욱 전력 효율적
- **엣지 배포**: 모델이 모바일 및 임베디드 시스템에서 실행 가능

## INT8과 INT4 양자화

**INT8 (8비트 정수)** 는 가장 일반적인 학습 후 양자화 접근 방식입니다. 각 부동소수점 값은 8비트 정수로 매핑되어 메모리를 4배 줄이면서 모델 동작을 놀랍게도 잘 보존합니다:

```
quantized_value = round((original_value - zero_point) / scale)
```

여기서 scale과 zero_point는 모델의 값 범위에서 결정된 보정 매개변수입니다.

**INT4 (4비트 정수)** 양자화는 극도의 압축(8배 감소)을 제공하지만 더 신중한 처리가 필요합니다. 16개의 가능한 값만 있으므로 INT4 양자화는 본질적으로 손실이 있으며 제대로 구현하지 않으면 성능을 크게 저하시킬 수 있습니다.

## 학습 후 양자화 (PTQ)

학습 후 양자화는 모델이 완전히 학습된 후에 양자화를 적용하므로 재학습이 필요 없습니다:

1. **보정**: 대표 데이터셋에서 모델을 실행하여 가중치 및 활성화 분포에 대한 통계 수집
2. **스케일 계산**: 각 계층 또는 텐서에 대해 최적의 scale과 zero_point 계산
3. **양자화**: 가중치와 활성화를 더 낮은 정밀도로 변환
4. **평가**: 검증 데이터에서 양자화된 모델을 테스트하여 성능 확인

PTQ는 빠르고 실용적이지만 특히 INT4와 같은 극도의 양자화에서 정확도 손실을 야기합니다. 일반적인 정확도 감소는 모델 크기와 양자화 비트폭에 따라 1-5%입니다.

## 양자화 인식 학습 (QAT)

양자화 인식 학습은 양자화를 학습 프로세스에 통합합니다:

1. **가짜 양자화**: 학습 중에 각 단계에서 가중치가 양자화되고 역양자화되어 양자화의 영향을 시뮬레이션
2. **그래디언트 계산**: 양자화 연산을 통해 그래디언트 계산
3. **학습**: 모델이 양자화 제약에 적응
4. **미세조정**: 모델이 양자화 손실을 보상하도록 학습

QAT는 모델이 양자화 제약 하에서 효과적으로 작동하도록 학습하기 때문에 PTQ보다 더 나은 정확도를 달성합니다. 그러나 모델을 재학습하기 위해 학습 데이터와 계산 자원이 필요합니다.

## GPTQ (생성형 사전학습 트랜스포머 양자화)

GPTQ는 최적 뇌 양자화 프레임워크에 기반한 효율적인 INT4 양자화 방법입니다. 계층별로 작동합니다:

- **계층별 접근**: 이전에 양자화된 계층의 정보를 사용하여 한 번에 한 계층씩 양자화
- **헤시안 기반 스케일링**: 최적의 양자화 스케일을 결정하기 위해 2차 정보(헤시안 행렬) 사용
- **속도**: 70억 매개변수 모델을 시간이 아닌 분 단위로 양자화
- **품질**: 극도의 압축에도 불구하고 INT4에서 합리적인 성능 달성

GPTQ는 오픈소스 모델의 INT4 양자화 표준이 되어 커뮤니티 모델 공유 및 배포를 가능하게 했습니다.

## AWQ (활성화 인식 가중치 양자화)

AWQ는 활성화 크기를 고려하여 GPTQ를 개선합니다:

- **활성화 중요성**: 모든 가중치가 동일하게 중요하지는 않음; 더 큰 활성화를 처리하는 가중치가 더 중요
- **선택적 정밀도**: 더 중요한 가중치를 높은 정밀도로 유지하거나 덜 중요한 가중치에서 더 많은 오류를 허용하여 보호
- **더 나은 정확도**: AWQ는 INT4에서 GPTQ보다 1-2% 더 나은 정확도 달성
- **유사한 속도**: GPTQ의 효율성 장점 유지

AWQ는 현재 INT4 양자화의 최고 수준을 나타내며 효율성과 정확도의 균형을 맞춥니다.

## GGUF 형식과 양자화 변형

GGUF (GPT 생성 통합 형식)는 효율적인 추론을 위해 설계된 파일 형식입니다:

- **다양한 양자화 레벨**: GGUF는 여러 비트폭(2비트, 3비트, 4비트, 5비트, 6비트, 8비트) 지원
- **CPU 추론**: GGUF 모델은 GPU 없이 CPU에서 효율적으로 실행 가능
- **이식성**: 표준화된 형식으로 모델이 다양한 프레임워크에서 실행 가능
- **광범위한 채택**: llama.cpp 및 Ollama와 같은 인기 있는 도구에서 가능

GGUF 양자화는 GPTQ보다 덜 공격적(일반적으로 4-5비트 집중)이지만 다양한 하드웨어에서 더 나은 전체 추론 효율성을 제공합니다.

## bitsandbytes: GPU 양자화

bitsandbytes 라이브러리는 GPU 추론을 위한 양자화를 활성화합니다:

- **8비트 양자화**: LLM.int8() 양자화 중 메모리 감소
- **4비트 양자화**: nf4 (중첩 부동소수 4) 형식을 사용한 추가 메모리 감소
- **재학습 불필요**: 이미 학습된 모델에서 작동
- **QLoRA 통합**: 양자화된 모델의 매개변수 효율적인 미세조정 가능
- **seamless 통합**: Hugging Face transformers 라이브러리와 직접 작동

bitsandbytes는 GPU 하드웨어에서 미세조정 및 추론을 위해 실제로 광범위하게 사용됩니다.

## 정확도 대 효율성 트레이드오프

양자화의 근본적인 도전은 압축과 정확도의 균형입니다:

| 비트폭 | 메모리 절감 | 일반적인 정확도 감소 | 속도 | 사용 사례 |
|-------|----------|-----------------|------|----------|
| FP32  | 1배      | 0%              | 기준 | 참조    |
| INT8  | 4배      | 1-2%            | 2-3배 | 서버/GPU |
| INT4  | 8배      | 3-8%            | 4-8배 | 모바일/CPU |
| INT2  | 16배     | 10-20%          | 10-15배 | 극도 엣지 |

더 큰 모델(13B 이상)은 더 작은 모델보다 양자화를 더 잘 견딥니다. 작업 특정 요구사항이 허용 가능한 정확도 손실을 결정합니다.

## 실질적 배포 고려사항

**모델 선택**: 양자화가 잘 알려진 모델로 시작하세요; 일부 아키텍처는 양자화 친화적입니다.

**배치 양자화**: INT8은 일반적으로 보정만 필요하지만 INT4는 허용 가능한 성능을 위해 미세조정이 필요할 수 있습니다.

**혼합 정밀도**: 임베딩 및 출력 계층과 같은 중요한 계층을 더 높은 정밀도로 유지하면서 다른 것들을 양자화하는 것을 고려하세요.

**하드웨어 호환성**: 배포 하드웨어가 양자화 형식을 지원하고 약속된 속도 개선을 달성할 수 있는지 확인하세요.

**벤치마킹**: 본 배포 전에 특정 작업에서 양자화된 모델을 항상 테스트하여 실제 정확도 영향과 지연 시간 개선을 측정하세요.

양자화는 현실적인 LLM 배포를 위한 필수 인프라가 되었으며, GPU에서 모바일 장치까지 다양한 하드웨어에서 강력한 모델을 실행할 수 있게 했습니다.
