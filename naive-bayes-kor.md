# 나이브 베이즈 분류기

## 소개

나이브 베이즈 분류기는 단순화된 가정을 가진 베이즈 정리에 기초한 확률적 알고리즘입니다: 클래스 레이블이 주어질 때 피처는 조건부 독립입니다. 과도하게 단순화된 가정에도 불구하고, 나이브 베이즈는 특히 [텍스트 분류](/text-classification-kor.html)와 스팸 필터링에서 매우 효과적입니다. 빠르고 해석 가능하며 적당한 계산 리소스만 필요합니다.

## 베이즈 정리

나이브 베이즈는 베이즈 정리를 적용하여 후행 확률을 계산합니다:

**공식:**
```
P(클래스 | 피처) = P(피처 | 클래스) × P(클래스) / P(피처)
```

여기서:
- **P(클래스 | 피처)**: 후행 확률(우리가 구하려는 것)
- **P(피처 | 클래스)**: 우도(클래스가 주어질 때 피처의 확률)
- **P(클래스)**: 사전 확률(훈련 데이터의 클래스 빈도)
- **P(피처)**: 증거(비교를 위한 상수)

**분류를 위해:**
```
예측된_클래스 = argmax_c P(c) × ∏ P(f_i | c)
```

가장 높은 후행 확률을 가진 클래스를 선택합니다.

## 나이브 독립 가정

핵심 단순화 가정: 클래스가 주어질 때 피처는 조건부 독립입니다:

```
P(x₁, x₂, ..., xₙ | c) = ∏ P(x_i | c)
```

이는 계산을 극적으로 단순화하지만 실제로는 거의 참이 아닙니다. 이러한 과도한 단순화에도 불구하고, 나이브 베이즈는 **확률 상쇄**와 **독립 가정 위반에 대한 강건성** 때문에 종종 좋은 성능을 보입니다.

## 변형과 확률 모델

다양한 확률 모델은 다양한 피처 유형과 데이터 분포에 적합합니다.

### 가우시안 나이브 베이즈

연속 수치 피처의 경우 가우시안(정규) 분포를 가정합니다:

```
P(x_i | c) = (1/√(2πσ_c²)) × exp(-(x_i - μ_c)² / (2σ_c²))
```

**파라미터:**
- μ_c: 클래스 c에서 피처 i의 평균
- σ_c²: 클래스 c에서 피처 i의 분산

훈련 데이터에서 추정됩니다.

**사용 사례:**
- 연속 분포의 실수 피처
- 물리 측정값과 센서 데이터

### 다항식 나이브 베이즈

이산 계수 데이터, 특히 [텍스트 분류](/text-classification-kor.html)에 유용합니다:

```
P(x_i | c) = (count_i + 1) / (total_count + 어휘_크기)
```

여기서 count_i는 클래스 c에서 피처 i의 발생 횟수입니다.

**핵심 성질:**
- 문서의 단어 빈도 모델링
- 문서 분류에 자연스러운 적합
- NLP 응용에서 널리 사용

### 베르누이 나이브 베이즈

이진 피처(존재/부재), 문서 분류에 일반적입니다:

```
P(x_i | c) = P_i^(x_i) × (1 - P_i)^(1 - x_i)
```

여기서 P_i는 클래스 c에서 피처 i의 확률입니다.

**사용 사례:**
- 이진 피처 벡터
- 이진 단어 지시자를 가진 문서 분류

## 라플라스 평활화

평활화는 훈련 중에 피처 값이 클래스에 나타나지 않을 때 0 확률을 방지합니다.

**문제:**
P(x_i | c) = 0이면 전체 곱이 0이 됩니다.

**라플라스 평활화 해결책:**
```
P(x_i | c) = (count_i + α) / (total_count + α × K)
```

여기서:
- α: 평활화 파라미터(일반적으로 α = 1)
- K: 피처 i의 가능한 값의 개수

이는 모든 확률이 0이 아닌 상태를 유지하도록 보장합니다.

## 텍스트 분류

나이브 베이즈는 고차원 희소 피처와 단어 분포의 확률적 모델링으로 인해 [텍스트 분류](/text-classification-kor.html)에 탁월합니다.

**과정:**
1. 문서를 단어로 [토큰화](/tokenization-kor.html)
2. 각 클래스에 대한 단어 빈도 계수 구축
3. 라플라스 평활화 적용
4. 클래스 확률을 비교하여 새 문서 분류

**텍스트에서 나이브 베이즈가 작동하는 이유:**
- 단어의 높은 차원성 vs. 적당한 샘플 크기
- 독립 가정이 단어 중요도를 포착
- 대규모 텍스트 데이터셋에 적합한 빠른 계산
- 해석 가능: 학습된 피처 확률 검사 가능

## 스팸 필터링

나이브 베이즈는 효과적인 이메일 스팸 필터링으로 유명해졌습니다:

**훈련:**
- 스팸과 합법적 이메일의 단어 확률 학습
- P(단어 | 스팸)과 P(단어 | 합법)저장

**예측:**
- P(스팸 | 이메일의_단어)과 P(합법 | 이메일의_단어) 계산
- 어느 것이 더 높은지에 따라 분류

**스팸 필터링의 장점:**
- 스팸이 진화함에 따라 적응(재훈련 가능)
- 빠름, 실시간 필터링 가능
- 미지의 단어에 대한 우아한 성능 저하(평활화)

## 장점

- **속도**: 훈련과 예측이 계산상 빠름
- **단순성**: 이해하고 구현하기 쉬움
- **확률적**: 예측에 대한 확률 추정값 제공
- **해석 가능**: 학습된 피처-클래스 관계 검사 가능
- **소규모 데이터셋**: 제한된 훈련 데이터로 잘 작동
- **고차원**: 많은 피처를 효과적으로 처리
- **강건성**: 독립 가정 위반에도 불구하고 좋은 성능
- **텍스트**: [텍스트 분류](/text-classification-kor.html)와 NLP 작업에 뛰어남

## 단점

- **과도한 단순화 가정**: 피처 독립이 현실에서 거의 성립하지 않음
- **피처 상관성**: 피처 관계를 포착하지 않음
- **영빈도 문제**: 미지 데이터에 평활화 필요
- **불균형 데이터**: 극도로 불균형 클래스에서 어려움
- **연속 피처**: 가우시안 가정이 모든 분포에 맞지 않을 수 있음
- **후행 확률**: 잘 정정되지 않음, 특히 많은 피처의 경우
- **드문 사건**: 극도의 클래스 불균형이 문제 야기
- **피처 선택**: 피처 상호작용을 활용할 수 없음

## 실무 고려사항

**전처리:**
- 피처 스케일링 불필요
- 미리 결측값 처리
- 텍스트: [토큰화](/tokenization-kor.html), 소문자 변환, 불용어 제거

**파라미터 튜닝:**
- 평활화 파라미터 α(일반적으로 1.0)
- 노이즈 감소를 위한 피처 선택
- 불균형 데이터에 대한 클래스 가중치

**모델 선택:**
- 피처 유형에 따라 변형 선택(가우시안/다항식/베르누이)
- 성능 평가를 위한 [교차 검증](/cross-validation-kor.html)
- 다른 방법과 확률 정정 비교

**사용 시기:**
- [텍스트 분류](/text-classification-kor.html)(주요 사용 사례)
- 스팸 및 이메일 필터링
- 감정 분석
- 문서 분류
- 확률 출력이 있는 의료 진단

**피해야 할 때:**
- 피처 의존성이 중요할 때
- 조정 없이 극도로 불균형 데이터
- 잘 정정된 확률이 필수적일 때
- 해석 가능성이 가치가 없을 때

## 다른 분류기와의 비교

**vs. 로지스틱 회귀:**
- 나이브 베이즈: 확률적 생성 모델
- 로지스틱 회귀: 판별적, 종속 피처에 더 좋음
- 나이브 베이즈: 종종 훈련이 더 빠름

**vs. SVM:**
- [SVM](/svm-kor.html): 복잡한 결정 경계에 더 좋음
- 나이브 베이즈: 고차원 텍스트에 더 좋음
- 나이브 베이즈: 더 해석 가능

**vs. 랜덤 포레스트:**
- [랜덤 포레스트](/decision-trees-kor.html): 피처 상호작용을 더 잘 포착
- 나이브 베이즈: 더 빠르고 해석 가능
- 나이브 베이즈: 제한된 데이터로 더 좋음

## 요약

나이브 베이즈는 특히 텍스트 처리와 스팸 필터링에서 분류의 초석 알고리즘으로 남아 있습니다. 단순성, 속도, 해석 가능성, 놀라운 효과의 결합이 실용적인 도구이자 교육 기초로서 귀중합니다. 이의 가정과 한계를 이해하면 실무자가 현대적 딥러닝 접근법과 함께 적절하게 사용할 수 있습니다.
