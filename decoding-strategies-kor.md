# 디코딩 전략

## 디코딩 문제

언어 모델이 어휘의 다음 토큰에 대한 확률 분포를 생성하면 선택에 직면합니다: **어떤 토큰을 생성하도록 선택할까요?** 이 겉보기에 간단한 결정은 출력 품질, 다양성, 계산 비용에 극적으로 영향을 미칩니다. 잘못된 전략은 반복적이고 일관성 없으며 지나치게 일반적인 텍스트를 생성합니다.

디코딩은 모델 확률을 실제 텍스트로 변환하는 알고리즘입니다. 완벽한 언어 모델이 있어도 좋지 않은 디코딩은 품질을 망칩니다. 반대로 영리한 디코딩은 모델 제약을 극복할 수 있습니다.

## 탐욕 디코딩

가장 간단한 접근 방식: **항상 가장 높은 확률을 가진 토큰을 선택합니다**.

```
next_token = argmax(p(vocab))
```

### 장점
- 가장 빠른 계산 (단계당 단일 argmax)
- 결정론적이고 재현 가능
- 사실 기반 작업에 잘 작동

### 단점
- 다양성 없음: 같은 입력이 항상 동일한 출력 생성
- 루프에 쉽게 갇힘
- 따분하고 반복적인 텍스트 생성
- 가능한 덜 가능하지만 유효한 연속 탐색 안 함

탐욕 디코딩은 기준이며 생성 작업에는 거의 최선의 선택이 아닙니다.

## 빔 서치

**빔 서치**는 여러 가설(시퀀스)을 동시에 유지하고 각 단계에서 k개의 가장 유망한 가설을 확장합니다.

### 빔 서치 작동 방식

1. k(빔 너비)개의 가설로 시작, 각 점수 = log p(접두사)
2. 각 가설에 대해 모든 다음 토큰에 대한 확률 계산
3. 전역적으로 상위 k*|vocab| 후보 선택
4. 상위 k 후보를 유지하고 다음 단계를 위해 재설정
5. 모든 빔이 시퀀스 끝 토큰에 도달할 때까지 계속

### 핵심 개념: 길이 정규화

원본 로그 확률은 더 짧은 시퀀스로 편향합니다(곱셈 적음). **길이 정규화**는 점수를 시퀀스 길이로 나누어 더 길고 자세한 출력을 선호합니다:

```
score = (1/length^α) * log(p(sequence))
```

여기서 α는 일반적으로 0.5-1.0 범위입니다.

### 장점
- 병렬로 여러 가설 탐색
- 국소 최적 시퀀스 찾음
- 탐욕보다 번역/요약에 나음

### 단점
- 계산 비용이 많이 듭니다 (탐욕보다 k배 느림)
- 여전히 가능한 경로를 선호하며 창의적인 대안을 놓침
- 길이 정규화는 조정 필요
- 높은 빔 너비(k=5+)는 수확 체감

빔 서치는 번역과 구조화된 생성에 잘 작동하지만 개방형 작업에는 일반적으로 느껴질 수 있습니다.

## 샘플링 기반 방법

샘플링은 **확률성**을 도입합니다—최고 확률을 항상 선택하기보다는 분포에서 무작위로 선택합니다.

### 순수 샘플링

모델의 확률 분포에서 직접 다음 토큰을 샘플링합니다:

```
next_token ~ p(vocab)
```

이는 다양성을 생성하지만 매우 낮은 확률의 무의미한 토큰을 포함할 수 있습니다. 대부분의 모델은 나쁜 완성의 긴 꼬리를 가집니다.

## 온도 스케일링

**온도**는 확률 분포의 선명도를 제어합니다. 간단하지만 강력한 매개변수입니다.

분포는 다음이 됩니다:
```
p_t(token) = exp(logits/T) / Σ exp(logits/T)
```

- **T < 1 (차가움)**: 분포가 선명해지고 고확률 토큰을 더욱 가능하게 함
- **T = 1 (정상)**: 원본 확률
- **T > 1 (뜨거움)**: 분포가 평탄해지고 낮은 확률 토큰에 더 많은 기회 제공

### 실제 사용
- 낮은 온도(0.3-0.7)는 사실적이고 일관된 출력
- 높은 온도(1.0-2.0)는 창의적이고 다양한 출력
- 온도는 모델 의존적; 경험적으로 조정

## 상위-k 샘플링

**상위 k개의 가장 높은 확률 토큰**에서만 샘플링한 후 재정규화합니다.

```
p_filtered(token) = {
  p(token) / Z,  if token in top-k
  0,             otherwise
}
```

여기서 Z는 정규화 상수입니다.

### 장점
- 불합리한 토큰 제거 (긴 꼬리)
- 상위 후보의 다양성 유지
- 구현이 간단
- 경험적으로 잘 작동

### 단점
- k는 hard cutoff입니다—상위-k 바로 밖의 합리적인 토큰 드롭
- 사용 사례별 k 조정 필요
- 다른 토큰은 다른 꼬리 분포를 가짐

## 상위-p (핵) 샘플링

상위-k에 대한 더 영리한 대안: p 분수의 확률 질량을 포함하는 **동적 부분 집합**에서 샘플링합니다.

### 작동 방식

1. 내림차순 확률로 토큰 정렬
2. 누적 확률이 ≥ p가 되는 만큼의 토큰만 선택
3. 이 부분 집합에서 재정규화 및 샘플링

예: p=0.9이면 누적 확률이 90%에 도달할 때까지 토큰 샘플링을 계속합니다.

### 장점
- 적응형: 자연스럽게 변하는 분포 처리
- 다양하고 유동적인 텍스트에 잘 작동
- 대부분의 애플리케이션에 좋은 기본값
- 상위-k보다 덜 하이퍼매개변수 조정

### 단점
- 상위-k보다 약간 더 많은 계산
- 여전히 p 조정 필요 (일반적으로 0.85-0.95)

상위-p는 많은 애플리케이션에서 표준이 되었으며 종종 현대 LLM API의 기본값입니다.

## 반복 페널티

모델은 토큰이나 구절을 반복하는 경향이 있습니다. **반복 페널티**는 이미 나타난 토큰의 확률을 할인합니다.

시퀀스에서 n번 나타난 토큰:
```
p_penalized(token) = p(token) / (1 + n * penalty)
```

공통 페널티 값: 1.0-2.0

### 사용 시기
- 창의적 쓰기 (따분한 반복 피함)
- 장문 생성
- 채팅 및 대화

### 주의
- 너무 높은 페널티는 반복을 불가능하게 할 수 있음 (문법적일 때도)
- 번호 매기기 목록(1, 2, 3...) 같은 패턴 방해 가능

## 최소-p 샘플링

절대 수가 아닌 **상대 확률**을 기반으로 필터링하는 최근 기술입니다.

최소 확률 임계값 p_min의 경우:
```
keep tokens where: p(token) ≥ p_min * max(p(vocab))
```

따라서 상위 토큰의 확률이 0.8이고 min-p=0.1이면 확률 0.08까지 토큰을 유지합니다.

### 장점
- 상대 필터링은 상위-k의 hard cutoff보다 더 원칙적
- 품질과 다양성 간의 좋은 균형
- "사망의 손" 문제 감소(유효한 연속이 없음)

## 일반적 샘플링

정보 이론 원칙에 기반합니다. **엔트로피 범위** 내의 토큰을 샘플링합니다.

아이디어: 정보 콘텐츠가 분포의 평균에 가까운 토큰을 유지합니다. 이는 극단적(너무 확실함)과 꼬리(너무 가능성 낮음) 영역을 모두 피합니다.

### 실제 효과
- 상위-k보다 덜 반복적이고 더 유동적인 텍스트 생성
- 컨텍스트 전체에서 변하는 엔트로피를 더 잘 처리
- 조정 없이 영점 성능이 좋음

## 제약 디코딩

많은 애플리케이션에서 우리는 **출력을 특정 패턴으로 제한**하려고 합니다:
- JSON 형식
- 함수 호출
- SQL 쿼리
- 정규 표현식

### 접근 방식

**문법 기반**: 형식 문법(EBNF)에 대해 출력을 강제합니다. 유효한 완성으로 이어질 수 있는 연속만 허용합니다.

**토큰 마스킹**: 유효한 다음 토큰을 유지하고 샘플링 전에 유효하지 않은 토큰을 마스킹합니다. 일반적으로 문법 기반보다 빠름.

**안내 디코딩**: 다양성을 유지하면서 원하는 출력으로 방향을 전환하도록 확률을 조정합니다.

### 트레이드오프
- 유효한 형식을 보장하지만 품질을 감소시킬 수 있음
- 일부 출력을 불가능하게 할 수 있음 (문법이 커버하지 않으면)
- 적절한 계산 오버헤드 추가

## 전략 결합

실제로 대부분의 시스템은 여러 전략을 결합합니다:

```
1. Temperature = 0.8 (중간 온도)
2. Top-p = 0.95 (핵 샘플링)
3. Repetition penalty = 1.1 (반복 피함)
4. Min length = 10 (짧은 답 피함)
5. Max length = 500 (무한 생성 피함)
```

이 균형 잡힌 접근 방식은 명백한 실패 모드를 피하면서 일관성을 유지하는 동시에 다양성을 제공합니다.

## 올바른 전략 선택

**사실/일관된 출력의 경우**:
- 낮은 온도(0.3-0.5)
- 빔 서치 또는 탐욕
- 샘플링 없음

**창의적/다양한 출력의 경우**:
- 높은 온도(1.0-1.5)
- 상위-p 샘플링
- 반복 페널티

**구조화된 출력의 경우**:
- 제약 디코딩
- 일관성을 위해 잠재적으로 낮은 온도

**균형 잡힌 사용의 경우**:
- 온도 0.7-0.9
- 상위-p 0.9-0.95
- 반복 페널티 1.0-1.2

최상의 전략은 작업, 모델, 사용자 기대에 따라 다릅니다. 실험과 측정 품질(메트릭 또는 인간 평가를 통해)은 여전히 필수입니다.
