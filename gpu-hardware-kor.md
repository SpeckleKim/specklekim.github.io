# AI 하드웨어

## 소개

딥러닝과 AI 워크로드는 전통적인 CPU를 넘어 특화된 하드웨어가 필수입니다. GPU(그래픽 처리 장치)는 현대 AI 인프라의 근간이 되어 있으며, 대규모 학습과 추론에 필요한 거대한 병렬성을 가능하게 합니다. GPU 아키텍처, 메모리 시스템, 하드웨어 선택 이해는 효율적인 AI 시스템 구축의 핵심입니다.

## GPU 아키텍처 기초

### CUDA 코어와 테ensor 코어

NVIDIA의 GPU 아키텍처는 두 가지 주요 연산 단위로 이루어집니다:

**CUDA 코어**는 병렬 계산을 위해 설계된 범용 처리 장치입니다. 각 CUDA 코어는 독립적으로 단순한 산술 연산을 실행할 수 있어 대규모 병렬 워크로드에 이상적입니다. 최신 GPU는 수천 개의 CUDA 코어를 스트리밍 멀티프로세서(SM)로 조직화합니다.

**테ensor 코어**는 신경망의 기본 단위인 행렬 연산을 위해 특화된 프로세서입니다. 테너 코어는 단일 클록 사이클에 작은 행렬에 대한 융합 곱셈-누적(FMA) 연산을 수행합니다. 예를 들어 NVIDIA A100은 SM당 108개의 테너 코어를 가지고 있으며 다양한 정밀도(FP32, FP16, TF32, INT8)를 지원합니다.

각 GPU 세대에서 테너 코어 대 CUDA 코어의 비율이 증가하고 있으며, 이는 하드웨어 설계의 AI 중심 진화를 반영합니다.

### 메모리 계층

GPU 메모리는 엄격한 계층을 따릅니다:

- **레지스터**: 초고속 스레드 전용 메모리 (나노초 지연)
- **공유 메모리**: SM 간의 빠른 캐시 (마이크로초 지연)
- **L1/L2 캐시**: 글로벌 메모리 자동 캐싱
- **글로벌 메모리(VRAM)**: 가장 큼, 느린 GPU 메모리 (마이크로초 지연, 높은 대역폭)
- **호스트 메모리(RAM)**: PCIe로 접근 가능한 시스템 RAM

메모리 대역폭은 AI 워크로드에 매우 중요합니다. A100은 최대 2TB/s 대역폭(NVLink 포함)을 제공하고, V100 같은 이전 GPU는 PCIe로 900GB/s를 제공합니다.

## NVIDIA 아키텍처

### A100 (Ampere, 2020)

NVIDIA A100은 다음의 기능으로 AI 컴퓨팅의 표준을 설정했습니다:
- 40GB 또는 80GB HBM2 메모리
- SM당 108개 테너 코어, 총 108개 SM
- TF32, BF16, FP16, INT8 기본 지원
- 내장 희소성 가속화 (희소 모델의 50% 높은 처리량)
- 학습과 추론 모두에 강력함

### H100 (Hopper, 2022)

H100은 경계를 더욱 밀어붙였습니다:
- 3.2TB/s 대역폭의 80GB HBM3 메모리
- SM당 4개 행렬 엔진으로 높은 테너 처리량
- 희소 테너 연산을 위한 DPX
- 자동 혼합 정밀도를 갖춘 Transformer Engine
- LLM 학습과 추론에 훨씬 빠름

### 기타 아키텍처

- **V100 (Volta, 2017)**: 테너 아키텍처 개척자, 이전 배포에서 여전히 사용
- **L40S (Ada, 2023)**: 추론 워크로드에 최적화, 혼합 정밀도 지원
- **RTX 4090 (Ada Consumer)**: 소규모 학습의 저렴한 대안 (24GB VRAM)

## TPU와 대체 가속기

### Google TPU

Tensor Processing Unit은 TensorFlow 워크로드용으로 설계된 주문형 집적회로(ASIC)입니다:
- TPUv4: 행렬 곱셈의 최대 275 테라플롭스
- 조밀한 정규 계산에 뛰어남 (특히 트랜스포머 모델)
- GCP와 통합되어 원활한 확장 가능
- 특정 워크로드에서 GPU보다 전력 효율적

### NPU와 엣지 가속기

Neural Processing Unit은 엣지 장치를 대상으로 합니다:
- Apple Neural Engine: M-시리즈 칩의 16코어 NPU
- Qualcomm Hexagon NPU: Snapdragon 프로세서의 NPU
- INT8/INT4 추론에 최적화되어 저전력 소비

## 주요 성능 지표

### TFLOPS

초당 부동소수점 연산 수를 측정합니다:
- A100: 312 TFLOPS (FP32), 1,248 TFLOPS (TF32)
- H100: 989 TFLOPS (FP32), 1,456 TFLOPS (TF32)
- 중요한 기초이지만 메모리 대역폭 병목을 고려하지 않음

### 메모리 대역폭

메모리와 계산 사이의 데이터 처리량을 측정합니다:
- 대역폭 = 데이터 폭 × 클록 속도 × 2 (DDR의 경우)
- 산술 강도가 낮은 모델에 중요 (낮은 계산-메모리 비율)
- H100: HBM3로 3,200 GB/s
- 추론 워크로드에서 점점 더 중요해짐

## 하드웨어 선택 지침

### 학습용

- **예산 제약 없음**: NVLink/InfiniBand가 있는 H100 클러스터
- **예산 의식**: 모델 크기에 따라 A100 또는 RTX 4090
- **다중 노드 확장**: GPU 상호연결 대역폭 우선순위
- **메모리 요구사항**: 더 큰 모델은 40GB+ VRAM 필요

### 추론용

- **처리량 우선**: L40S 또는 RTX 4090
- **지연 시간 우선**: 낮은 배치 크기, 작은 모델을 위해 CPU 추론 고려
- **추론 당 비용**: 조밀한 정규 워크로드의 경우 TPU
- **엣지 배포**: NPU 또는 INT8 양자화 모델

## 비용 고려사항

GPU 비용은 다음으로 구성됩니다:
- **자본**: 고성능 GPU당 $10k-$40k
- **전력 소비**: A100 (250W) vs H100 (700W)
- **냉각 인프라**: GPU는 상당한 열 발생
- **총 소유 비용**: 수년 상각을 계산해야 함
- **스팟 가격**: 클라우드 공급자는 선점 가능 인스턴스에 대해 70-80% 할인 제공

## 미래 동향

- **칩 밀도 증가**: 5nm 및 3nm 공정으로 더 많은 코어 가능
- **특화된 정밀도**: BF16이 학습의 표준이 됨
- **온칩 포토닉스**: GPU 간 통신을 위해 광학 상호연결로 전기 대체
- **ASIC 특화**: 특정 모델 아키텍처용 맞춤형 실리콘

## 결론

올바른 하드웨어 선택은 올바른 알고리즘 선택만큼 중요합니다. 워크로드의 계산-메모리 비율, 정밀도 요구사항, 확장성 필요를 이해하는 것이 최적의 하드웨어 선택을 안내합니다. GPU 환경은 빠르게 진화하지만, 병렬성, 메모리 계층, 대역폭 주변의 기본 원칙은 AI 가속의 중심으로 남습니다.
