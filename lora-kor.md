# LoRA와 효율적 미세조정

매개변수 효율적 미세조정(Parameter-Efficient Fine-Tuning, PEFT) 방법은 중대한 실질적 도전을 해결합니다: 대규모 언어 모델의 완전 미세조정은 계산적으로 불가능합니다. LoRA(Low-Rank Adaptation)는 지배적 접근 방식으로 떠올라 거대한 모델의 비용 효율적 적응을 가능하게 합니다.

## 완전 미세조정의 병목

완전 미세조정은 모든 모델 매개변수를 업데이트하므로 다음이 필요합니다:
- **메모리**: 모든 매개변수에 대한 옵티마이저 상태와 그래디언트 저장. 70억 모델은 70GB 이상 RAM 필요
- **계산**: 각 학습 샘플에 대해 모든 계층을 통한 역전파
- **시간**: 비싼 하드웨어를 사용해도 며칠 또는 몇 주의 학습
- **인프라**: 엔터프라이즈급 GPU(H100s, A100s) 필요

이것은 역설을 만듭니다: 미세조정이 적응에 필수적이지만 잘 자금을 받은 조직만 감당할 수 있습니다. PEFT 방법은 모델 맞춤화를 대중화합니다.

## LoRA: 저계수 적응

LoRA는 간단하지만 효과적인 통찰을 소개합니다: 적응은 매개변수의 작은 부분만 효과적으로 업데이트해야 합니다. 가중치 행렬 W를 직접 업데이트하는 대신 LoRA는 학습 가능한 저계수 분해를 추가합니다:

```
W' = W + ΔW = W + BA
```

여기서:
- W는 원래 사전학습 가중치 행렬
- B와 A는 훨씬 적은 매개변수를 가진 저계수 행렬
- 크기 (d_in × d_out)의 가중치 행렬에서 계수 r을 사용하면:
  - 원래 매개변수: d_in × d_out
  - LoRA 매개변수: d_in × r + r × d_out (일반적으로 r << d)

일반적인 LoRA 계수=8인 70억 모델의 경우:
- 완전 미세조정: ~28GB 매개변수
- LoRA 미세조정: ~12MB 학습 가능한 매개변수

이 2000배 학습 가능한 매개변수 감소는 경쟁력 있는 성능을 유지하면서 메모리와 계산 요구사항을 극적으로 줄입니다.

## LoRA 학습 프로세스

1. **초기화**: B를 0으로, A를 무작위로 초기화(또는 반대)하여 ΔW = 0으로 시작
2. **고정 베이스**: 사전학습된 가중치 W는 전체 학습 동안 고정
3. **적응 계산**: 순전파 중 W' = W + BA 계산
4. **LoRA만 업데이트**: 그래디언트는 B와 A 매개변수만 업데이트
5. **배포**: ΔW를 W로 다시 병합: W_new = W + BA, 또는 추론을 위해 분리하여 유지

이 접근 방식은 기존 추론 코드와의 호환성을 유지하면서 학습 비용을 크게 줄입니다.

## QLoRA: 양자화된 모델의 효율적 미세조정

QLoRA는 LoRA를 양자화된 모델로 확장하여 더욱 공격적인 압축을 가능하게 합니다:

**핵심 혁신**: LoRA를 4비트 양자화와 결합
- 기본 모델을 4비트로 양자화(크기 28GB에서 3.5GB로 감소)
- 미세조정을 위해 LoRA 어댑터 추가
- 8비트 옵티마이저 상태를 사용하여 미세조정

**결과**: 70억 모델은 QLoRA 미세조정을 위해 7GB GPU 메모리만 필요하며, 소비자 GPU에서 접근 가능하게 만듭니다.

QLoRA는 필요한 메모리의 5% 미만으로 완전 미세조정과 비교 가능한 성능을 달성하여 개별 연구자의 대규모 모델 적응을 실용적으로 만듭니다.

## 어댑터: 모듈식 미세조정

어댑터는 트랜스포머 계층 사이에 학습 가능한 모듈을 도입합니다:

```
x' = Adapter(x) + x  # 잔차 연결
```

**아키텍처**:
- 트랜스포머 블록에 삽입된 작은 병목 계층
- 더 낮은 차원(예: 64)으로 다운프로젝션하고, 처리한 후 다시 업프로젝션
- 일반적으로 추가 매개변수의 0.5-3%

**장점**:
- 모듈식: 다양한 작업을 위해 다양한 어댑터를 구성할 수 있음
- 조합 가능: 다중 작업 학습을 위해 여러 어댑터 결합
- 유연함: 다양한 계층 유형에 적용 가능

어댑터는 작업당 하나의 어댑터가 단일 LoRA보다 더 유연한 다중 작업 시나리오에 특히 유용합니다.

## 프리픽스 튜닝

프리픽스 튜닝은 각 시퀀스의 시작 부분에 학습 가능한 토큰을 추가합니다:

**핵심 아이디어**: 모델 가중치가 아닌 프롬프트 프리픽스만 학습 가능
- 입력 시작에 k개의 학습 가능한 토큰 추가
- 이러한 "소프트 프롬프트"는 경사하강법을 통해 최적화
- 모델 가중치는 고정된 상태 유지

**특성**:
- 매우 매개변수 효율적(k × hidden_dim 매개변수)
- 맥락 내 학습처럼 작동하지만 최적화됨
- 작업 특정 또는 명령 조건부 가능

프리픽스 튜닝은 프롬프트 기반 학습에 특히 효과적이며 프롬프트 엔지니어링 기법과 잘 작동합니다.

## IA3: 억제에 의한 주입 어댑터

IA3는 요소별 곱셈을 적용하여 작업 특정 스케일링을 주입합니다:

**메커니즘**:
- 주의력의 쿼리, 키, 값 벡터에 대한 스케일링 계수 학습
- 피드포워드 네트워크에 대한 스케일링 계수 학습
- 최소한의 매개변수 오버헤드(스케일링 벡터만)

**특성**:
- 극도의 매개변수 효율성: LoRA보다 적은 매개변수
- 복잡한 적응을 위해 LoRA보다 덜 유연
- 주의력 기반 아키텍처에 전문화

IA3는 최대 매개변수 효율성이 중요할 때 유용하지만 LoRA보다 덜 일반적으로 사용됩니다.

## PEFT 방법 비교

| 방법 | 매개변수 (70억) | 메모리 (QLoRA) | 유연성 | 조합성 | 사용 사례 |
|------|----------------|----------------|--------|--------|----------|
| 완전 미세조정 | 28GB | 70GB+ | 최고 | 불가 | 연구/무제한 예산 |
| LoRA | 12MB | 7GB | 높음 | 좋음 | 범용, 단일 작업 |
| QLoRA | 12MB | 7GB | 높음 | 좋음 | 제한된 GPU, 소비자 장치 |
| 어댑터 | 100MB | 10GB | 높음 | 탁월 | 다중 작업 학습 |
| 프리픽스 튜닝 | <10MB | 7GB | 중간 | 좋음 | 프롬프트 기반 학습 |
| IA3 | <5MB | 7GB | 낮음 | 나쁨 | 극도의 효율성 필요 |

## PEFT와의 실질적 구현

**Hugging Face PEFT 라이브러리**는 사용하기 쉬운 구현을 제공합니다:

```python
from peft import LoraConfig, get_peft_model

config = LoraConfig(
    r=8,  # 저계수
    lora_alpha=32,
    target_modules=["q_proj", "v_proj"],
    lora_dropout=0.05
)

model = get_peft_model(base_model, config)
# 이제 표준 학습 루프로 미세조정
```

**실질적 팁**:
- 계수=8로 시작하고 작업 복잡성에 따라 조정
- 더 높은 계수 = 더 높은 표현력이지만 더 많은 매개변수
- 주요 모듈 대상: 쿼리/값 프로젝션이 대부분의 적응을 포착
- 메모리 제약 시나리오에서 양자화와 결합

## 각 방법을 사용할 때

**LoRA**: 대부분의 미세조정 시나리오의 기본 선택; 효율성과 효과의 탁월한 균형.

**QLoRA**: GPU 메모리가 심각하게 제한될 때; 10-20배 적은 메모리로 성능 유지.

**어댑터**: 다중 작업 또는 다중 도메인 시나리오; 모듈성이 매개변수 효율성보다 중요.

**프리픽스 튜닝**: 프롬프트 기반 학습 또는 맥락 내 학습과의 호환성이 원할 때.

**IA3**: 극도의 매개변수 효율성이 중요하고 작업이 곱셈형 스케일링에 적합할 때.

## 프로덕션 시스템에 미치는 영향

PEFT 방법은 언어 모델 배포의 경제를 근본적으로 변화시켰습니다:

- **개별 연구자**는 이제 소비자 GPU에서 70억-700억 모델을 미세조정할 수 있음
- **빠른 실험**이 느린, 비싼 학습 파이프라인을 대체
- **전문 모델**은 틈새 도메인과 작업에 실용적
- **다중 작업 시스템**은 공유 기본 모델을 작업 특정 어댑터와 함께 사용 가능

LoRA 및 관련 방법은 업계 표준이 되었으며 다양한 애플리케이션 전반에 걸친 전문 언어 모델의 확산을 가능하게 했습니다.
