# 워드 임베딩: 원-핫 인코딩에서 밀집 표현으로

워드 임베딩은 단어를 연속 공간의 밀집 벡터로 표현합니다. 단어 간의 의미론적 및 통사론적 관계를 포착하며, 현대 NLP의 기초가 되었습니다. 희소 원-핫 표현에서 밀집 임베딩으로의 진화는 언어 모델링 능력을 근본적으로 변화시켰습니다.

## 원-핫 인코딩의 문제점

임베딩 이전에 표준 접근 방식은 원-핫 인코딩이었습니다:

```
어휘: ["고양이", "개", "쥐", "뛴다", "잔다"]
고양이: [1, 0, 0, 0, 0]
개:   [0, 1, 0, 0, 0]
쥐: [0, 0, 1, 0, 0]
```

**한계:**
- **희소성**: 0이 아닌 요소가 하나뿐; 계산상 낭비적
- **높은 차원성**: 어휘 크기가 벡터 차원 결정
- **의미 정보 없음**: 직교 벡터는 유사성 신호 없음
- **저조한 일반화**: 미지의 단어 표현 불가능

원-핫 인코딩은 모든 단어를 완전히 독립적으로 취급하며, "고양이"와 "개"가 의미론적으로 관련 있다는 것을 캡처할 방법이 없습니다.

## 분포 가설 (Distributional Hypothesis)

핵심 통찰: *비슷한 맥락에서 나타나는 단어는 비슷한 의미를 갖는다*.

```
"고양이가 매트 위에 앉았다."
"개가 매트 위에 앉았다."
→ "고양이"와 "개"는 비슷한 표현을 가져야 함
```

이 가설은 모든 현대 워드 임베딩의 기초입니다. 대규모 말뭉치를 분석함으로써, 비슷한 맥락(주변 단어)에서 나타나는 단어들이 임베딩 공간의 가까운 영역에 위치해야 한다는 것을 발견할 수 있습니다.

## Word2Vec: 게임 체인저

구글의 2013년 Word2Vec는 대규모 레이블 없는 말뭉치에서 밀집 임베딩을 효율적으로 학습하는 방법을 도입했습니다. 두 가지 아키텍처:

### Skip-gram 모델

주어진 단어로부터 주변 맥락 단어 예측:

```
입력: "앉았다"
예측: {"위에", "매트", "고양이", "에"} (±2 윈도우에서)
```

모델은 예측 오류를 최소화하는 단어 벡터를 학습합니다. 비슷한 맥락을 가진 단어들은 비슷한 벡터를 학습합니다.

**훈련:**
1. 말뭉치에서 맥락 윈도우를 가진 단어 선택 (예: ±5 단어)
2. 단어 벡터를 얕은 [신경망](/neural-kor.html)에 공급
3. 맥락 단어에 대한 확률 분포 예측
4. [역전파](/backpropagation-kor.html)로 단어 벡터 업데이트
5. 수십억 단어-맥락 쌍에 대해 반복

**장점:**
- 드문 단어에 잘 작동 (각 출현이 훈련 예제)
- 의미론적으로 의미 있음: "왕 - 남자 + 여자 ≈ 여왕"
- 의미론적 및 통사론적 관계 포착

### 연속 단어 주머니 (Continuous Bag of Words, CBOW)

역 작업: 맥락 단어가 주어지면 목표 단어 예측:

```
맥락: {"매트", "위에", "고양이", "에"}
예측: "앉았다"
```

CBOW는 맥락 벡터를 평균화하고 중심 단어를 예측합니다.

**비교:**
| Skip-gram | CBOW |
|---|---|
| 드문 단어에 좋음 | 빠르고, 빈번한 단어에 좋음 |
| 각 맥락이 훈련 샘플 생성 | 여러 맥락이 하나로 병합 |
| 통사론적 관계 잘 포착 | 좋은 의미론적 관계 |

## GloVe: 전역 벡터

스탠포드의 GloVe는 행렬 인수분해와 지역 맥락 윈도우 통찰을 결합합니다. 전역 공현 통계를 명시적으로 포착합니다:

**동기:**
- Word2Vec은 지역 맥락 윈도우만 사용 (전역 패턴 누락)
- 행렬 인수분해는 전역 공현을 포착 (하지만 맥락 무시)
- GloVe는 두 접근법을 병합

**알고리즘:**
1. 말뭉치에서 단어 공현 행렬 구성
2. 모델은 단어-맥락 공현 확률 예측
3. 가중 최소제곱 최적화가 벡터 학습
4. 벡터는 전역 패턴과 지역 맥락 포착

**핵심 통찰:** 공현 빈도로 손실을 가중치화하여, 희귀하거나 매우 일반적인 조합이 아닌 의미 있는 단어 쌍에 모델 집중.

GloVe 임베딩은 종종 유추 작업에서 우수한 성능을 보여주며, 특히 좋은 의미론적 조직을 나타냅니다.

## FastText: 부분단어 임베딩

페이스북의 FastText는 Word2Vec을 부분단어 수준으로 확장합니다:

```
"달리고있다" = ["<달", "달", "리고", "고있", "있다>"]
```

각 단어는 문자 n-그램 벡터의 합으로 표현됩니다.

**장점:**
- 형태론적으로 복잡한 언어 처리
- 드문 단어와 어휘 외 단어에 더 나음
- 문자 수준 패턴 포착 ("달리고"와 "달리는"은 형태론적 부분문자열 공유)

**단점:**
- 더 큰 모델 (부분단어 임베딩 저장 필요)
- 느린 추론

## 맥락 임베딩: ELMo

초기 언어 모델 (ELMo)은 맥락 종속 임베딩을 도입했습니다. 단어당 하나의 고정 벡터 대신:

```
"은행"이 "강변"에서 → 다른 벡터
"은행"이 "저축은행"에서 → 다른 벡터
```

ELMo는 언어 모델링에 훈련된 양방향 LSTM을 사용하여 맥락에 종속된 표현을 생성합니다.

**아키텍처:**
- 양방향 LSTM 언어 모델 훈련 (토큰 → ±맥락 → 다음 토큰)
- 하위 작업을 위해 모든 LSTM 계층의 숨겨진 상태 연결
- 임베딩은 모든 계층의 가중 조합

**중요성:**
- 맥락이 표현을 동적으로 변경
- 다양한 작업이 다양한 계층에서 이점
- 거의 모든 하위 NLP 작업에서 성능 향상
- 사전 훈련된 맥락화된 임베딩의 시대 도입

## 임베딩 시각화

임베딩은 차원성 축소를 사용하여 시각화할 수 있습니다 (t-SNE, UMAP):

```
의미론적 클러스터가 자연스럽게 나타남:
- 국가들이 함께 클러스터
- 남성/여성 직업 쌍이 정렬
- 관련 개념이 가까이 그룹화
```

이 시각화는 단순 공현 통계에서 의미 있는 의미론적 구조가 나타난다는 것을 보여줍니다.

## 유추 작업

고전적 평가: 단어 유추 해결:

```
"남자"가 "여자"에 대한 것처럼 "왕"은 ?에 대한 것이다
→ king - man + woman ≈ queen

"서울"이 "한국"에 대한 것처럼 "도쿄"는 ?에 대한 것이다
→ tokyo - paris + france ≈ japan
```

이것이 작동하는 이유는 임베딩이 관계 구조를 포착하기 때문입니다:

```
king - man ≈ woman - queen
(성별 관계 벡터 포착)
```

유추 작업은 임베딩이 다음을 인코딩함을 보여줍니다:
- 의미론적 관계 (동의어, 반의어, 상위개념)
- 통사론적 관계 (단수/복수, 동사 시제)
- 지리적 관계 (수도-국가)
- 문화적 관계 (영화-배우)

## 임베딩이 중요한 이유

1. **정보 밀도**: 밀집 벡터는 모든 차원에 정보 전달
2. **유사성**: 임베딩 간 코사인 거리는 의미론적 유사성 상관
3. **합성성**: 단어 벡터를 조합 가능 (king - man + woman ≈ queen)
4. **전이**: 한 작업에서 학습한 임베딩이 종종 다른 작업에 이점
5. **효율성**: 밀집 벡터의 희소 연산이 종종 원본 희소성 처리보다 빠름

## 한계와 진화

전통 워드 임베딩의 한계:
- 다의성을 무시하는 단일 표현 (다중 의미)
- 정적 벡터는 맥락에 적응할 수 없음
- 드문 단어는 저조한 표현
- 실용적 의미 포착 불가능

이런 한계들이 맥락화된 임베딩 (ELMo, [BERT](/bert-kor.html))을 동기부여했으며, 작업 및 맥락 특화 표현을 학습합니다.

## 현대적 임베딩 환경

현대 NLP는 일반적으로 Word2Vec/GloVe를 직접 사용하지 않고:
- BERT가 맥락화된 임베딩 생성
- 문장-[BERT](/bert-kor.html) (SBERT)가 의미론적으로 의미 있는 문장 벡터 계산
- 특정 도메인을 위한 특화된 모델 (임상 [BERT](/bert-kor.html), 과학 BERT)
- 검색 모델이 의미론적 검색을 위해 임베딩 공간 사용

하지만 Word2Vec과 GloVe는 여전히 가치 있습니다:
- 자원 제약 환경을 위한 경량 대안
- 임베딩 기초에 대한 좋은 이해
- 대규모 유사성 검색에 효율적
- 해석 가능하고 잘 연구됨

## 결론

워드 임베딩은 의미가 대규모 텍스트 말뭉치의 분포 패턴에서 나타난다는 것을 보여줌으로써 NLP를 혁신했습니다. Word2Vec의 효율성과 단순성은 임베딩을 대규모로 실용적으로 만들었고, GloVe와 FastText는 전역 통계와 형태론적 구조를 통합하는 방법을 보여주었습니다. 원-핫에서 Word2Vec으로 맥락화된 임베딩으로의 진행은 NLP의 기본 궤적을 보여줍니다: 점점 더 정교한 더 풍부한 언어 정보를 포착하는 표현. 임베딩 기반 방법을 이해하는 것은 현대 [트랜스포머](/llm-kor.html) 기반 모델을 위한 필수 기초를 제공합니다.
