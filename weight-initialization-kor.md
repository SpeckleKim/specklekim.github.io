# 가중치 초기화: 딥러닝의 기초 설정

가중치 초기화는 깊은 신경망 훈련에서 중요하면서도 종종 간과되는 측면입니다. 부실한 초기화는 느린 수렴, 막힌 기울기, 완전한 훈련 실패로 이어질 수 있는 반면, 적절한 초기화는 학습을 가속화하고 최종 모델 성능을 개선합니다.

## 초기화가 중요한 이유

무작위 초기화에서 깊은 네트워크를 훈련하는 것은 선행 정보 없이 복잡한 지형을 항해하는 것과 유사합니다. 초기화는 다음을 결정합니다:

1. **수렴 속도**: 좋은 초기화는 훈련 시간을 획기적으로 단축
2. **최종 모델 품질**: 네트워크가 최적 해결책에 도달하는지 여부 영향
3. **안정성**: 기울기 폭발이나 소실 방지
4. **재현성**: 비볼록 문제에서 다양한 초기화는 다양한 해로 이어짐

신중한 초기화 없이 네트워크는 나쁜 국소 최소값에 갇히거나 훈련하지 못할 수 있습니다.

## 영(0) 초기화의 문제

가장 명백한 초기화 전략—모든 가중치를 0으로 설정—은 치명적입니다:

```
W = 0이면 모든 신경원이 동일한 출력 계산
모든 역전파 기울기가 동일
층이 아무것도 학습하지 않음 (모든 신경원이 동일하게 유지)
```

이 실패는 무작위화가 왜 필수인지 보여줍니다. 그러나 무작위 초기화는 신중하게 스케일되어야 합니다.

## 기본 무작위 초기화

단순히 정규분포 N(0, 1)에서 추출하면 깊은 네트워크에서 문제가 생깁니다:

- **깊은 층**: 활성화가 층을 통해 지수적으로 증가/감소
- **기울기 흐름**: 깊은 네트워크에서 기울기 소실
- **분산 불일치**: 네트워크 출력 분산이 층별로 변함

층을 고려: y = σ(Wx + b). W ~ N(0, 1)이고 x의 분산이 1이면, y는 층 깊이에 따라 분산이 폭발/소실합니다.

## Xavier/Glorot 초기화

Xavier 초기화는 층 전체에 일관된 활성화 분산을 유지합니다:

```
W ~ Uniform[-√(6/(n_in + n_out)), √(6/(n_in + n_out))]
또는 동등하게
W ~ Normal(0, √(2/(n_in + n_out)))
```

핵심 아이디어: 팬인(n_in)과 팬아웃(n_out)에 따라 초기화 분산을 스케일링합니다.

**설계 원칙**:
- 전활성화 분산 = 1
- 역방향 기울기 분산 ≈ 1 유지

**최적 대상**: 시그모이드와 하이퍼볼릭 탄젠트 활성화 함수

**한계**: 층 간 선형 활성화 가정 (시그모이드/tanh에는 대략 참, ReLU에는 거짓)

## He/Kaiming 초기화

ReLU 네트워크용으로 개발된 He 초기화는 ReLU의 성질을 고려합니다:

```
W ~ Normal(0, √(2/n_in))  [He uniform 버전도 존재]
```

ReLU는 신경원의 절반을 죽입니다 (음의 활성화 → 0), 따라서 초기화 분산은 전체 분산을 유지하기 위해 2배 증가해야 합니다.

**핵심 통찰력**: 전활성화의 절반만 활성화
- Xavier: 모든 신경원이 출력에 기여한다고 가정
- He: ReLU의 음수 값 영점화를 고려

**스케일링 계수**:
```
초기화_스케일 = √(2/n_in)  [ReLU용]
대
초기화_스케일 = √(1/n_in)  [Xavier를 통한 시그모이드/tanh용]
```

**최적 대상**: ReLU, LeakyReLU, ELU 및 다른 ReLU 계열 활성화

## 직교 초기화

순환 및 합성곱 네트워크의 경우, 직교 초기화는 직교 행렬을 사용합니다:

```
W는 직교 행렬로 초기화 (W^T W = I)
입력 노름을 네트워크 전체에 보존
기울기 폭발/소실 방지
```

장점:
- 모든 특이값 = 1, 기울기 흐름 완벽
- 많은 층의 RNN에 특히 효과적
- 적절한 스케일링과 결합하면 잘 작동

구현: 임의 가우스 행렬의 QR 분해 사용.

## 층-순차 유닛-분산 (LSUV)

LSUV는 네트워크를 통한 계산 후 초기화를 미세 조정합니다:

1. 가중치 초기화 (예: Xavier)
2. 출력 분산을 계산하기 위해 네트워크를 통한 순전파
3. 출력 분산 ≈ 1이 되도록 가중치 조정
4. 각 층에 대해 반복

장점:
- 표준 방법보다 경험적으로 우수
- 비선형 후 실제 분산 해결
- 다양한 활성화 함수에서 더 나음
- 더 비용이 많이 듦 (초기화 중 순전파 필요)

## 다양한 활성화 함수에 대한 초기화

| 활성화 | 방법 | 스케일 | 이유 |
|-------|------|-------|------|
| 시그모이드/Tanh | Xavier | √(1/n_in) | [-1,1] 기울기 죽임, 신중한 스케일 필요 |
| ReLU | He | √(2/n_in) | 신경원 절반 비활성, 분산 2배 필요 |
| Leaky ReLU | He 변형 | √(2/(n_in(1+α²))) | 누수 계수 α 고려 |
| ELU | He | √(2/n_in) | ReLU와 유사 |
| SELU | SELU 특화 | √(1/n_in) | 네트워크가 자기정규화 |
| 선형 (출력) | Xavier/He | 의존성 | 이전 층과 일치 |

## 잔여 네트워크와 초기화

스킵 연결(ResNets)이 있는 네트워크:
- 표준 He 초기화 여전히 작동
- 스킵 연결이 기울기 소실 문제 완화
- 매우 깊은 네트워크의 훈련 가능
- 그러나 초기 훈련 안정성을 위해 초기화는 여전히 중요

## 배치 정규화와의 상호작용

배치 정규화 사용 시:
- 초기화가 덜 중요해짐
- 배치 정규화가 스케일링 효과 흡수
- 더 넓은 초기화 범위 사용 가능
- 여전히 He 또는 Xavier 사용이 좋은 실무

## 현대 딥러닝 초기화

현재 최고 실무:
1. **ReLU 네트워크에 He 사용**: 현대 아키텍처의 기본 선택
2. **적절한 스케일링 사용**: N(0, 1)만 사용하지 말 것
3. **경험적으로 검증**: 훈련 초기에 활성화 분포 확인
4. **아키텍처 고려**: 트랜스포머, 스킵 연결, 정규화 층이 필요 영향
5. **과도하게 생각하지 말 것**: 대부분 현대 프레임워크는 좋은 초기화 기본값

## 일반적 초기화 실수

- N(0, 1)을 스케일링 없이 사용
- 모든 층에 동일한 초기화
- ReLU와 시그모이드 조정 미실행
- 배치 정규화 효과 무시
- 편향을 무작위로 초기화 (보통 b = 0)
- 기본 동기 이해 부족

## 실무 초기화 체크리스트

새로운 아키텍처 구현 시:
1. 사용된 활성화 함수 선택
2. 적절한 초기화 방법 선택
3. 네트워크 실행 및 첫 층 활성화 통계 확인
4. 활성화가 폭발/소실하지 않는지 확인
5. 초기 훈련 중 기울기 노름 모니터링
6. 필요 시 조정 (분산 스케일, 다른 방법 시도)

## 검증 방법

초기화가 효과적인지 확인:
```python
# 초기화 후, 훈련 전:
# 활성화 통계 확인
assert activation_mean ≈ 0
assert activation_std ≈ 0.5-1.0

# 기울기 노름은 층 전체에서 유사해야 함
for layer in network:
    assert gradient_norm ≈ constant
```

## 결론

가중치 초기화는 단순한 기술적 세부사항이 아니라 성공적인 딥러닝의 기초입니다. Xavier와 He 초기화는 일반적인 병리를 방지하는 이론적으로 동기 부여된 접근법을 제공합니다. 다양한 초기화 체계의 근거—층 깊이, 활성화 함수, 네트워크 아키텍처를 설명—을 이해하면 실무자가 정보 있는 선택을 할 수 있습니다.

현대 프레임워크는 좋은 초기화를 기본값으로 만들지만, 이러한 원칙을 이해하면 훈련 실패를 디버깅하고 더 나은 아키텍처를 설계하는 데 도움이 됩니다. 핵심 통찰력은 여전히: 네트워크 구조와 활성화 함수를 기반으로 무작위 가중치를 적절히 스케일링하여 안정적인 순전파와 역전파 정보 흐름을 유지합니다.
