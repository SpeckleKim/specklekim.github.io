# 교차검증과 모델 선택

교차검증은 모델 성능을 평가하고 최적의 모델을 선택하기 위한 핵심 기법입니다. 단순한 훈련-테스트 분할과 달리, 교차검증은 일반화 오류의 더욱 견고한 추정치를 제공하고 성능 추정의 분산을 감소시킵니다.

## 단순 훈련-테스트 분할의 문제점

데이터를 한 번만 훈련 세트와 테스트 세트로 나누면, 모델 성능 추정치는 각 세트에 포함된 샘플에 크게 달라집니다. 특히 데이터가 제한적일 때 다른 무작위 분할은 매우 다른 성능 추정치를 생성할 수 있으며, 이는 모델 비교를 어렵게 만듭니다.

## 훈련-검증-테스트 분할

3계층 분할은 표준적인 접근 방식입니다:
- **훈련 세트** (60-70%): 모델 파라미터 학습에 사용
- **검증 세트** (10-20%): 하이퍼파라미터 튜닝 및 특성 선택에 사용
- **테스트 세트** (10-20%): 최종 평가용으로 예약, 단 한 번만 사용

검증 세트는 모델 개발 중 테스트 세트 과적합을 방지합니다.

## k-Fold 교차검증

k-Fold CV는 데이터를 k개의 동등한 크기 폴드로 반복 분할하고 k개의 모델을 훈련합니다:

1. 폴드 1: 폴드 2-k로 훈련, 폴드 1로 검증
2. 폴드 2: 폴드 1, 3-k로 훈련, 폴드 2로 검증
3. 이런 식으로 반복

최종 점수는 모든 k 검증 점수의 평균입니다. 일반적으로 k=5 또는 k=10을 사용합니다. 장점:
- 훈련과 검증 모두에 모든 데이터 사용
- 성능 추정의 분산 감소
- 훈련-테스트 분할이 데이터를 낭비하는 작은 데이터셋에 유리

CV 추정의 표준 오류는 대략 `σ / √k`입니다. 여기서 σ는 폴드 점수의 표준 편차입니다.

## 계층화된 k-Fold 교차검증 (Stratified k-Fold)

불균형한 클래스를 가진 분류 문제에서, 계층화된 k-fold는 각 폴드가 원본 데이터셋과 대략 동일한 클래스 분포를 유지하도록 합니다. 이는 한 폴드가 실수로 한 클래스만 포함하는 상황을 방지하여 오도하는 성능 추정을 피합니다.

예: 데이터셋이 90% 클래스 A, 10% 클래스 B라면, 각 폴드도 대략 90%-10% 분포를 유지해야 합니다.

## 리브-원-아웃 교차검증 (Leave-One-Out Cross-Validation, LOOCV)

LOOCV는 극단적인 경우로 k = n (샘플 수)입니다. n-1 샘플로 훈련하고, 1 샘플로 검증한 후 n번 반복합니다. 장점:
- 반복마다 최대 훈련 데이터 사용
- 결정적 (폴드 선택에 무작위성 없음)
- 일반화 오류의 거의 불편향 추정치 제공

단점:
- 계산상 매우 비용이 많이 듦 (n개 모델 학습)
- 폴드가 크게 겹치므로 높은 분산
- 작은 데이터셋이나 단순 모델을 제외하고는 거의 실용적이지 않음

## 시계열 교차검증

표준 k-fold는 시계열에 부적절합니다. 시간 순서를 위반하기 때문입니다. 시계열 CV는 포워드 체이닝 접근을 사용합니다:

**Walk-forward 검증:**
- 폴드 1: [1-100] 훈련, [101-150] 테스트
- 폴드 2: [1-150] 훈련, [151-200] 테스트
- 폴드 3: [1-200] 훈련, [201-250] 테스트

훈련 세트는 항상 테스트 세트보다 시간상 앞에 있습니다. 이는 시계열 데이터의 인과 구조를 존중합니다.

## 모델 비교 전략

### 모델 간 비교
교차검증은 다른 모델 아키텍처의 공정한 비교를 허용합니다:
- 모델 A: 평균 CV 점수 = 0.855, 표준편차 = 0.021
- 모델 B: 평균 CV 점수 = 0.847, 표준편차 = 0.028

통계 검정(예: 폴드 점수에 대한 쌍 t-검정)을 사용하여 차이가 유의한지 결정합니다.

### 중첩 교차검증 (Nested Cross-Validation)
하이퍼파라미터 선택과 모델 비교를 동시에 수행할 때, 데이터 누수를 방지하기 위해 중첩 CV를 사용합니다:
- **외부 CV** (예: 5-fold): 모델 비교용
- **내부 CV** (예: 5-fold): 각 외부 폴드 내에서 하이퍼파라미터 튜닝용

외부 폴드는 하이퍼파라미터 최적화 중에 완전히 건드리지 않으므로 편향 없는 비교를 보장합니다.

## 정보 기준: AIC와 BIC

모수 모델에서 모델 선택을 위해, 정보 기준은 적합도와 복잡도를 균형 맞춥니다:

**아카이케 정보 기준 (Akaike Information Criterion, AIC):**
```
AIC = 2k - 2 ln(L)
```
여기서 k = 파라미터 수, L = 최대우도

**베이지안 정보 기준 (Bayesian Information Criterion, BIC):**
```
BIC = k ln(n) - 2 ln(L)
```
여기서 n = 관측치 수

낮은 값이 더 좋습니다. BIC는 AIC보다 복잡도를 더 강하게 처벌합니다 (특히 큰 n의 경우), 모델 복잡도에 더 보수적입니다.

AIC는 예측에 더 나으며, BIC는 "참" 모델을 찾는 데 더 나습니다. 목표가 예측일 때는 AIC를, 해석 가능한 희소 모델을 원할 때는 BIC를 사용합니다.

## 실무 권장사항

1. **작은 데이터셋 (n < 1000):** 계층화된 5-fold 또는 10-fold CV 사용
2. **중간 데이터셋 (1000 < n < 100,000):** 5-fold CV 사용
3. **큰 데이터셋 (n > 100,000):** 단일 훈련-검증-테스트 분할로 충분
4. **불균형한 분류:** 항상 계층화된 k-fold 사용
5. **시계열:** 항상 포워드 체이닝 또는 확장 윈도우 CV 사용
6. **하이퍼파라미터 튜닝:** 중첩 CV 또는 별도의 검증 세트 사용

## 일반적인 실수

- **데이터 누수:** 폴드 분할 전에 전처리(정규화, PCA)를 수행하면 검증 추정을 오염시킵니다
- **CV 점수 최적화:** CV 점수를 하이퍼파라미터로 사용하면 노이즈에 대해 선택되므로 최종 평가에는 별도의 테스트 세트를 사용합니다
- **너무 적은 폴드:** k < 3은 높은 분산 추정을 제공합니다
- **시간 순서 무시:** 시계열 데이터에 무작위 k-fold를 사용하면 과도하게 낙관적인 추정이 보장됩니다

교차검증은 선택사항이 아니며, 신뢰할 수 있는 모델 선택과 정직한 성능 보고에 필수적입니다.
