# 에이전트 메모리 시스템

## 소개

[AI 에이전트](/agent-kor.html)는 컨텍스트 유지, 과거 상호 작용 학습, 정보 기반 의사 결정을 위해 메모리가 필요합니다. 수년에 걸친 구현된 메모리를 가진 인간과 달리, AI [에이전트](/agent-kor.html)는 명시적 메모리 시스템에만 전적으로 의존합니다. 효과적인 에이전트 메모리는 단기 컨텍스트, 장기 지식, 삽화적 회상, 의미론적 이해를 포함합니다.

## 단기 메모리 (컨텍스트 윈도우)

### 정의

모델이 현재 처리하는 토큰 시퀀스:
- 일반적 [컨텍스트 윈도우](/context-window-kor.html): 4k-200k 토큰
- 최신 [LLM](/llm-kor.html) ([GPT](/gpt-kor.html)-4 Turbo, Claude 3): 100k-200k 토큰
- 더 큰 컨텍스트 = 접근 가능한 더 많은 대화 이력

### 도전

**토큰 계산**: 대화의 각 턴이 토큰 사용
- 일반적 턴: 500 토큰
- 긴 대화: 빠르게 컨텍스트 소진

**주의 복잡도**: 긴 시퀀스 처리는 O(n²) 복잡도
- 더 긴 시퀀스 = 느린 추론
- 메모리 대역폭 병목

### 최적화 전략

**요약화**: 주기적으로 오래된 컨[텍스트 요약](/text-summarization-kor.html)
```
오래된 컨텍스트: [1000개 토큰의 상세 대화]
요약: "사용자가 분기별 수익 목표 논의, Q3 초점 선호"
새 컨텍스트: [요약 + 최근 메시지]
```

**슬라이딩 윈도우**: 최근 N개 메시지만 유지
- 절충: 이전 컨텍스트 손실
- 대화형 [에이전트](/agent-kor.html)에 좋음

**계층적 압축**: 다단계 요약 (분 → 시간 → 일)
- 고수준 패턴 보존
- 장기 메모리 액세스 가능

## 장기 메모리 (벡터 저장소)

### 의미론적 메모리

사실 및 개념에 대한 구조화된 지식:
- 지식 그래프: "Python은 프로그래밍 언어"
- 임베딩: 조밀한 벡터 표현
- 의미론적 유사성 기반 검색

**저장소**: [벡터 데이터베이스](/vector-databases-kor.html) (Pinecone, Weaviate, Milvus)

**검색**: 
```
사용자 쿼리: "웹 개발 모범 사례"
임베딩: 조밀 벡터로 변환
검색: 유사한 임베디드 지식 찾기
반환: 상위 5개 관련 사실/문서
```

### 삽화적 메모리

특정 이벤트 및 상호 작용의 메모리:
- "사용자 X가 3월 15일에 분기 수익에 대해 물어봤음"
- 타임스탬프와 함께
- 개인화 및 일관성 활성화

**구현**:
- 상호 작용 필사본을 벡터 DB에 저장
- 상호 작용 임베딩, 과거 유사 상호 작용 검색
- 현재 상호 작용에 컨텍스트 제공

## 메모리 유형

### 작업 메모리

현재 문제 해결을 위한 임시 저장소:
```
작업: 고객 생명주기 가치 계산
작업 메모리:
  - 고객 ID: 12345
  - 구매 이력: [구매 목록]
  - 현재 시간: 2024년 3월
  - 공식: sum(purchases) * (1 + retention_rate)
```

- 활동 계산 중 사용
- 작업 완료 후 폐기

### 의미론적 메모리

지속적, 추상적 지식:
- "머신러닝은 학습 데이터 필요"
- "금리가 대출 승인에 영향"
- 도메인 특화 사실

**저장 전략**: 지식 그래프 + 임베딩

### 삽화적 메모리

특정 이벤트 및 컨텍스트:
- "3월 15일 사용자와의 대화"
- "모델이 테스트 실행에서 95% 정확도 달성"
- "시스템이 오후 2:30에 타임아웃 오류 발생"

**저장 전략**: 타임스탐프, 검색 가능한 데이터베이스

## 메모리 관리

### 저장 제한

메모리 시스템은 무한정 증가할 수 없습니다:

**하드 제한**: 데이터베이스 크기 제약
**실질 제한**: 비용, 검색 지연 시간

**전략**:
- 만료: 시간 T 후 오래된 메모리 폐기
- 압축: 오래된 메모리 요약
- 보관: 오래되면 느린 저장소로 이동
- 가지치기: 중복 메모리 제거

### 검색 효율

관련 메모리를 빠르게 찾기:

**벡터 유사성 검색**:
- 사용자 질문 → 임베딩 → 벡터 DB 검색 → 유사 메모리 검색
- 빠름: 일반적으로 10-100ms

**하이브리드 검색**:
- 벡터 유사성 + 키워드 검색 결합
- 더 나은 적용 범위

### 메모리 우선순위

일부 메모리가 다른 것보다 더 중요합니다:

**빈도 기반**: 최근/자주 액세스하는 메모리가 더 오래 유지

**중요도 기반**: 중요 메모리 (보안, 사용자 선호도) 항상 이용 가능

**관련성 기반**: 현재 작업과 관련된 메모리 우선

## 계층적 메모리 조직

### 메모리 수준

```
레벨 1: 컨텍스트 윈도우
  현재 대화 (최근 메시지)
  
레벨 2: 대화 요약
  이전 대화의 추상화된 요약
  
레벨 3: 사용자 프로파일
  장기 사용자 선호도, 이력
  
레벨 4: 도메인 지식
  일반 사실, 시스템 문서
```

**검색 전략**:
1. [컨텍스트 윈도우](/context-window-kor.html) 확인 (가장 빠름)
2. 찾지 못하면 대화 요약 쿼리
3. 찾지 못하면 사용자 프로파일 쿼리
4. 찾지 못하면 도메인 지식 쿼리 (가장 느림)

## 실질 메모리 구현

### 간단한 접근

최근 메시지 목록:
```python
memory = [
    {"role": "user", "content": "잔액은?"},
    {"role": "assistant", "content": "잔액은 $5000"},
    {"role": "user", "content": "거래 이력 표시"}
]
```

간단함, 짧은 대화에 작동합니다.

### 벡터 DB 접근

상호 작용 임베드 및 저장:
```python
# 저장
embedding = embed("사용자가 잔액에 대해 물어봤음")
vector_db.add(embedding, {"timestamp": now, "content": "..."})

# 검색
query_embedding = embed(user_query)
similar_memories = vector_db.search(query_embedding, top_k=5)
```

장기 메모리에 더 좋습니다.

### 하이브리드 접근

여러 메모리 유형 결합:
- [컨텍스트 윈도우](/context-window-kor.html)의 최근 컨텍스트
- 전통 DB의 요약
- 벡터 DB의 전체 상호 작용
- 지식 그래프의 의미론적 지식

## 메모리 증강 생성 (MAG)

### 프로세스

```
사용자 쿼리
    ↓
관련 메모리 검색
    ↓
메모리로 프롬프트 증강
    ↓
컨텍스트와 함께 응답 생성
```

예:
```
사용자: "지난해 총 지출은?"

검색된 메모리:
- 1월: $500
- 2월: $600
- ...
- 12월: $700

증강된 컨텍스트: "사용자의 월간 지출: [목록]. 
연간 총액을 계산합니다."

응답: 지난해 총 지출은 $7,200입니다
```

## 도전

**환각**: 모델이 메모리를 만들어낼 수 있음
**노후화**: 메모리가 시간이 지나면서 구식이 될 수 있음
**프라이버시**: 사용자 데이터 저장은 프라이버시 문제 야기
**일관성**: 다른 시간의 메모리 충돌
**검색 실패**: 관련 메모리가 검색되지 않음

## 모범 사례

- **분리성**: 중단 없이 메모리 추가/제거 용이
- **검색성**: 콘텐츠/시간/관련성별 효율적 검색
- **신뢰성**: 정확한 저장, 손상 없음
- **프라이버시**: 민감 메모리 암호화, 액세스 감사
- **만료**: 비용 관리를 위해 오래된 메모리 자동 삭제

## 미래 방향

- **지속적 학습**: 새로운 상호 작용에서 메모리 업데이트
- **망각**: 비관련 메모리 선택적 망각
- **메모리 통합**: 유사 메모리 병합
- **멀티모달 메모리**: 이미지, 오디오, 문서 저장
- **분산 메모리**: 여러 [에이전트](/agent-kor.html) 간 공유 메모리

## 결론

효과적인 메모리 시스템은 지능형 [에이전트](/agent-kor.html) 구축에 필수입니다. 최적의 접근은 사용 사례에 따라 달라집니다: 짧은 대화는 [컨텍스트 윈도우](/context-window-kor.html)만 필요합니다. 장기 에이전트는 벡터 DB와 지식 그래프가 필요합니다. 에이전트가 더 정교해지면서 일관성, 프라이버시, 성능 유지를 위해 메모리 관리가 점점 더 중요해집니다.
