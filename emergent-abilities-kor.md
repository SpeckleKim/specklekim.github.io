# LLM의 창발적 능력

[대규모 언어 모델](/llm-kor.html)의 가장 매혹적인 발견 중 하나는 특정 규모 임계값에서의 예상치 못한 능력의 창발입니다. 이 "창발적 능력"은 스케일링 중에 갑자기 나타납니다—더 작은 모델에서는 보이지 않지만 더 큰 모델에서는 견고하게 존재합니다. 이 현상은 언어 모델이 능력을 어떻게 획득하는지에 대한 우리의 이해에 도전하며 모델 개발에 관한 심오한 질문을 제기합니다.

## 창발적 능력 정의

창발적 능력은 형식적으로 다음과 같이 정의됩니다:

1. **더 작은 모델에서 부재**: 임계값 이하 모델에서 거의 무작위 수준의 성능
2. **더 큰 모델에서 존재**: 임계값 이상 모델에서 무작위 이상 상당한 성능
3. **급격한 전환**: 성능이 임계값에서 비선형적으로 점프
4. **예측되지 않음**: 능력이 명시적으로 학습되지 않음; 모델이 "알아냄"

주요 구분:
- **부드러운 스케일링**: 능력이 점진적으로 개선 ([스케일링 법칙](/scaling-laws-kor.html))
- **창발**: 능력이 갑자기 나타남 (상전이)

## 고전적 예시

### 사고의 연쇄 프롬프팅

가장 유명한 예는 사고의 연쇄 추론을 포함합니다:

```
프롬프트: "상점에 사과가 5개 있습니다. 3개는 빨강, 2개는 녹색입니다.
빨간 사과 2개를 선택하는 방법은 몇 가지입니까?"

더 작은 모델 (< 10B 파라미터):
- 무작위 추측
- 구조화된 추론 보이지 않음

더 큰 모델 (> 50B 파라미터):
- "먼저, 3개의 빨간 사과 중 2개를 선택해야 합니다"
- "그것은 C(3,2) = 3가지입니다"
- 올바른 추론 단계
```

Wei et al. (2022)은 사고의 연쇄 추론이 약 50-100B 매개변수에서 창발함을 보여줍니다:
- 10B 이하: 복잡한 추론에 약 5-10% 정확도
- 10-50B: 약 30-50% 정확도
- 100B 이상: 약 70-90% 정확도 (사고의 연쇄 프롬프팅)

### 산술 및 기호 추론

모델은 갑작스러운 능력을 입증합니다:
- **다중 자리 산술**: 큰 숫자에 대한 정확한 연산
- **알고리즘 작업**: 정렬, 검색, 패턴 완성
- **코드 생성**: 설명으로부터 컴파일 가능한 코드 작성

이 능력은 다양한 임계값에서 나타납니다:
- 덧셈: 약 2-5B 파라미터에서 창발
- 곱셈: 약 10-20B 파라미터에서 창발
- 일반 프로그래밍: 약 50B+ 파라미터에서 창발

### 맥락 내 학습

더욱 놀라운 것: 프롬프트의 예에서 새로운 작업을 학습하는 능력:

```
프롬프트의 소수 샷 예:
"fox -> fox_box, dog -> dog_box, cat -> ?"

더 작은 모델: 패턴을 식별할 수 없음
더 큰 모델 (10B+): 올바르게 "cat_box" 답변

더욱 미묘한: 모델이 명시적 지시 없이 작업 구조 학습
```

### 번역 및 다국어 작업

모델은 규모에서 갑자기 능숙한 번역가가 됩니다:

```
모델 크기 0.5B: 대부분 엉망인 번역
모델 크기 2B: 일부 올바른 단어, 깨진 문법
모델 크기 10B+: 먼 언어 간 유창하고 정확한 번역

이것은 명시적 병렬 말뭉치 학습 (모든 언어 쌍)없이 창발합니다
```

## 논쟁: 실제인가 아니면 가공물인가?

모두가 창발적 능력이 진정한 "창발"이라고 동의하지는 않습니다. 이는 AI 연구에서 열정적인 논쟁입니다.

### "실제 창발" 진영

옹호자들은 주장합니다:
1. **불연속적 능력 점프**: 급격한 전환이 경험적으로 관찰 가능
2. **설계자들이 예상하지 못함**: 능력이 명시적으로 설계되지 않음
3. **임계값 현상**: 능력에서 명확한 절단점
4. **다단계 추론**: 더 작은 모델에서 없는 합성적 이해 필요
5. **미세한 증거**: 상세 연구는 많은 도메인에서 창발을 보여줌

**주요 논문**: Wei et al. (2022), Suzgun et al. (2022)

### "가공물" 진영

회의론자들은 주장합니다:
1. **측정 가공물**: 창발는 부분적으로 능력을 측정하는 방식의 가공물
   - 정확도 메트릭은 전부 아니면 무 (맞음/틀림)
   - 기저의 능력은 부드럽게 스케일될 수 있지만, 측정은 이산적
   - 예: 통과가 70% 정확도를 필요로 하면, 69%→71% 개선은 갑작스러운 창발처럼 보임

2. **작업 정의**: 작업을 정의하는 방식이 창발이 나타나는지 영향
   - 동일한 기저 능력이 측정에 따라 "창발"할 수도, 아닐 수도 있음
   - 메트릭 전환 (혼란도 vs. 정확도)이 창발 관찰 변경

3. **벤치마크 포화**: 초기 [벤치마크](/benchmarks-kor.html)는 큰 모델에 너무 쉬움
   - 더 작은 모델은 천장에서 고원
   - 더 큰 모델은 더 미세한 성능 표시
   - 창발처럼 보이지만 가공물일 수 있음

**주요 논문**: Schaeffer et al. (2023) - "Are Emergent Abilities of Large Language Models a Mirage?"

### 현재 합의

연구는 **둘 다 참**임을 시사합니다:

1. **일부 창발은 실제**: 다단계 추론과 같은 능력은 아마도 규모 필요
2. **일부 창발은 측정 가공물**: 평가 메트릭은 점진적 개선 숨김
3. **미묘함이 중요**: 창발이 "실제"인지는 정의와 측정에 따름
4. **더 나은 메트릭 필요**: 점진적 스케일링을 임계값에서만 구분하는 미세한 평가 필요

대부분의 연구자는 이제 다음에 집중:
- 정확도 옆에 연속 메트릭 (손실, 혼란도) 사용
- 최종 성능이 아닌 학습 역학 연구
- 임계값이 아닌 점진적 스케일링을 드러내는 [벤치마크](/benchmarks-kor.html) 설계

## 창발 뒤의 메커니즘

여러 이론이 능력이 규모에서 창발하는 이유를 설명합니다:

### 이론 1: 그로킹
모델이 먼저 학습 데이터를 암기하고, 그 후 "그록"하여 추상 패턴:

```
단계 1: 학습 데이터 적합 (과적합)
단계 2: 유휴 기간 (일반화 학습)
단계 3: 테스트 세트에서 갑작스러운 개선 (창발적 능력)
```

이는 학습 손실이 0이어도 지연된 창발을 설명합니다.

### 이론 2: 합성적 추론
더 큰 모델은 더 단순한 학습 패턴을 결합할 수 있습니다:

```
학습된 패턴:
- "지시 따름"
- "수학 연산 수행"
- "단계 결합"

더 작은 모델: 연산을 신뢰성 있게 연쇄할 수 없음
더 큰 모델: 패턴 합성 → 다단계 추론
```

### 이론 3: 분산 표현
스케일은 더 풍부하고 더 구별되는 표현을 가능하게 합니다:

```
더 작은 모델: 특징이 얽혀있고, 분리하기 어려움
더 큰 모델: 개념의 명확한 분리
            → 정확한 추론 가능하게 함
```

### 이론 4: 상전이
물리적 유추: 물리의 상전이처럼:

```
얼음 ←→ 물 ←→ 증기 (온도 임계값)

모델 능력은 유사하게 X하는 능력이:
- 갑자기 나타날 수 있는 임계점을 가질 수 있음
- 매개변수 수와 상관관계
- 물리의 보편성 개념과 유사
```

## 모델 개발에 대한 함의

창발 이해는 실질적 결과를 가집니다:

### 1. 계산 할당
능력이 특정 임계값에서 창발하면:
- 사고의 연쇄가 50B가 필요하면 7B 모델 학습할 이유 없음
- 하지만 50B는 다른 낭비적 매개변수 가질 수 있음
- 능력 임계값을 향한 목표 개발 시사

### 2. 작업 설계
창발은 우리가 다음을 해야 함을 시사:
- [벤치마크](/benchmarks-kor.html)를 통과/불통 만이 아닌 점진적 스케일링을 드러내도록 설계
- 암기와 일반화 분리
- 여러 평가 메트릭 사용

### 3. 스케일링 전략
팀은 창발을 염두에 두고 스케일링을 계획할 수 있습니다:
- 필요한 능력의 임계값 식별
- 그 임계값을 향해 계산 예산
- 임계값 이상의 한계 이익에 덜 집중

### 4. 학습 역학
창발은 다음을 시사:
- 조기에 학습을 중단하지 마세요 (숨겨진 "그로킹" 진전)
- 모델 동작은 더 많은 규모로 갑자기 변경할 수 있음
- 창발적 능력은 예측 불가능 (연구 방향!)

## 창발적 능력 vs. 학습된 동작

중요한 구분:

**창발적 능력:**
- 학습 목표에 없음
- 명시적으로 프로그램됨
- 규모와 최적화에서 발생
- 예: 학습에서 산술 연산 없이 다중 자리 산술

**학습된 동작:**
- 데이터에서 직접 학습
- 작업 지정 따름
- 부드럽게 스케일
- 예: 레이블된 데이터에 대한 분류 정확도

대부분의 "창발적 능력"은 실제로 다음 때문에 갑작스럽게 스케일되는 능력입니다:
- 학습 신호가 간접적
- 드러나려면 상당한 용량 필요
- 측정이 연속 (손실)이 아닌 이산적 (통과/불통)

## 미결 질문

진전에도 불구하고, 우리는 창발을 완전히 이해하지 못합니다:

1. **예측 가능성**: 어떤 능력이 어떤 규모에서 창발할지 예측할 수 있을까?
2. **필요성**: 특정 능력이 큰 규모를 필요로 하는 기본적 이유가 있을까?
3. **일반성**: 창발 원칙이 도메인 전반에 전이될까?
4. **제어 가능성**: 원하는 능력의 창발을 의도적으로 유도할 수 있을까?
5. **효율성**: 거대한 스케일링 없이 창발된 능력을 달성할 수 있을까?

## 결론

LLM의 창발적 능력—사고의 연쇄 추론부터 맥락 내 학습, 번역까지—AI의 가장 매혹적인 현상 중 하나를 나타냅니다. 창발이 진정으로 불연속적인지 부분적으로 측정 가공물인지는 여전히 논쟁 중이지만, 현상 자체는 실제이고 중요합니다.

창발 이해는 모델 개발을 어떻게 생각하는지를 재구성합니다: 능력이 반드시 규모의 부드러운 함수가 아니며, 예상치 못한 능력이 갑자기 나타날 수 있고, 우리의 평가 방법이 우리가 관찰하는 것에 크게 영향을 미칩니다.

모델이 계속 스케일됨에 따라, 창발 연구는 능력 예측, 계산 할당 효율화, 궁극적으로 더 능숙하고 정렬되고 유용한 언어 모델 구축에 필수적으로 남을 것입니다.
