# 활성화 함수: 비선형성 도입

활성화 함수는 뉴런 출력에 적용되는 비선형 변환입니다. 활성화 함수가 없다면 계층을 쌓아도 선형 변환일 뿐이므로 [신경망](/neural-kor.html)은 선형 관계만 학습할 수 있습니다. 활성화 함수는 네트워크가 복잡한 비선형 패턴을 학습하도록 합니다.

## 비선형성이 중요한 이유

활성화 함수가 없는 [신경망](/neural-kor.html)은 단지 행렬 곱셈입니다. 여러 선형 변환을 합치면 결국 하나의 선형 변환이 되므로 깊은 네트워크도 얕은 네트워크와 동등합니다. 실제 문제의 복잡성을 표현할 수 없습니다.

활성화 함수는 비선형성을 도입하여 네트워크가 의사결정 경계를 구부리고 비틀 수 있게 해줍니다. 이를 통해 임의의 함수를 학습할 수 있습니다.

## 시그모이드 함수(Sigmoid)

시그모이드 함수는 입력을 (0, 1) 범위로 매핑합니다:

`σ(x) = 1 / (1 + e^(-x))`

**특징**: 부드러운 기울기, 출력을 확률로 해석 가능.

**단점**: 깊은 네트워크에서 [기울기 소실](/vanishing-gradients-kor.html) 문제. 출력이 0을 중심으로 하지 않아 학습이 느림. 지수 계산으로 비용이 많음.

**사용 사례**: 확률 해석이 필요한 이진 분류 출력층.

## 쌍곡탄젠트 함수(Tanh)

Tanh는 입력을 (-1, 1) 범위로 매핑합니다:

`tanh(x) = (e^x - e^(-x)) / (e^x + e^(-x))`

**특징**: 0을 중심의 출력(빠른 수렴), 시그모이드보다 강한 기울기.

**단점**: 매우 깊은 네트워크에서도 [기울기 소실](/vanishing-gradients-kor.html) 문제 발생 가능.

**사용 사례**: 은닉층, 특히 RNN과 이전 아키텍처.

## ReLU (정류 선형 단위)

ReLU는 가장 인기 있는 활성화 함수입니다:

`ReLU(x) = max(0, x)`

**특징**: 간단하고 빠른 계산. 양수 값에서 [기울기 소실](/vanishing-gradients-kor.html) 없음. 깊은 네트워크 학습 효과적.

**단점**: "죽은 ReLU" 문제—뉴런이 계속 0을 출력할 수 있음. x=0에서 미분 불가(실무상 무시).

**사용 사례**: 현대 깊은 네트워크의 은닉층 기본 선택.

## Leaky ReLU

Leaky ReLU는 죽은 ReLU 문제를 해결합니다:

`Leaky ReLU(x) = max(αx, x)` (여기서 α는 작은 값, 보통 0.01)

**특징**: 작은 음수 기울기 허용, 뉴런 고사 방지. ReLU보다 약간 더 많은 계산.

**단점**: 튜닝할 하이퍼파라미터 α. 많은 경우 실질적 개선 미미.

**사용 사례**: 죽은 ReLU 현상 관찰 시. ReLU의 안정적 대안.

## ELU (지수 선형 단위)

ELU는 음수 값으로 부드럽게 전환합니다:

`ELU(x) = x (x > 0일 때), 아니면 α(e^x - 1)`

**특징**: 음수 포화 특성(평균이 0에 더 가까움), ReLU보다 부드러움.

**단점**: 지수 계산으로 ReLU보다 느림. 한계적 개선.

**사용 사례**: 평균 0 활성화 원할 때. [합성곱 신경망](/cnn-deep-dive-kor.html).

## GELU (가우시안 오차 선형 단위)

GELU는 누적 가우시안 분포로 입력을 가중합니다:

`GELU(x) = x * Φ(x)` (Φ는 누적 정규분포)

**특징**: ReLU의 부드러운 근사. 최신 [트랜스포머](/llm-kor.html)와 대규모 모델에서 인기.

**단점**: 더 많은 계산 비용. 라이브러리 없이 구현이 복잡.

**사용 사례**: [트랜스포머](/llm-kor.html) 모델([BERT](/bert-kor.html), [GPT](/gpt-kor.html)), 최첨단 아키텍처.

## Swish 함수

Swish(스케일 시 SiLU라고도 함)는 시그모이드 게이팅을 사용합니다:

`Swish(x) = x * σ(x)`

**특징**: 자체 게이팅 메커니즘. 많은 경우 ReLU 능가. 부드럽고 단조성 없음.

**단점**: ReLU보다 더 많은 계산. 널리 채택되지 않음.

**사용 사례**: 성능 개선 추구하는 최신 아키텍처. EfficientNet 등.

## Mish 함수

Mish는 부드럽고 단조성이 없는 활성화 함수입니다:

`Mish(x) = x * tanh(softplus(x))` (softplus(x) = ln(1 + e^x))

**특징**: 자체 [정규화](/regularization-kor.html). 최신 아키텍처에서 좋은 성능. 전체적으로 부드러움.

**단점**: 계산 비용. GELU보다 덜 확립됨.

**사용 사례**: 최첨단 모델. 새로운 아키텍처 실험.

## Softmax 함수

Softmax는 벡터를 확률 분포로 변환합니다:

`softmax(x_i) = e^(x_i) / Σ(e^(x_j))`

**특징**: 출력의 합이 1, 확률로 해석 가능. 미분 가능.

**단점**: 은닉층에 부적합(출력층만). 수치적으로 불안정(로그합-지수 트릭 필요).

**사용 사례**: 다중 분류 출력층. 최종 계층에만 적용.

## 비교 및 실무 지침

**은닉층의 경우**:
- ReLU로 시작: 빠름, 효과적, 낮은 계산량
- ReLU 문제 발생 시 Leaky ReLU 시도
- [트랜스포머](/llm-kor.html)에서는 GELU 사용
- 성능 개선 원할 때 Swish 고려

**출력층의 경우**:
- 이진 분류: 시그모이드
- 다중 분류: Softmax
- 회귀: 선형

**현대 추세**: GELU가 [대규모 언어 모델과 트랜스포머](/llm-kor.html)의 표준. ReLU는 [합성곱 신경망](/cnn-deep-dive-kor.html)에서 여전히 표준. Swish와 Mish는 최첨단 연구에서 주목받음.

## 요약

올바른 활성화 함수 선택은 학습 속도, 수렴성, 최종 성능에 영향을 미칩니다. 현대 네트워크는 은닉층에 ReLU 변형을 주로 사용하고 출력층에 문제별 함수를 사용합니다. GELU는 [트랜스포머](/llm-kor.html) 기반 모델에서 표준이 되었으며, 딥러닝의 패러다임 전환을 반영합니다.
