# K-최근접 이웃

## 소개

K-최근접 이웃(KNN)은 분류와 회귀를 위한 가장 간단하면서도 효과적인 머신러닝 알고리즘 중 하나입니다. 기본 원리는 유사한 인스턴스는 유사한 레이블을 가진다는 것입니다. KNN은 비모수적이고 게으른 학습기로, 기본 데이터 분포에 대한 가정을 하지 않고 예측 시간까지 계산을 미룹니다.

## 알고리즘

### 기본 메커니즘

KNN은 전체 훈련 데이터를 저장하고 K개의 최근접 이웃을 기반으로 예측합니다:

1. **훈련 데이터 저장** (해당 레이블 포함)
2. **각 테스트 인스턴스에 대해:**
   - 모든 훈련 예제까지의 거리 계산
   - K개의 최근접 이웃 식별
   - 다수 클래스(분류) 또는 평균값(회귀)으로 분류

### 예측 규칙

**분류:**
```
예측된_클래스 = 가장_흔한_클래스(K_최근접_이웃)
```

K개 이웃 간의 투표가 예측을 결정합니다.

**회귀:**
```
예측된_값 = 평균(K_최근접_이웃_값)
```

K개 이웃 값의 평균이 예측을 제공합니다.

## 거리 메트릭

올바른 거리 메트릭을 선택하는 것은 KNN 성능에 중요합니다.

### 유클리드 거리

가장 일반적인 메트릭으로 직선 거리를 측정합니다:
```
d(x, y) = √(Σ(x_i - y_i)²)
```

연속 수치 피처에 잘 작동합니다. 척도 차이에 민감합니다.

### 맨해튼 거리(L1)

절대 차이의 합:
```
d(x, y) = Σ|x_i - y_i|
```

유클리드보다 이상치에 더 강건합니다. 고차원 데이터에 더 좋습니다.

### 민코프스키 거리

두 메트릭을 모두 포함하는 일반화:
```
d(x, y) = (Σ|x_i - y_i|^p)^(1/p)
```

p=2일 때 유클리드, p=1일 때 맨해튼이 됩니다.

### 해밍 거리

범주형/이진 데이터의 경우:
```
d(x, y) = 다른 위치의 개수
```

다른 속성의 개수를 측정합니다.

### 코사인 거리

텍스트와 고차원 희소 데이터:
```
d(x, y) = 1 - (x · y) / (||x|| × ||y||)
```

크기가 아닌 벡터 간의 각도를 측정합니다.

## K 선택하기

하이퍼파라미터 K는 성능에 크게 영향을 미칩니다:

**작은 K (예: K=1):**
- 세부 사항을 포착하며 노이즈를 밀접하게 따름
- 과적합 위험
- 높은 분산, 낮은 편향

**큰 K:**
- 더 부드러운 결정 경계
- 중요한 패턴을 과도하게 평활화할 수 있음
- 낮은 분산, 높은 편향
- 노이즈에 더 강건

**일반적인 관행:**
- 최적 K를 찾기 위한 [교차 검증](/cross-validation-kor.html)
- 종종 K = √(n) 또는 소규모 데이터셋의 경우 K = 3-5
- 이진 분류의 경우 홀수 K(동점 방지)

## 가중 KNN

표준 KNN은 모든 이웃을 동등하게 취급합니다. 가중 KNN은 더 가까운 이웃에 더 높은 중요도를 할당합니다:

**균등 가중치(표준):**
```
가중치_i = 1/K (모든 이웃)
```

**거리 기반 가중치:**
```
가중치_i = 1 / 거리_i
```

**역 제곱 거리:**
```
가중치_i = 1 / (거리_i²)
```

가중 변형은 이상치 영향을 줄이고 예측을 개선합니다.

## KD-트리와 공간 인덱싱

대규모 데이터셋의 경우 모든 훈련 포인트를 검색하는 것은 비효율적입니다. KD-트리는 k차원 공간에서 데이터를 계층적으로 구성합니다.

**장점:**
- 검색 시간을 O(n)에서 O(log n)으로 감소
- 축 정렬 분할을 사용하여 공간을 재귀적으로 분할
- 효율적인 범위 쿼리 활성화

**Ball 트리**와 **LSH(Locality-Sensitive Hashing)**는 매우 고차원에서의 대안입니다.

## 차원의 저주

KNN은 고차원에서 심각하게 영향을 받습니다:

**문제:**
- 차원이 증가하면 거리가 의미 없어짐
- 모든 포인트가 대략 등거리에 위치
- 고차원 부피는 모서리에 집중

**예:**
- 2D에서 거리 1 내 이웃은 원을 형성
- 100D에서 동일 거리 구는 대부분의 공간을 놓침

**해결책:**
- [차원 축소](/dimensionality-reduction-kor.html)(PCA, 피처 선택)
- 거리 메트릭 학습
- 대신 트리 기반 방법 사용
- 신중한 [피처 엔지니어링](/feature-engineering-kor.html)

## 응용

**분류:**
- 추천 시스템(항목 유사도)
- 의료 진단
- 이미지 분류
- 필기 인식

**회귀:**
- 유사 예제의 가격 예측
- 주식 시장 분석
- 함수 근사

**혼합:**
- [이상 탐지](/anomaly-detection-kor.html)(이웃이 멀리 떨어져 있음)
- 불균형 분류(가중 투표)

## 장점

- **간단하고 직관적**: 이해하고 구현하기 쉬움
- **훈련 불필요**: 게으른 학습이 훈련 시간 감소
- **다목적**: 분류, 회귀, 순위 매김에 작동
- **비모수적**: 데이터 분포에 가정 없음
- **효과적인 기준선**: 종종 놀랍도록 좋은 결과 제공
- **적응적**: 자연스럽게 국소 데이터 구조에 적응

## 단점

- **계산 비용**: n개 훈련 샘플에 대해 O(n) 예측 시간
- **저장소**: 전체 데이터셋 저장 필요
- **피처 스케일링**: 피처 크기 차이에 민감
- **차원의 저주**: 고차원에서 성능 저하
- **무관한 피처**: 모든 피처를 동등하게 취급
- **불균형 데이터**: 다수 클래스가 투표를 지배할 수 있음
- **해석**: 관계를 설명하는 모델이 없음

## 실무 고려사항

**전처리:**
- 동일한 척도로 피처 [정규화](/regularization-kor.html)
- 무관한 피처 제거
- 결측값 처리

**파라미터 튜닝:**
- 최적 K에 대한 [교차 검증](/cross-validation-kor.html)
- 다른 거리 메트릭 시도
- 불균형 데이터에 대해 가중 변형 고려

**최적화:**
- 중간 차원에서는 KD-트리 사용
- 고차원의 경우 근사 방법(LSH)
- KNN 적용 전 [차원 축소](/dimensionality-reduction-kor.html)

**사용 시기:**
- 소규모-중규모 데이터셋
- 비선형 결정 경계
- 비교를 위한 기준선
- 예제를 통한 해석 가능성이 가치있을 때

## 요약

K-최근접 이웃은 게으른 학습을 예시하는 기초 알고리즘입니다. 단순성, 효과성, 다목적성이 실무자에게 귀중합니다. 그러나 계산 비용과 차원의 저주가 더 작은 데이터셋과 저차원 문제로의 응용을 제한합니다. KNN의 강점과 약점을 이해하는 것은 적절한 알고리즘과 전처리 기법을 선택하는 데 필수적입니다.
