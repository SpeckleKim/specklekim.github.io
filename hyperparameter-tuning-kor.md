# 하이퍼파라미터 튜닝

하이퍼파라미터 튜닝은 모델 훈련의 최적 구성을 찾는 과정입니다. 데이터에서 학습된 모델 파라미터(가중치, 편향)와 달리, 하이퍼파라미터는 학습 과정 자체를 제어하며 훈련 전에 설정해야 합니다.

## 하이퍼파라미터 vs 파라미터

**파라미터**는 훈련 중에 데이터에서 학습됩니다:
- [신경망](/neural-kor.html) 가중치와 편향
- 결정 트리 분할 값
- 선형 회귀 계수

**하이퍼파라미터**는 훈련 전에 설정되고 학습 방식을 제어합니다:
- [학습률](/learning-rate-scheduling-kor.html), 배치 크기, 에포크 수
- [정규화](/regularization-kor.html) 강도 (λ)
- 모델 아키텍처 (깊이, 너비)
- [드롭아웃](/dropout-kor.html) 비율, 조기 종료 인내심
- 트리 깊이, [앙상블](/ensemble-methods-kor.html)의 트리 수

하이퍼파라미터는 튜닝하는 것이고, 파라미터는 자동으로 최적화됩니다.

## 그리드 서치

그리드 서치는 이산 집합에서 하이퍼파라미터의 모든 조합을 철저히 평가합니다.

**알고리즘:**
1. 하이퍼파라미터 값의 그리드 정의
2. 각 조합에 대해 모델 훈련
3. [교차검증](/cross-validation-kor.html)을 사용한 평가
4. 최고 CV 점수의 조합 선택

**예:**
```
학습률 ∈ {0.001, 0.01, 0.1}
배치_크기 ∈ {16, 32, 64}
드롭아웃 ∈ {0.2, 0.5}

전체 조합: 3 × 3 × 2 = 18개 모델
```

**장점:**
- 철저함: 최고 조합 찾기 보장
- 구현과 이해가 간단
- 매우 병렬화 가능 (독립적으로 조합 평가)
- 작은 검색 공간에서 잘 작동

**단점:**
- 하이퍼파라미터 수에 따라 지수적 증가
- 나쁜 영역에서의 낭비되는 계산
- 이산값만 가정 (연속 범위 아님)
- 예: 5개 하이퍼파라미터 × 각 10개 값 = 100,000개 모델

그리드 서치는 4개 이상의 하이퍼파라미터로 비실용적입니다.

## 무작위 서치

무작위 서치는 검색 공간에서 하이퍼파라미터 조합을 무작위로 샘플링합니다.

**알고리즘:**
1. 각 하이퍼파라미터에 대한 범위 정의
2. 무작위 조합 샘플링 (종종 더 효율적으로 분포)
3. 각 샘플에 대해 모델 훈련
4. [교차검증](/cross-validation-kor.html)을 사용한 평가
5. 최고 CV 점수의 샘플 선택

**반복 수:** 같은 하이퍼파라미터 수로 그리드 서치보다 훨씬 적음

**예:**
```
학습률 ~ Uniform(0.0001, 0.1)
배치_크기 ~ Discrete Uniform({16, 32, 64, 128})
드롭아웃 ~ Uniform(0.1, 0.7)

1000+ 대신 50개 무작위 조합 샘플
```

**장점:**
- 높은 차원에서 더 효율적
- 그리드 포인트에 편향되지 않음
- 연속 하이퍼파라미터 범위 처리 가능
- 종종 그리드 서치 비용의 10-20%로 경쟁 솔루션 찾음

**단점:**
- 절대 최고를 찾을 보장 없음
- 그리드 서치보다 체계적이지 않음
- 여전히 나쁜 영역에서 낭비

**모범 사례:** 무작위 서치는 고차원 문제에서 그리드 서치를 능가합니다.

## 베이지안 최적화

베이지안 최적화는 확률 모델을 사용하여 하이퍼파라미터 공간을 지능적으로 샘플링합니다.

**핵심 아이디어:** 대리 모델(보통 가우시안 프로세스)을 구축하여 하이퍼파라미터에서 모델 성능을 예측한 후, 이 모델을 사용하여 다음 시도할 하이퍼파라미터를 결정합니다.

**알고리즘:**
1. 초기 무작위 하이퍼파라미터 샘플 (보통 5-10개)
2. 모델 훈련 및 성능 기록
3. 확률론적 대리 모델 적합
4. 수집 함수(예: 예상 개선) 사용하여 다음 유망 하이퍼파라미터 선택
5. 선택된 하이퍼파라미터로 모델 훈련
6. 새로운 관측으로 대리 모델 업데이트
7. 단계 4-6 반복

**수집 함수:**
- **예상 개선 (EI):** 최고 근처 시도(착취)와 불확실 영역 시도(탐색) 균형
- **상한 신뢰도 (UCB):** 불확실 영역 선호 낙관적 추정
- **개선 확률 (PoI):** 가능한 개선 선호 탐욕적 접근

**장점:**
- 그리드/무작위 서치보다 훨씬 효율적
- 과거 평가를 활용하여 미래 선택 정보화
- 비싼 목적지에서 작동 (예: 훈련이 몇 시간)
- 혼합 파라미터 유형 처리 (연속, 이산, 범주)
- 종종 무작위 서치보다 나은 솔루션 찾음

**단점:**
- 구현이 더 복잡
- 대리 모델 적합이 오버헤드 추가
- 모델 구축을 위해 관측 필요
- 그리드/무작위 서치보다 병렬화 어려움

**일반적 도구:** 가우시안 프로세스 (GP), 트리 구조화 파젠 추정기 (TPE)

## Hyperband와 BOHB

Hyperband는 조기 종료와 자원 할당을 사용하여 하이퍼파라미터 검색을 가속화합니다.

**Hyperband 알고리즘:**
1. 많은 무작위 하이퍼파라미터 구성에 동등 자원 할당
2. 최소 자원으로 모든 구성 평가
3. 최악 성능 반 폐기
4. 생존자에게 더 많은 자원 할당
5. 단일 구성이 남을 때까지 단계 3-4 반복

**자원:** 훈련 에포크, 데이터 크기, 또는 측정 가능한 양

**81개 구성 예:**
- 라운드 1: 81개 구성을 각 1 에포크로 훈련
- 라운드 2: 상위 27개 유지, 각 3 에포크로 훈련
- 라운드 3: 상위 9개 유지, 각 9 에포크로 훈련
- 라운드 4: 상위 3개 유지, 각 27 에포크로 훈련
- 라운드 5: 상위 1개 유지, 81 에포크로 훈련

**BOHB (베이지안 최적화 + Hyperband):**
- 베이지안 최적화와 Hyperband 결합
- TPE를 사용하여 어떤 구성을 승격할지 선택
- 비싼 하이퍼파라미터 검색에 매우 효율적

**장점:**
- Hyperband 단독보다 훨씬 빠름
- 계산의 지능형 할당
- 많은 평가를 병렬로 실행 가능
- 최소 수동 튜닝 필요

## Optuna

Optuna는 사용 용이성을 강조하는 현대식 하이퍼파라미터 최적화 프레임워크입니다.

**특징:**
- 정교한 베이지안 최적화 (TPE, CMA-ES)
- 검색 공간을 Python으로 자연스럽게 정의
- 자동 가지치기 (유망하지 않은 시도의 조기 종료)
- 병렬 및 분산 최적화
- 시각화 및 분석 도구
- 시도 히스토리 저장

**예:**
```python
import optuna

def objective(trial):
    lr = trial.suggest_float('lr', 1e-5, 1e-1, log=True)
    batch_size = trial.suggest_int('batch_size', 16, 128)
    dropout = trial.suggest_float('dropout', 0.1, 0.7)

    # 모델 훈련 및 검증 점수 반환
    score = train_and_evaluate(lr, batch_size, dropout)
    return score

study = optuna.create_study(direction='maximize')
study.optimize(objective, n_trials=100)
```

**장점:**
- 직관적 Python API
- 자동 가지치기는 계산 절약
- 반복적 개발에 탁월
- 내장 시각화
- 병렬화 용이

## Ray Tune

Ray Tune은 분산 컴퓨팅을 위해 설계된 확장 가능한 하이퍼파라미터 튜닝 프레임워크입니다.

**특징:**
- 클러스터 전체에 [분산 훈련](/distributed-training-kor.html)
- 딥러닝 프레임워크 통합
- 여러 튜닝 알고리즘 (베이지안, PBT, 모집단 기반 훈련)
- 체크포인팅 및 오류 내성
- 모집단 기반 훈련 (PBT): 훈련 중 하이퍼파라미터 진화

**최적:**
- 대규모 분산 실험
- 매우 비싼 훈련 실행
- 복잡한 실험 워크플로우
- 분산 클러스터를 사용하는 팀

## 실무 튜닝 전략

### 1. 단순하게 시작
- 처음에 1-2개의 가장 중요한 하이퍼파라미터만 튜닝
- 불완전한 그리드 서치 사용 (파라미터당 몇 개 값)
- 합리적인 범위 식별

### 2. 거친에서 미세로
- 첫 번째 통과: 넓은 범위의 무작위 서치
- 유망한 영역 식별
- 두 번째 통과: 최고 주변 미세한 그리드

### 3. 중요도 순위
- 민감도별로 하이퍼파라미터 우선순위 지정
- [학습률](/learning-rate-scheduling-kor.html)이 종종 가장 중요
- 배치 크기, [정규화](/regularization-kor.html) 강도 다음
- 모델 아키텍처 세부사항 마지막

### 4. 교차검증은 필수
- 테스트 세트에서 절대 튜닝 금지 (하이퍼파라미터 공간에서 과적합 야기)
- 정직한 평가를 위해 중첩 CV 사용
- 최종 평가 전용으로 테스트 세트 예약

### 5. 병렬 검색
- 여러 구성을 동시에 평가
- 그리드 및 무작위 서치는 완벽하게 병렬화
- 베이지안 최적화는 부분적으로 병렬화
- 모든 사용 가능한 계산 자원 활용

### 6. 조기 종료
- 훈련 중 검증 성능 모니터링
- 유망하지 않은 구성 조기 중지
- Hyperband와 Optuna는 자동 가지치기 포함
- 나쁜 구성에서 대규모 계산 절약

## 일반적인 하이퍼파라미터 범위

**신경망:**
- [학습률](/learning-rate-scheduling-kor.html): 1e-5 ~ 1e-1 (로그 스케일)
- 배치 크기: 16 ~ 256
- [드롭아웃](/dropout-kor.html): 0.2 ~ 0.5
- L2 [정규화](/regularization-kor.html): 1e-6 ~ 1e-3

**결정 트리:**
- 최대 깊이: 3 ~ 20
- 리프당 최소 샘플: 1 ~ 20
- 기준: gini 또는 entropy

**그래디언트 부스팅:**
- [학습률](/learning-rate-scheduling-kor.html): 0.001 ~ 0.3
- 추정자 수: 100 ~ 1000
- 최대 깊이: 3 ~ 10
- 부분표본: 0.5 ~ 1.0

## 하이퍼파라미터 튜닝 체크리스트

1. 명확한 목표 메트릭 정의 (정확도, AUC, MSE)
2. 튜닝 전 데이터를 훈련/검증/테스트로 분할
3. 검색 전략 선택 (고차원에서 무작위 > 그리드)
4. 도메인 지식을 기반으로 합리적인 범위 설정
5. 정직한 평가를 위해 [교차검증](/cross-validation-kor.html) 사용
6. 가능할 때 병렬화
7. 진행 및 수렴 모니터링
8. 최고 하이퍼파라미터 문서화
9. 테스트 세트 성능 보고 (튜닝에 절대 사용 금지)
10. 계산 비용 vs 개선 고려

하이퍼파라미터 튜닝은 필수이지만 모든 계산 예산을 소비하지 않아야 합니다. 데이터 품질, 특성 엔지니어링, 튜닝에 적절히 자원을 할당합니다.
