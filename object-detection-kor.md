# 객체 탐지: 객체의 위치 파악 및 분류

## 소개

객체 탐지는 이미지 분류를 확장하여 여러 객체와 그 위치를 식별합니다. 최신 접근법은 두 가지로 나뉩니다: 먼저 영역을 제안한 후 분류하는 2단계 탐지기와 경계 상자를 직접 예측하는 1단계 탐지기입니다. 각 전략은 정확도와 속도 사이의 다양한 트레이드오프를 제공합니다.

## 2단계 탐지기

2단계 탐지기는 탐지를 영역 제안과 분류 단계로 분리하며 일반적으로 더 높은 정확도를 달성합니다.

**R-CNN 계열 진화:**
- **R-CNN (2014)**: 선택적 탐색을 사용한 영역 제안, 각각을 CNN에 입력하여 특징 추출, SVM으로 분류
- **Fast R-CNN (2015)**: RoI 풀링 계층으로 제안 간 계산 공유 활성화
- **Faster R-CNN (2015)**: 선택적 탐색을 영역 제안 네트워크(RPN)로 대체, 속도 대폭 향상
- **Cascade R-CNN (2017)**: 여러 품질 임계값에 걸친 탐지의 단계적 정제
- **Mask R-CNN (2017)**: Faster R-CNN을 인스턴스 분할 브랜치로 확장

## 1단계 탐지기

1단계 탐지기는 특징맵에서 경계 상자를 직접 예측하여 속도를 위해 정확도를 일부 희생합니다.

**YOLO 시리즈:**
- **YOLOv1 (2016)**: 탐지를 공간 격자의 회귀 문제로 처리하는 선도적 접근
- **YOLOv2 (2016)**: 배치 정규화, 다중 규모 예측, 앵커 상자 도입
- **YOLOv3-v8 (2018-2023)**: 더 나은 백본과 학습 전략으로 점진적 개선
- **YOLOv9**: 고급 스케일링 및 효율성 혁신

**SSD (Single Shot MultiBox Detector, 2015):** 다양한 규모의 객체 탐지를 위해 다중 규모 특징맵을 사용합니다.

## 앵커 기반 vs 앵커 프리

**앵커 기반:** 각 위치의 사전 정의된 경계 상자 템플릿. 신중한 앵커 설계가 필요하지만 유용한 사전 정보를 제공합니다.

**앵커 프리:** FCOS, CenterNet, 키포인트 기반 접근법은 객체 중심과 치수를 직접 예측하여 설계 선택과 하이퍼매개변수를 줄입니다.

## DETR과 비전 트랜스포머

**DETR (2020):** 탐지 트랜스포머는 탐지를 집합 예측 문제로 처리합니다. 객체용 학습 가능한 쿼리를 사용하는 인코더-디코더 아키텍처입니다. NMS 같은 수작업 요소를 제거하여 종단간 학습을 가능하게 합니다.

**장점:** 개념적 명확성, NMS 회피 집합 기반 손실
**단점:** 수렴 속도 느림, CNN 기반 탐지기보다 더 많은 데이터 필요

## RT-DETR

실시간 탐지에 최적화된 효율적 DETR 변형. 트랜스포머의 강점을 CNN 효율성과 결합하여 실무 배포를 위해 경쟁력 있는 속도-정확도 트레이드오프를 달성합니다.

## 평가 지표

**mAP (평균 정밀도):**
- 각 클래스별 정밀도(AP) 계산
- IoU (교집합 대 합집합) 임계값이 긍정 예측 결정 (일반적으로 0.5)
- mAP@0.5는 단일 임계값 사용; mAP@[0.5:0.95]는 IoU 임계값 전체 평균
- 탐지기 성능 비교의 표준 지표

**다른 지표:**
- **정밀도/재현율 곡선:** 거짓 양성과 음성 간 트레이드오프
- **FPS (초당 프레임 수):** 속도 측정
- **레이턴시:** 실시간 응용에 중요

## 현대 동향

최신 탐지기는 점점 더 트랜스포머 아키텍처, CNN과 주의 메커니즘을 결합한 하이브리드 접근, 배포용 효율적 설계를 채택합니다. 최근 작업은 다음에 초점을 맞춥니다:

- 몇 샷 및 제로 샷 탐지
- 교차 도메인 일반화
- 엣지 장치용 효율적 탐지기
- 자기 지도 사전 학습

## 실무 고려사항

탐지기 선택 시: 목표 정확도(mAP), 속도 요구사항(FPS/레이턴시), 계산 리소스, 배포 플랫폼을 고려하세요. 2단계 탐지기는 정확도에서 탁월하고 1단계는 속도를 지배합니다. 트랜스포머는 개념적 명확성을 제공하되 계산 비용이 있습니다.

## 결론

객체 탐지는 여전히 컴퓨터 비전의 초석입니다. 아키텍처 선택(2단계 vs 1단계, 앵커 기반 vs 앵커 프리, CNN vs 트랜스포머)을 이해하면 특정 응용에 대한 정보에 기반한 결정을 내릴 수 있습니다. 이 분야는 계속 더 효율적이고 일반화 가능한 접근법으로 진화하고 있습니다.
