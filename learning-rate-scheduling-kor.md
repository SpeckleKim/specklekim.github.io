# 학습률 스케줄링: 동적 최적화 전략

학습률 스케줄링은 훈련 중 학습률을 조정하는 실무입니다. 고정 학습률은 종종 타협안입니다: 초기 진행을 위해 충분히 높으면서도 발산을 피하기 위해 충분히 낮아야 합니다. 동적 스케줄링은 훈련의 다양한 단계에 학습률을 적응시켜 수렴 속도와 최종 모델 성능을 획기적으로 개선합니다.

## 고정 학습률의 문제

훈련 전체에서 단일 고정 학습률을 사용하는 것은 내재적 한계가 있습니다:

```
높은 학습률: 빠른 초기 진행, 하지만 최솟값 초과
낮은 학습률: 정확한 최적화, 하지만 느린 초기 수렴
타협: 고정 비율은 초기나 후기 훈련 모두에 최적이 아님
```

초기 훈련은 공격적 학습(큰 단계)이 이롭고, 미세 조정에는 신중한 작은 단계가 필요합니다. 단일 고정 비율은 두 단계를 잘 처리할 수 없습니다.

## 단계 감소

가장 간단하고 직관적인 스케줄링 접근은 고정 간격에서 학습률을 감소합니다:

```
매 N 에포크마다 학습률에 감소_계수를 곱함 (예: 0.1)
lr(epoch) = lr_0 * decay_factor^(epoch // N)

예: 0.1에서 시작, 30 에포크 후 0.01, 60 에포크 후 0.001
```

특성:
- 구현과 이해가 간단
- 많은 작업에서 실무상 잘 작동
- 하이퍼매개변수: 초기 lr, 감소 계수, 스케줄 간격
- 이산적 점프는 간단한 불안정성 야기 가능

**일반적 스케줄**:
- 매 30 에포크마다 0.1로 감소 (ResNets)
- 매 10 에포크마다 0.5로 감소 (RNN)
- 작업 의존적이고 그리드 검색으로 발견 많음

## 지수 감소

연속적 지수 감소는 점진적으로 학습률을 감소합니다:

```
lr(t) = lr_0 * exp(-λt)  λ는 감소 비율
또는 이산적으로:
lr(epoch) = lr_0 * decay_rate^epoch
```

장점:
- 부드러운 연속 감소
- 학습률 급격한 점프 없음
- 감소 비율 매개변수를 통한 유연한 제어
- 불연속 없는 부드러운 기울기 흐름

단점:
- 하이퍼매개변수 선택(감소 비율) 조정 필요
- 스케줄 길이에 따라 너무 빠르거나 느리게 감소 가능

## 코사인 어닐링

통계역학의 어닐링에서 영감을 받은 코사인 어닐링은 학습률을 부드럽게 감소합니다:

```
lr(t) = lr_min + (lr_max - lr_min) * (1 + cos(πt/T)) / 2
t는 현재 반복, T는 총 훈련 반복
```

주요 성질:
- lr_max에서 시작하여 lr_min으로 감소
- 부드러운 코사인 함수 (급격한 변화 없음)
- T 반복에서 완전한 감소 사이클 완성
- 수학적으로 우아하고 잘 동기 부여됨

**장점**:
- 점프 없는 부드러운 전환
- 훈련 끝까지 거의 0에 가까운 학습률 도달
- 경험적으로 우수한 성능
- 총 훈련 반복 수 지식 필요

## 워밍업 전략

많은 현대 훈련 절차는 학습률 워밍업을 사용합니다—훈련 초기에 점진적으로 학습률 증가:

```
처음 N_warmup 단계: 0에서 lr_max까지 선형 증가
남은 단계: 감소 (코사인, 단계, 지수)

lr(t) = (t / N_warmup) * lr_max,  t < N_warmup인 경우
```

워밍업이 도움이 되는 이유:
- 네트워크가 초기에 작은 단계로 가중치 조정 허용
- 좋지 않은 초기화에서 발산 방지
- 특히 [배치 정규화](/batch-normalization-kor.html)와 함께 훈련 안정화
- [트랜스포머](/llm-kor.html)에서 특히 중요
- [배치 정규화](/batch-normalization-kor.html)를 포함한 깊은 학습에서 일반적

**워밍업 기간**: 보통 총 훈련 단계의 5-10%

## OneCycleLR

주기적 학습률 기법에서 영감을 받은 단일 사이클에서 증가와 감소 단계를 결합합니다:

```
전반부: lr_min에서 lr_max로 증가 (선형 또는 지수)
후반부: lr_max에서 lr_min으로 감소
총: 전체 훈련 중 1 사이클
```

구현 세부사항:
- 모멘텀은 학습률과 역으로 변함
- 모멘텀 기반 최적화기(모멘텀이 있는 [SGD](/optimizers-kor.html), [Adam](/optimizers-kor.html) 변형) 작동
- 비균등 사이클에 분수 매개변수 사용 가능 (예: 30% 증가, 70% 감소)

장점:
- 우수한 경험적 결과
- 간단한 이단계 접근법
- 학습률 변동으로부터의 [정규화](/regularization-kor.html) 효과
- 광범위한 하이퍼매개변수 조정 필요 감소

## 주기적 학습률

단일 사이클을 넘어 주기적 학습률 진동:

```
lr은 주기적으로 lr_min과 lr_max 사이 진동
삼각형(선형 증가/감소), 지수, 또는 다른 파형 사용 가능
주기는 고정 또는 적응 가능
```

직관:
- 날카로운 최솟값 탈출 도움
- 속도 진동을 통한 [앙상블](/ensemble-methods-kor.html) 유사 행동 시뮬레이션
- 과적합 감소 가능
- 일반화 개선 가능

변형:
- 삼각형: 선형 증가/감소
- 삼각형2: 반복으로 진폭 감소
- 지수: 부드러운 지수 증가/감소

## 적응적 학습률

일부 스케줄러는 훈련 진행에 따라 적응합니다:

**검증 기반**: 검증 손실 모니터링, 정체 시 감소
**손실 기반**: 훈련 손실이 개선 중단 시 감소
**기울기 기반**: 기울기 통계에 따라 적응

이들은 훈련 중 모니터링 필요 (시간 기반 스케줄이 아님).

## 웜 재시작

학습률 재시작은 주기적으로 국소 최솟값 탈출 허용:

```
여러 재시작 사이클, 각각 코사인 어닐링
lr(t) = lr_min + (lr_max - lr_min) * (1 + cos(π(t mod T_cur)/T_cur)) / 2
T_cur은 각 재시작으로 증가
```

**장점**:
- 각 재시작은 높은 학습률의 "신선한 시작"
- 손실 경관의 다양한 영역 탐색
- 최종 모델 품질 개선 가능
- 탐색(높은 lr)과 활용(낮은 lr) 결합

## LR 찾기

훈련 전 좋은 학습률 범위를 찾는 실무 기법:

1. 작은 학습률로 시작
2. 훈련 중 점진적으로 학습률 증가 (1-2 에포크)
3. 각 학습률에서 손실 기록
4. 손실 대 학습률 그리기
5. 손실이 가장 빠르게 감소하는 곳에서 학습률 선택

```python
# 의사코드
best_loss = infinity
for lr in log_range(1e-5, 1):
    train_one_batch()
    if loss > 4 * best_loss:
        break  # 손실 폭발 시 중단
    record(lr, loss)
plot(lrs, losses)
# 최솟값이 아닌 가파른 감소 영역에서 lr 선택
```

**장점**:
- 학습률 선택에 대한 원칙적 접근
- 종종 수동 조정을 이김
- 빠르게 실행 (1-2 에포크)
- 정확한 값이 아닌 범위 발견

**한계**:
- 단일 사이클을 사용할 때 가장 잘 작동
- 학습률 스케줄에 최적이 아닐 수 있음
- 그래프 해석 필요

## 실무 지침

학습률 스케줄을 선택할 때:

1. **표준 감독 학습**: 단계 감소 또는 코사인 어닐링으로 시작
2. **트랜스포머**: 워밍업 (보통 10k 단계) + 코사인 어닐링
3. **CNN**: 단계 감소는 마일스톤 에포크와 잘 작동
4. **RNN**: 지수 감소 또는 단계 감소, 종종 더 작은 시작 비율
5. **미세 조정**: 더 낮은 학습률 및 잠재적 다른 스케줄
6. **불확실할 때**: 먼저 LR 찾기를 사용하여 좋은 기초선 설정

## 스케줄 비교

| 스케줄 | 부드러움 | 성능 | 복잡성 | 최적 |
|-------|--------|------|-------|------|
| 고정 | - | 기초선 | 없음 | 간단한 경우 |
| 단계 감소 | 이산 | 좋음 | 낮음 | [CNN](/cnn-deep-dive-kor.html), 표준 작업 |
| 지수 | 부드러움 | 좋음 | 중간 | 긴 훈련 |
| 코사인 | 부드러움 | 우수 | 중간 | 현대 딥러닝 |
| 워밍업+코사인 | 부드러움 | 우수 | 중간 | [트랜스포머](/llm-kor.html), 대형 모델 |
| OneCycleLR | 부드러움 | 우수 | 낮음 | 빠른 훈련, 작은 데이터 |
| 주기적 | 주기 | 좋음 | 중간 | 탐색 필요 |
| 웜 재시작 | 주기 | 좋음 | 중간 | 긴 훈련, 다중 목표 |

## 구현 고려사항

- **배치 크기 상호작용**: 더 큰 배치는 종종 더 큰 학습률 필요
- **모델 아키텍처**: 더 깊은 모델은 더 신중한 스케줄 필요 가능
- **최적화기 선택**: 다양한 최적화기([SGD](/optimizers-kor.html), [Adam](/optimizers-kor.html))는 다양한 스케줄 필요 가능
- **데이터 증강**: 더 강한 증강은 더 높은 학습률 허용 가능
- **수렴 모니터링**: 훈련과 검증 지표 모두 추적

## 일반적 실수

- 너무 높은 학습률로 시작 (발산)
- 배치 크기 변경 시 스케줄 조정 미실행
- 한 최적화기용으로 설계된 스케줄을 다른 최적화기와 사용
- 복잡한 아키텍처에 대해 워밍업 고려 안 함
- 스케줄이 수렴이 아닌 최종 모델 품질을 영향 잊음
- 학습률을 너무 낮게 설정 (훈련 시간 활용 부족)

## 결론

학습률 스케줄링은 훈련을 타협(단일 비율)에서 최적화된 프로세스로 변환합니다. 워밍업을 포함한 코사인 어닐링 같은 현대 접근법은 최고 수준의 딥러닝에서 표준이 되었습니다. 핵심 통찰력은 학습률을 훈련 단계와 맞추는 것입니다: 초기의 큰 탐색적 단계, 후기의 세밀한 작은 단계.

고전적 단계 감소를 사용하든 현대적 주기적 방법을 사용하든, 사려 깊은 학습률 스케줄링은 효율적으로 최적 해결책에 도달하는 데 필수적입니다. 좋은 초기화와 [정규화](/regularization-kor.html)와 결합하면, 이는 성공적인 딥러닝 훈련의 기초를 형성합니다.
