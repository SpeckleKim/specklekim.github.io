# 특성 공학: 특성 생성의 예술과 과학

특성 공학은 실제 머신러닝에서 가장 중요한 측면으로 자주 인용됩니다. 고품질 특성은 모델 성능을 극적으로 향상시킬 수 있으며, 알고리즘 진전을 능가합니다. 이 가이드는 특성 생성, 변환, 선택, 그리고 최선의 관행을 다룹니다.

## 특성 유형 이해하기

**수치 특성:**
- 연속: 무한 정확도의 실수 (가격, 온도, 높이)
- 이산: 정수 개수 (항목 수, 사용자 ID 개수)
- 서수: 정렬된 범주 (1-5 평점, 교육 수준)
- 척도 조정 필요 및 변환 가능

**범주 특성:**
- 명목: 내재적 순서 없음 (색상, 국가, 제품 범주)
- 서수: 자연스러운 순서 (낮음/중간/높음, 초급/중급/고급)
- 높은 기수: 많은 고유값 (도시명, 제품 SKU)
- 많은 알고리즘이 사용하기 전에 인코딩 필요

**시간 특성:**
- 시간 기반 패턴 (요일, 월, 계절성)
- 지속 시간 (마지막 구매 이후 시간, 계정 나이)
- 래그 특성 (전날 판매, 과거 값)
- 예측 및 시계열 분석에 중요

**텍스트 특성:**
- 특수화된 전처리 및 인코딩 필요
- Bag-of-words, TF-IDF, 또는 임베딩 접근
- 범위를 벗어나지만 점점 중요해짐

## 범주 특성 인코딩

**원-핫 인코딩:**
- 각 범주에 대한 이진 열 생성
- 예: 색상 {빨강, 파랑, 녹색} → [1,0,0], [0,1,0], [0,0,1]
- 차원성 대폭 증가
- 저기수 특성에 적합 (< 10개 범주)

**레이블 인코딩:**
- 각 범주에 정수 할당
- 예: {빨강, 파랑, 녹색} → {0, 1, 2}
- 차원성 감소
- 위험: 모델이 없는 곳에 순서 해석
- 트리 기반 모델이나 서수 특성에만 사용

**목표 인코딩 (평균 인코딩):**
- 범주를 그 범주의 평균 목표값으로 대체
- 예: 도시를 그 도시의 평균 주택 가격으로 인코딩
- 높은 기수 특성에 강력
- 위험: 신중하게 구현하지 않으면 목표 누출

**빈도 인코딩:**
- 범주를 데이터셋의 빈도로 대체
- 간단하고 차원성 유지
- 빈도가 예측적일 때 유용
- 다른 방법보다 정보 손실 적음

**서수 인코딩:**
- 순서 보존하여 정수 할당
- 서수 특성의 경우: 낮음=0, 중간=1, 높음=2
- 정렬된 데이터에 원-핫보다 더 자연스러움
- 내재적 순서 존중

## 특성 척도 조정 및 정규화

**표준화 (Z-점수):**
- 변환: (x - 평균) / 표준편차
- 결과: 평균=0, 표준편차=1
- 근사적 정규 분포 가정
- 이상치에 민감

**최소-최대 정규화:**
- 변환: (x - 최소) / (최대 - 최소)
- 결과: [0, 1] 범위의 값
- 값 간 관계 보존
- 이상치에 매우 민감 (단일 극값이 척도 조정 영향)

**견고한 척도 조정:**
- 변환: (x - 중앙값) / IQR
- 중앙값 및 사분위수 범위 사용
- 표준화보다 이상치에 덜 민감
- 극값을 가진 데이터에 좋음

**로그 변환:**
- 변환: log(x + 1) 또는 log(x)
- 우측 편향 분포의 편향성 감소
- 큰 값은 압축하고 작은 값 확장
- 가격, 소득, 개수 데이터에 유용

**척도 조정 시점:**
- 거리 기반 알고리즘에 항상 (KNN, K-평균, SVM)
- 그래디언트 기반 알고리즘에 (선형 회귀, 신경망)
- 트리 기반 모델에는 덜 중요 (척도 불변)
- 수렴 개선 및 특성 우월성 방지

## 특성 생성 및 변환

**다항식 특성:**
- 거듭제곱 및 상호작용 생성
- 예: x² 또는 x₁ × x₂
- 비선형 관계 포착
- 차원성 급속 증가
- 다항식 회귀에 유용

**빈닝 (이산화):**
- 연속을 범주로 변환
- 고정폭 빈 또는 분위수 기반 빈
- 잡음 및 이상치 영향 감소
- 정보 손실이지만 비선형 패턴 포착 가능

**집계 및 도메인 지식:**
- 여러 특성 결합: 총액 = 가격 × 수량
- 비율 생성: ROI = 수익 / 투자
- 도메인 전문성은 종종 강력한 특성 드러냄
- 고객_생명시간_가치 = 모든 구매 합
- 평균_주문_가치 = 총 지출 / 주문 수

**상호작용 특성:**
- 관련 특성 결합: 나이 × 소득
- 비선형 관계 포착
- 트리 모델을 극적으로 향상 가능
- 특성 개수에 따라 지수적 성장

**날짜에서의 시간 특성:**
- 요일, 월, 분기, 년
- 휴일 여부, 주말 여부
- 기준점으로부터의 일수
- 계절 패턴에 대한 순환 인코딩

## 특성 선택 방법

**필터 방법 (훈련 전):**
- 목표와의 상관관계: 상관도 높은 특성 선택
- 분산 임계값: 거의 상수인 특성 제거
- 통계 검정: 범주형에 카이제곱, 수치형에 ANOVA
- 빠르고 계산 비용이 저렴
- 특성 상호작용 무시

**래퍼 방법 (모델 훈련과 함께):**
- 재귀적 특성 제거: 반복적으로 특성 제거
- 전진 선택: 특성 없이 시작하여 최고 추가
- 후진 제거: 모두로 시작하여 최악 제거
- 계산 비용이 많지만 상호작용 고려
- 훈련 데이터에 과적합 위험

**포함 방법 (훈련 중):**
- 트리 모델의 특성 중요도
- L1 정규화 (라쏘)는 자연스럽게 특성을 0으로 조정
- XGBoost 특성 중요도
- 필터보다 낫고 래퍼보다 비용 적음
- 모델 특정이지만 종종 일반화 가능

**차원성 감소:**
- 주성분 분석 (PCA): 선형 결합
- 특성 추출로 더 적은 특성으로 총 분산 보존
- 차원성을 위해 해석 가능성 교환
- 매우 고차원 데이터에 유용

**모델의 특성 중요도:**
- 트리 모델은 직접 중요도 점수 제공
- SHAP 값은 각 특성의 예측 기여도 표시
- 순열 중요도: 섞을 때 성능 감소 측정
- 진정 예측적 특성 식별 도움

## 도메인 지식의 중요성

**왜 중요한가:**
- 주제 전문가는 데이터 관계 이해
- 훈련 전에 중복 또는 비관련 특성 식별 가능
- 강력한 특성 결합 제안
- 특성이 중요한 이유 해석

**도메인 지식 영향의 예:**
- 신용 위험: 소득보다 부채 대 소득 비율이 더 예측적
- 전자상거래: 패션에는 계절 패턴이 중요하지만 기본 상품에는 덜함
- 의료: 약물 상호작용은 복잡한 비선형 패턴 생성
- 제조: 장비 나이는 사용 시간보다 더 중요할 수 있음

**도메인 전문가와의 협업:**
- 데이터 수집 및 측정 프로세스 이해
- 비즈니스 규칙 및 제약 학습
- 인과 관계를 가진 프록시 변수 식별
- 공학 특성이 직관적으로 합리적임을 검증

## 자동화된 특성 공학

**Featuretools:**
- 관계형 데이터베이스에서 특성 생성
- 자동화된 집계 및 변환
- 복잡한 관계에 대한 깊은 특성 합성
- 구조화된 데이터에 수동 작업 감소

**자동화의 제한:**
- 도메인 전문성을 대체할 수 없음
- 일반 특성이 전문가 설계보다 약함
- 허위 상관관계 생성 위험
- 특성 개수 폭발로 선택 필요

**최선의 관행:**
- 자동화를 시작점으로 사용
- 도메인 전문가가 정제 및 검증
- 자동화된 + 수동 공학 결합
- 항상 비즈니스 논리에 대해 검증

## 특성 공학의 일반적 실수

**데이터 누출:**
- 훈련 데이터에서 미래 정보 사용
- 목표 인식 특성 생성 (적절한 검증 없이)
- 훈련/테스트 분할 전 전처리
- 재앙적: 개발에서 뛰어난 성능, 생산에서 실패

**과도한 공학:**
- 너무 많은 특성 생성
- 제한된 데이터로 복잡한 상호작용
- 훈련 데이터에 과적합
- 모델은 느리고 해석 어려움

**특성 상호작용 무시:**
- 각 특성을 고립으로 분석
- 강력한 결합 놓침
- 선형 모델에 특히 문제적

**일관되지 않은 전처리:**
- 훈련과 테스트에 다른 척도 조정
- 평균을 전체 데이터에서 계산 (테스트 포함)
- 올바른 워크플로: 훈련에서 적합, 테스트에 적용

## 특성 공학 워크플로

1. **데이터 이해:**
   - 분포 및 관계 탐색
   - 데이터 품질 문제 식별
   - 도메인 전문가와 상담

2. **정제 및 전처리:**
   - 누락된 값 처리
   - 이상치 수정 (또는 특성으로 인코딩)
   - 데이터 형식 표준화

3. **특성 생성:**
   - 도메인 주도 특성 생성
   - 수학적 변환
   - 상호작용 및 집계

4. **척도 조정 공학:**
   - 적절한 척도 조정 방법 적용
   - 범주 변수 인코딩
   - 훈련/테스트 간 일관성 보장

5. **특성 선택:**
   - 중복/비관련 제거
   - 적절한 선택 방법 사용
   - 모델 복잡성 균형

6. **검증:**
   - 보유 검증 세트에서 테스트
   - 데이터 누출 확인
   - 해석 가능성 보장
   - 기준선과 비교

## 결론

특성 공학은 깊은 도메인 지식이 머신러닝 실무를 만나는 곳입니다. 데이터 이해, 신중한 특성 생성, 개선 검증에 시간을 투자하세요. 알고리즘은 변하지만 좋은 특성은 여러 방법에 걸쳐 가치 있습니다. 간단하게 시작하고, 엄격하게 검증하며, 약한 알고리즘을 우수한 특성이 보조 알고리즘의 특성이 불가에서 능가한다는 것을 기억하세요.
