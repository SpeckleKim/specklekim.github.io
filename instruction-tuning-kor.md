# 지시 튜닝

## 사전학습에서 지시 따르기로

다음 토큰 예측만으로 학습한 대규모 언어 모델은 놀라운 능력을 보여주지만 기본 작업 따르기에서 종종 실패합니다. 모델은 셰익스피어 구절을 아름답게 완성할 수 있지만 "프랑스의 수도는 무엇입니까?"를 지시로 표현할 때 답변하는 데 어려움을 겪을 수 있습니다.

**지시 튜닝**은 이 격차를 해소합니다. 모델이 명시적 지시를 따르고 작업 특정 출력을 생성하는 것을 배우는 미세 조정 단계입니다. 단순히 다음으로 가능한 토큰을 예측하는 대신 모델은 작업을 이해하고 제약을 따르고 유용한 응답을 전달하는 것을 학습합니다.

### 사전학습만으로는 충분하지 않은 이유

원본 인터넷 데이터에 대한 사전학습은 압축을 최적화합니다—가능한 연속을 예측합니다. 이는 광범위한 지식을 구축하지만 지시 따르기를 직접 최적화하지 않습니다. 순전히 웹 텍스트로 학습한 모델은 다음을 할 수 있습니다:
- 질문을 답변 필요로 인식하지 못함
- 텍스트를 요약하기보다 계속함
- 암시적 작업 제약을 놓칠 수 있음
- 특수 형식 지시를 무시할 수 있음

지시 튜닝은 이 행동들을 명시적으로 가르칩니다.

## 지시 튜닝 데이터셋

지시 튜닝의 기초는 **(지시, 입력, 출력) 트리플의 신중하게 구성된 데이터셋**입니다.

### 데이터 형식

일반적인 지시 튜닝 예:
```
지시: 이 텍스트를 한 문장으로 요약합니다.
입력: 인공지능의 진화는 여러 단계를 통해 진행되었습니다...
출력: AI는 여러 단계를 통해 진화했으며 좁은 전문가 시스템에서
      더 일반적인 능력으로 발전했습니다.
```

또는 더 간결하게 (입력 없는 작업의 경우):
```
지시: 광합성이란 무엇입니까?
출력: 광합성은 식물이 빛 에너지를 화학 에너지로 변환하는 과정입니다...
```

### 데이터셋 규모 및 다양성

효과적인 지시 튜닝이 필요합니다:
- **수량**: 수천에서 수십만 개의 예 (일반적으로 1K-100K)
- **다양성**: 여러 작업 유형(요약, QA, 추론, 번역 등)에 걸친 예
- **품질**: 최소한의 오류로 고품질 출력
- **작업 변형**: 견고성을 가르치기 위해 같은 작업의 다양한 표현

## 주목할 만한 지시 튜닝 접근법

### FLAN (자연언어로 주석된 언어 모델 미세 조정)

Google의 FLAN 데이터셋은 수십 개의 작업 범주 전체에서 다양한 지시 예를 제공합니다. FLAN은 다음을 결합합니다:
- 원본 데이터셋 (수백 개의 기존 벤치마크)
- 데이터셋을 지시 형식으로 변환하는 템플릿
- 작업 특정 변형

**핵심 통찰**: 원본 Wikipedia에 대해 학습하는 대신 설정된 작업 컬렉션에서 유도된 형식화된 지시에 미세 조정합니다. 이것이 모델이 지시를 따르면서 원본 작업의 성능을 유지하는 것을 가르칩니다.

### InstructGPT

OpenAI의 InstructGPT는 인간 피드백을 통한 지시 튜닝이 유용성과 안전성을 크게 향상시킴을 입증했습니다:

1. **감독된 미세 조정 (SFT)**: 기본 모델을 고품질 지시 예에 미세 조정
2. **보상 모델 학습**: 인간이 선호하는 출력 예측 모델 학습
3. **강화 학습**: PPO를 사용하여 보상 최대화하도록 모델 업데이트

이 3단계 프로세스는 인간 피드백과 결합되어 더 정렬되고 능력 있는 모델을 생성했습니다.

## 지시 튜닝 데이터셋 생성

### 크라우드소싱 및 인간 주석

주석자를 고용하여 지시를 작성하고 고품질 출력 제공:
1. 주석자들이 다양한 지시 작성
2. 주석자들이 금 표준 출력 제공
3. 품질 관리가 낮은 품질 예 제거
4. 검증이 다양성 및 커버리지 보장

이는 노동 집약적이지만 고품질 데이터를 생성합니다.

### 기존 데이터셋에서 변환

많은 벤치마크 데이터셋(SQuAD, GLUE, SuperGLUE)을 지시 형식으로 변환할 수 있습니다:

**SQuAD를 지시 형식으로:**
```
지시: 통로를 기반으로 질문에 답합니다.
입력: 통로: [...] 질문: [...]
출력: [답]
```

**GLUE 감정 작업을 지시로:**
```
지시: 문장의 감정을 긍정 또는 부정으로 분류합니다.
입력: 이 영화는 정말 멋있었어요!
출력: 긍정
```

### Self-Instruct

순전히 인간 주석에만 의존하기보다 **self-instruct**는 모델 자체를 사용하여 지시와 출력을 생성합니다:

1. 인간이 작성한 지시의 작은 시드 세트로 시작
2. 모델을 사용하여 유사한 스타일로 새 지시 생성
3. 모델이 이 자체 생성 지시에 답변
4. 낮은 품질 출력 필터링; 학습 세트에 추가
5. 반복하여 데이터셋 확장

이는 다양성을 유지하면서 인간 주석 부담을 줄입니다.

## 템플릿 형식화

### 입력 및 출력 구조화

잘 설계된 템플릿은 모델이 작업 구조를 이해하는 데 도움이 됩니다:

**표준 형식:**
```
### 지시:
{지시}

### 입력:
{입력}

### 응답:
{출력}
```

**채팅 형식:**
```
인간: {지시}
{입력}
어시스턴트: {출력}
```

**Q&A 형식:**
```
Q: {지시}
A: {출력}
```

다양한 템플릿은 다양한 행동을 가르칩니다. 채팅 형식은 자연스럽게 대화 행동을 가르치며, Q&A 형식은 사실 작업에 적합합니다.

## 다중 작업 지시 튜닝

지시 튜닝은 단일 학습 실행에서 **다양한 작업 유형**을 결합할 때 가장 효과적입니다.

### 다중 작업이 중요한 이유

다양한 작업에 대해 학습하면:
- 단일 작업 분포에 과적합 방지
- 도메인 전체에서 적응 가르침
- 보이지 않은 작업으로의 일반화 개선
- 더 견고하고 유연한 모델 생성

### 작업 분배

실무자들은 일반적으로:
- 10-50+개의 서로 다른 작업 유형 포함
- 중요도 또는 빈도에 따라 작업 가중치
- 성능이 지연되는 작업 과다 샘플링
- 폭과 깊이 간 균형

잘 설계된 커리큘럼은 기초 작업(QA, 요약)과 전문 작업(추론, 코드 생성)을 혼합합니다.

## 지시 튜닝 원칙

### 명확하고 구체적인 지시

모델은 지시 품질에서 배웁니다. 효과적인 지시:
- 작업을 명시적으로 명시
- 제약을 명확하게 제공
- 도움이 될 때 예 제공
- 모호함 피함

**나쁜 예:** "AI에 대해 작성하십시오"
**더 나은 예:** "초보자용으로 적합한 기계 학습 소개 3단락을 작성하고 기술 용어가 없습니다."

### 출력 품질이 중요합니다

학습 출력의 품질은 모델 성능에 직접 영향을 미칩니다. 일반적인 관행:
- 고위험 도메인의 인간 전문가 사용
- 품질 관리 프로세스 구현
- 지식 기반 작업의 사실성 확인
- 스타일 및 형식화의 일관성 유지

### 규모 효율성

지시 튜닝은 놀랍도록 효율적입니다:
- 적절한 양(1K-10K 예)이 성능을 의미 있게 향상시킬 수 있음
- 작업당 더 많은 예보다 더 많은 작업 다양성 추가 종종 도움
- 잘 선별된 5K 예 데이터셋이 나쁘게 선별된 50K 데이터셋을 이깁니다

## 도전과제 및 고려사항

### 작업 유출

지시 튜닝 데이터가 평가 벤치마크의 예를 포함하면 성능 수치가 부풀려집니다. 이는 일반적인 함정입니다.

### 지시 따르기 vs 지식

지시 튜닝은 **작업 따르기**를 가르치며 지식을 가르치지 않습니다. 모델이 사전학습 중에 배우지 않은 사실을 가르칠 수 없습니다. 이것이 모델이 거짓 정보를 생성하기 위해 지시를 따르는 이유입니다.

### 보이지 않은 작업으로의 일반화

핵심 연구 목표: 다양한 작업에 대한 지시 튜닝이 완전히 새로운 작업에 대한 영점 성능을 가능하게 할까요? 초기 결과(InstructGPT, FLAN)는 그렇다고 시사하지만 일반화에는 한계가 있습니다.

### 계산 비용

지시 튜닝이 필요합니다:
- 고품질 데이터셋 생성 (비쌈)
- 완전 모델 미세 조정에 상당한 계산
- 모델/도메인당 신중한 하이퍼매개변수 조정

더 작은 모델은 더 큰 모델의 지시 따르기 능력과 일치하려면 더 많은 지시 튜닝 데이터가 필요할 수 있습니다.

## 새로운 기법

**프롬프트 튜닝**: 모델 가중치를 업데이트하는 대신 작업 특정 프롬프트 임베딩을 배웁니다. 더 효율적이지만 완전 지시 튜닝보다 덜 강력합니다.

**지시 반전**: 지시를 출력하기 위해 작성하는 대신 주어진 출력을 생성해야 하는 지시를 생성하는 것을 배웁니다. 모델이 배우는 것을 이해하는 데 유용합니다.

**조건부 학습**: 특정 작업이나 지시 유형에 대해 손실을 다르게 가중치화하여 특정 성능 목표에 최적화합니다.

지시 튜닝은 실용적이고 유용한 언어 모델을 구축하기 위해 필수가 되었습니다. 명시적으로 작업 따르기를 가르침으로써 실무자들은 원본 언어 모델을 능력 있는 어시스턴트로 변환합니다.
