# 마르코프 결정 과정: 강화학습의 기초

## 소개

마르코프 결정 과정(MDP)은 [강화학습](/rl-kor.html)의 기초가 되는 수학적 프레임워크를 제공합니다. [에이전트](/agent-kor.html)가 환경과 상호작용하여 관찰과 보상을 받는 순차 의사결정 문제를 형식화합니다. MDP를 이해하는 것은 RL 알고리즘을 파악하는 데 필수입니다.

## 주요 요소

**상태 (S):** 환경 구성의 완전한 설명입니다. [에이전트](/agent-kor.html)는 현재 상태를 관찰하고 의사결정해야 합니다. 상태 공간은 이산 또는 연속, 유한 또는 무한일 수 있습니다.

**행동 (A):** [에이전트](/agent-kor.html)가 사용 가능한 가능한 의사결정입니다. 각 상태에서 특정 행동이 합법적일 수 있습니다. 행동 공간은 에이전트의 제어 능력을 정의합니다.

**전이:** 결정론적 또는 확률적 동학입니다. 전이 함수 P(s'|s,a)는 현재 상태 s와 행동 a가 주어질 때 다음 상태 s'의 확률을 나타냅니다. 환경 동학은 종종 미지이므로 [에이전트](/agent-kor.html)는 경험에서 학습합니다.

**보상 (R):** 스칼라 피드백 신호입니다. 보상 함수 r(s,a)는 상태 s에서 행동 a의 즉각적 보상을 지정합니다. [에이전트](/agent-kor.html) 목표: 시간에 따른 누적 보상 최대화.

**할인 계수 (γ):** 미래 보상의 가중치입니다. 매개변수 0 ≤ γ ≤ 1은 즉각적 vs 장기 보상을 균형 있게 유지합니다. γ가 1에 가까우면 미래를 우선하고, 0에 가까우면 즉각적 보상을 우선합니다.

## 마르코프 성질

"미래는 현재가 주어질 때 과거와 무관하다":

```
P(S_{t+1}|S_t, A_t, S_{t-1}, ...) = P(S_{t+1}|S_t, A_t)
```

이 성질은 다루기 쉬운 계산을 가능하게 합니다. 상태는 모든 관련 과거 정보를 포함해야 하며, 그렇지 않으면 문제가 진정한 마르코프식이 아닙니다.

## 가치 함수

**상태 가치 함수 V(s):** 상태 s에서 시작하는 예상 누적 할인 보상:

```
V(s) = E[G_t | S_t = s] 여기서 G_t = R_{t+1} + γR_{t+2} + γ²R_{t+3} + ...
```

**행동 가치 함수 Q(s,a):** 상태 s에서 행동 a를 취하는 예상 누적 보상:

```
Q(s,a) = E[G_t | S_t = s, A_t = a]
```

두 함수 모두 효율적인 계산을 가능하게 하는 재귀 관계를 만족합니다.

## 벨만 방정식

가치 함수를 연결하는 기본 재귀 방정식:

**벨만 예상 방정식:**
```
V(s) = E[R_{t+1} + γV(S_{t+1}) | S_t = s]
V(s) = Σ_a π(a|s) Σ_{s',r} P(s',r|s,a)[r + γV(s')]
```

여기서 π는 정책(행동 선택 전략)입니다.

**벨만 최적성 방정식:**
```
V*(s) = max_a E[R_{t+1} + γV*(S_{t+1}) | S_t = s, A_t = a]
V*(s) = max_a Σ_{s',r} P(s',r|s,a)[r + γV*(s')]
```

이 방정식은 장기 보상을 즉각적 보상과 할인 미래 보상으로 분해합니다.

## 정책

정책 π는 행동 선택을 지정합니다: π(a|s) = 상태 s에서 행동 a의 확률.

**결정론적 정책:** π(a|s) ∈ {0,1}—항상 특정 행동 선택

**확률적 정책:** π(a|s) ∈ [0,1]—확률적 행동 선택

**정책 평가:** 주어진 정책 π에 대한 V^π 계산

**정책 개선:** 가치 함수 추정을 사용한 더 나은 정책 찾기

## 가치 반복

최적 가치 함수 계산:

**알고리즘:**
```
모든 s에 대해 V(s) = 0으로 초기화
수렴할 때까지 반복:
  각 상태 s에 대해:
    V(s) = max_a Σ_{s',r} P(s',r|s,a)[r + γV(s')]
```

벨만 최적성 방정식을 직접 풉니다. V*로의 수렴이 보장됩니다. 계산 비용: 반복당 O(|S|²|A|).

## 정책 반복

최적 정책 계산:

**알고리즘:**
```
정책 π를 무작위로 초기화
수렴할 때까지 반복:
  정책 평가: V^π 계산
  정책 개선: π' = greedy(V^π)
  If π' = π: 완료
  Else: π = π'
```

현재 정책 평가와 개선을 번갈아 가며 수행합니다. 일반적으로 가치 반복보다 빠른 수렴.

## 일반화된 정책 반복

가치 반복과 정책 반복을 통합하는 프레임워크:

1. **평가:** V를 현재 정책과 일치하도록 만들기
2. **개선:** 정책을 V에 대해 탐욕적으로 만들기

두 접근법 모두 최적 정책과 가치 함수로 나아갑니다.

## 실무 고려사항

**유한 vs 무한 범위:**
- 유한: 에피소드가 고정 스텝 후 종료
- 무한: 할인 계수가 유한합을 보장

**계속 vs 에피소드식:**
- 에피소드식: 자연스러운 에피소드 경계 (게임 종료, 작업 완료)
- 계속: 자연스러운 종료 없음

**모델 기반 vs 모델 프리:**
- 모델 기반: P(s'|s,a)와 R(s,a)를 알거나 학습
- 모델 프리: 경험에서 직접 학습

## 확장

**부분 관찰 가능 MDP (POMDP):** [에이전트](/agent-kor.html)가 실제 상태 대신 노이즈 신호를 관찰

**연속 상태/행동:** 무한 상태/행동 공간은 함수 근사 필요

**다중 에이전트:** 경합하거나 정렬된 목표를 가진 여러 [에이전트](/agent-kor.html)

## 결론

MDP는 상태, 행동, 전이, 보상을 통해 순차 의사결정을 형식화합니다. 벨만 방정식은 가치 함수의 재귀 구조를 드러냅니다. 가치 반복과 정책 반복은 최적 제어 알고리즘을 제공합니다. 이 기초는 이 핵심 개념을 바탕으로 하거나 확장하는 현대 RL 알고리즘 이해를 가능하게 합니다.
