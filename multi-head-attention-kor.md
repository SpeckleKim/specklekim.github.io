# 멀티헤드 어텐션

단일 헤드 [셀프 어텐션](/self-attention-kor.html)은 강력하지만, 멀티헤드 [어텐션](/attention-mechanism-kor.html)은 모델이 다양한 표현 부분공간에서 정보에 공동으로 주목하도록 합니다. 이 [어텐션 메커니즘](/attention-mechanism-kor.html)의 병렬화는 [트랜스포머](/llm-kor.html)를 실용적이고 효과적으로 만든 핵심 혁신 중 하나입니다.

## 왜 여러 어텐션 헤드를 사용하는가?

단일 [어텐션](/attention-mechanism-kor.html) 헤드는 하나의 부분공간에서 작동합니다. 멀티헤드 어텐션은 모델이:

1. **다양한 위치에 주목**: 일부 헤드는 근처 토큰에, 다른 헤드는 먼 토큰에 집중
2. **다양한 특징에 주목**: 문법 헤드 vs. 의미론 헤드 vs. 위치 헤드
3. **다양한 패턴 포착**: 단일 투영이 모든 관련 관계를 포착하지 못함
4. **중복성 제공**: 여러 관점이 부정확한 [어텐션](/attention-mechanism-kor.html) 패턴에 견딜 수 있게 함
5. **표현 용량 증가**: 계층 깊이 증가 없이 계층별 매개변수 증가

## 멀티헤드 메커니즘

h개 헤드와 d_model 차원에서, 각 헤드는 d_k = d_model / h 차원에서 작동합니다:

```
head_i = Attention(Q_i, K_i, V_i)
        = softmax(Q_i * K_i^T / sqrt(d_k)) * V_i

MultiHead(Q, K, V) = Concat(head_1, ..., head_h) * W_O
```

여기서:
- Q_i = Q * W_Q^i (i번째 헤드의 쿼리 투영)
- K_i = K * W_K^i (i번째 헤드의 키 투영)
- V_i = V * W_V^i (i번째 헤드의 밸류 투영)
- W_O는 출력 투영입니다 (d_model, d_model)

## 병렬 계산

멀티헤드 [어텐션](/attention-mechanism-kor.html)의 주요 장점은 **병렬 가능성**입니다:

- RNN과 달리 모든 h개 헤드가 동시에 계산됨
- 행렬 연산을 효율적으로 배치 처리 가능
- GPU는 병렬 행렬 곱셈에 최적화됨
- 현대 구현은 텐서 연산 사용: (batch, seq_len, h, d_k)

이 병렬 아키텍처는 대규모 데이터셋 학습을 가능하게 하여 NLP에 혁명을 일으켰습니다.

## 연결 및 투영

독립적인 [어텐션](/attention-mechanism-kor.html) 헤드를 계산한 후, 출력들이 연결됩니다:

```
concatenated = Concat(head_1, ..., head_h)  # shape: (seq_len, d_model)
output = concatenated * W_O
```

출력 투영 W_O는:
- d_model 차원으로 다시 매핑
- 서로 다른 헤드 간 관계 학습
- 다운스트림 계층이 멀티헤드 정보를 결합하게 함
- 헤드 표현 간 학습된 "믹서" 역할

## 다양한 헤드들은 무엇을 배우는가?

경험 연구에서 [어텐션](/attention-mechanism-kor.html) 헤드는 특화된 역할을 학습함을 보여줍니다:

- **위치 헤드**: 상대 토큰 위치와 거리 추적
- **문법 헤드**: 문법 구조 집중 (주어-동사, 수식어-명사)
- **의미론 헤드**: 장거리 내용 관계 포착
- **출력 헤드**: 이전 위치에 주목 (생성 준비)
- **드문 토큰 헤드**: 드물지만 중요한 토큰 특화
- **불용어 헤드**: 기능 단어와 관사 처리

이 특화는 명시적 감독 없이 나타나며 손실을 최소화하려는 학습의 자연스러운 결과입니다.

## 제거 연구 및 헤드 중요성

연구에서 다음을 보여줍니다:

1. **헤드 제거**: 많은 헤드가 학습 후 최소한의 영향으로 제거 가능
2. **중요도 vs. 중복**: 일부 헤드는 필수적, 다른 헤드는 중복적으로 보임
3. **위치 vs. 의미론**: 다양한 헤드 유형이 다양한 계층에서 중요
4. **헤드 기여도 변화**: 초기 계층은 더 문법적 패턴, 후기 계층은 의미론적
5. **중복성으로서의 견고성**: 중복 헤드는 분포 변화에 대한 보험 제공

[트랜스포머](/llm-kor.html)의 "로또 티켓 가설"은 많은 헤드가 중복적 로또 티켓이라고 제시합니다—학습 중 도움이 되지만 테스트 시간에는 엄격히 필요하지 않습니다.

## 헤드 수: 설계 선택

헤드 수는 다음으로 제약되는 하이퍼파라미터입니다:

- **d_model 나누어떨어짐**: d_model은 h로 나누어떨어져야 함
- **메모리와 계산**: 더 많은 헤드 = 순전파 당 더 많은 계산
- **d_k 크기**: 너무 작은 d_k는 각 헤드 용량을 제한
- **경험적 성능**: 기본 모델은 8-16개 헤드, 큰 모델은 12-16개 전형적

트레이드오프:
- 더 많은 헤드 → 더 나은 특화하지만 수렴 수익 감소
- 더 적은 헤드 → 빠르지만 덜 다양한 패턴 포착
- 매우 적은 헤드 (h=1) → 단일헤드 [어텐션](/attention-mechanism-kor.html)으로 축소
- 많은 헤드 → 각 헤드 용량 희석

## 헤드 간 정보 흐름

멀티헤드 [어텐션](/attention-mechanism-kor.html)은 "교차 헤드" 효과를 가능하게 합니다:

1. **암묵적 토론**: 다양한 헤드가 다양한 해석을 제안
2. **합의**: 출력 투영이 신뢰성으로 헤드 가중치를 학습
3. **오류 수정**: 한 헤드가 실패하면, 다른 헤드가 보상 가능
4. **앙상블 효과**: 약학습자가 결합하는 부스팅과 유사

이 [앙상블](/ensemble-methods-kor.html) 유사 속성은 [트랜스포머](/llm-kor.html)의 견고성과 일반화에 기여합니다.

## 어텐션 패턴 시각화

[어텐션](/attention-mechanism-kor.html) 가중치 시각화는 헤드 특화를 드러냅니다:

```
Head 1: 인접 토큰에 주목 (로컬 어텐션)
Head 2: 동사 전 명사에 주목 (문법)
Head 3: 관사-명사 쌍에 주목 (문법 역할)
Head 4: 장거리 의미론 어텐션
...
```

이 패턴은 데이터셋 전반에 유지되어 학습된 보편적 [어텐션](/attention-mechanism-kor.html) 전략을 시사합니다.

## 계산 복잡도

헤드별 멀티헤드 [어텐션](/attention-mechanism-kor.html) 복잡도:
- 쿼리/키/밸류 투영: O(h * d_model * d_k)
- [어텐션](/attention-mechanism-kor.html) 계산: O(h * n² * d_k)
- 출력 투영: O(h * d_v * d_model)

h개 헤드 추가가 계산을 h배로 곱하는 동안, 병렬 특성은 효율적 [GPU](/gpu-hardware-kor.html) 구현을 가능하게 하여 멀티헤드를 깊이 추가 대비 본질적으로 "무료"로 만듭니다.

## 비교: 단일 vs. 멀티헤드

| 측면 | 단일 헤드 | 멀티헤드 |
|------|---------|--------|
| 표현 부분공간 | 1 | h |
| 패턴 다양성 | 제한적 | 풍부 |
| 특화 | 없음 | 강함 |
| 견고성 | 취약 | 중복적 |
| 해석 | 단순 | 복잡하지만 통찰력 있음 |

## 현대적 변형

최근 연구는 다양한 대안을 탐구합니다:

- **그룹 쿼리 어텐션(GQA)**: 쿼리 헤드보다 적은 키/밸류 헤드 (메모리 감소)
- **멀티 쿼리 어텐션(MQA)**: 모든 쿼리 헤드에서 공유되는 단일 키/밸류 헤드
- **희소 멀티헤드**: 내용 기반 관련 헤드에만 주목
- **전문가 혼합**: 입력 기반 헤드 선택

이 변형들은 성능 유지 중 메모리를 감소시킵니다.

## 결론

멀티헤드 [어텐션](/attention-mechanism-kor.html)은 단일 투영 공간을 h개의 상호보완적 부분공간으로 변환하여 병렬 계산을 통한 풍부한 패턴 포착을 가능하게 합니다. 감독 없이 나타나는 특화된 헤드는 기울기 기반 학습이 자연스럽게 다양한 어텐션 전략을 어떻게 발견하는지를 보여줍니다. 이 설계 선택은 [트랜스포머](/llm-kor.html)를 실용적으로 만들고 효율적 병렬화와 다양한 작업 전반의 강력한 실증적 성능을 가능하게 하는 데 중추적이었습니다.

멀티헤드 [어텐션](/attention-mechanism-kor.html)의 특화, 중복성, 병렬화 역할을 이해하는 것은 모델 설계, 최적화, 해석 가능성 작업에 필수적입니다.
