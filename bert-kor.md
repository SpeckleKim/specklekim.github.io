# BERT와 마스크드 언어 모델링: 패러다임 전환

BERT(트랜스포머의 양방향 인코더 표현)는 우리가 NLP에 접근하는 방식을 근본적으로 변화시켰습니다. 특정 작업을 위해 모델을 훈련하는 대신, BERT는 레이블 없는 데이터에 대한 대규모 사전훈련 후 간단한 미세조정으로 거의 모든 하위 작업에서 최첨단 결과를 달성할 수 있음을 보여주었습니다. 이 패러다임—사전훈련, 그 다음 미세조정—은 현대 NLP의 기초가 되었습니다.

## 양방향성 통찰

이전 언어 모델은 단방향이었습니다:

```
단방향 (좌에서 우로):
"은행 매니저가 승인했다..."
 ↑   ↑                  ↑
"매니저"를 예측할 때, "은행"만 봄
우측 맥락 누락: "승인"
```

BERT는 양방향 맥락을 사용합니다:

```
양방향:
"은행 매니저가 승인했다..."
 ↑   ↑       ↑            ↑
"매니저"를 인코딩할 때:
- 좌측 맥락 봄: "은행"
- 우측 맥락 봄: "승인했다..."
- 훨씬 더 풍부한 표현!
```

**도전:** 표준 언어 모델링 (다음 토큰 예측)은 본질적으로 단방향입니다. 양방향 모델을 어떻게 훈련합니까?

BERT의 돌파구: 마스크드 언어 모델링.

## 마스크드 언어 모델링 (MLM)

다음 토큰 예측 대신, BERT는 토큰을 무작위로 마스크하고 이를 예측합니다:

```
원본: "안녕하세요 어떻게 지내세요 날씨는 좋나요"

마스크: "안녕하세요 어떻게 [MASK] 날씨는 [MASK] 나요"
        (토큰의 15%가 무작위로 마스크됨)

작업: 마스크된 토큰 ("지내세요", "좋은")을 예측
```

**마스킹 전략:**
- 선택된 토큰의 15%를 마스킹
- 선택된 토큰의 80%를 [MASK]로 교체
- 10%를 무작위 토큰으로 교체
- 10%를 변경하지 않음

**왜 100% [MASK]가 아닌가?**
- 모델이 의미 있는 표현을 학습하려면 실제 토큰을 봐야 함
- 80% [MASK], 10% 무작위, 10% 변경 없음은 강력한 훈련 신호 제공
- 이는 모델이 단순히 [MASK] 토큰을 암기하는 것을 방지

**훈련 목표:**
```
손실 = -log P(마스크된_토큰 | 맥락)
```

주변 맥락 (좌측과 우측 모두)이 주어진 원본 토큰 예측.

## NSP: 다음 문장 예측

BERT는 문장 수준 의미론을 학습하기 위해 두 번째 사전훈련 작업을 추가합니다:

```
예제 1 (IsNext):
문장 A: "그 남자가 가게에 갔다."
문장 B: "그는 빵을 샀다."
레이블: IsNext (B가 문서에서 A를 따름)

예제 2 (NotNext):
문장 A: "그 남자가 가게에 갔다."
문장 B: "펭귄은 날지 못하는 새다."
레이블: NotNext (무작위 문장)
```

**아키텍처:**
```
입력: [CLS] sent_A_토큰 [SEP] sent_B_토큰 [SEP]
       ↓       ↓              ↓        ↓
[CLS]는 특수 분류 토큰
[SEP]는 문장 경계 표시

출력: [CLS] 표현이 이진 분류기로 공급
      IsNext 또는 NotNext
```

**하위 작업에 대한 기여:**
NSP는 문장 관계가 필요한 작업에 도움:
- 질의 응답 (질문이 맥락과 관련)
- 자연어 추론 (전제가 가설과 관련)
- 의역 탐지 (의미론적으로 유사한 문장)

나중의 연구 (RoBERTa)는 NSP의 기여가 미미하고 많은 모델이 이 작업을 삭제했음을 보여주었습니다.

## BERT 아키텍처

BERT는 트랜스포머 인코더 스택입니다:

```
입력 계층 (토크나이제이션 + 임베딩)
    ↓
트랜스포머 인코더 블록 1 (다중 헤드 어텐션 + FFN)
    ↓
트랜스포머 인코더 블록 2
    ↓
... (다양한 BERT 크기를 위한 12-24 블록)
    ↓
출력 계층 (각 토큰 위치에 대한 임베딩)
```

**핵심 구성 요소:**

1. **워드피스 토크나이제이션**: ~30K 어휘를 가진 부분단어 토크나이제이션
2. **토큰 임베딩**: 각 토큰 → 밀집 벡터
3. **세그먼트 임베딩**: 어느 문장 (A 또는 B)인지 나타내는 토큰 수준 임베딩
4. **위치 임베딩**: 절대 위치 (1에서 최대_길이)
5. **트랜스포머 계층**: 자기 어텐션과 피드포워드 네트워크
6. **계층 정규화**: 훈련 안정화

**BERT 크기:**
- BERT-Base: 12 계층, 768 숨겨진 차원, 12 어텐션 헤드 (~110M 매개변수)
- BERT-Large: 24 계층, 1024 숨겨진 차원, 16 어텐션 헤드 (~340M 매개변수)

## 하위 작업을 위한 미세조정

BERT의 사전훈련은 일반 언어 이해를 학습합니다. 작업 특화 미세조정은 하위 애플리케이션에 이를 적응시킵니다.

**텍스트 분류:**
```
[CLS] 문장 토큰 [SEP]
        ↓
     BERT 인코딩
        ↓
   [CLS] 표현
        ↓
   선형 분류기
        ↓
   클래스 확률
```

간단: 단순히 [CLS] 토큰 표현 위에 선형 계층 추가.

**명명 개체 인식:**
```
[CLS] 고양이가 앉았다 [SEP]
        ↓
     BERT 인코딩
        ↓
토큰별 출력 (토큰당 하나의 레이블)
```

각 토큰 위치의 출력을 태깅에 사용합니다.

**질의 응답:**
```
[CLS] 질문 [SEP] 문단 [SEP]
        ↓
     BERT 인코딩
        ↓
답변의 시작 및 끝 위치 예측
```

BERT는 문단 내 답변을 스팬하도록 학습합니다.

**미세조정 접근법:**
1. 사전훈련 BERT 가중치로 초기화
2. 작업 특화 계층 추가
3. 레이블 작업 데이터로 훈련 (일반적으로 5-10 에포크)
4. 학습률: 작음 (2e-5 ~ 5e-5) 사전훈련 지식 보존

## BERT 변형과 개선

### RoBERTa (Facebook/Meta)

RoBERTa는 다음을 통해 BERT를 개선했습니다:
- **더 긴 훈련**: 더 많은 사전훈련 스텝
- **더 큰 배치**: 더 나은 기울기 추정
- **NSP 제거**: 작업은 최소 이점 제공
- **향상된 마스킹**: 동적 마스킹 (에포크당 마스크 패턴 변경)
- **정제 데이터**: 더 높은 품질 사전훈련 말뭉치

**결과:** 지속적으로 원본 BERT를 능가합니다.

### ALBERT (Google)

ALBERT는 효율성을 위해 매개변수 공유를 도입했습니다:
- **인수분해 임베딩**: 별도의 임베딩과 숨겨진 차원 (더 작은 임베딩, 전체 크기 숨겨짐)
- **계층 간 매개변수 공유**: 모든 트랜스포머 계층이 매개변수 공유
- **결과:** BERT-Large보다 10배 적은 매개변수, 유사한 성능

### DeBERTa (Microsoft)

DeBERTa는 어텐션을 개선했습니다:
- **분리된 어텐션**: 별도의 위치와 콘텐츠 어텐션
- **향상된 마스크 디코더**: 마스크된 토큰의 더 나은 처리
- **결과:** GLUE 벤치마크의 최신 (RoBERTa 능가)

### ELECTRA (Google)

ELECTRA는 생성 대신 판별적 사전훈련을 사용합니다:
- **생성기**: 작은 언어 모델이 손상된 텍스트 생성
- **판별기**: 더 큰 모델이 교체된 토큰 감지
- **결과:** 더 효율적인 사전훈련; 더 적은 계산으로 더 나은 성능

다른 변형은 도메인 특화를 해결합니다:
- **SciBERT**: 과학 논문
- **ClinicalBERT**: 의료 텍스트
- **FinBERT**: 금융 문서
- **LegalBERT**: 법률 문서

## BERT가 NLP를 혁신한 이유

### 1. **보편적 표현**

단일 사전훈련 모델이 작업 전체에서 작동합니다. 작업 특화 아키텍처 필요 없음.

### 2. **전례 없는 규모**

대규모 레이블 없는 텍스트에 대한 사전훈련 (위키백과 + 책 말뭉치, 3.3B 토큰)이 깊은 언어 구조를 학습합니다.

### 3. **간단한 미세조정**

최소 작업 특화 아키텍처. 종종 사전훈련 표현 위에 선형 계층만.

### 4. **뛰어난 성능**

BERT는 다음에서 최첨단을 달성했습니다:
- GLUE (일반 언어 이해 평가) 벤치마크
- SQuAD (질의 응답)
- NER (명명 개체 인식)
- 감정 분석
- 그리고 많은 것

### 5. **접근성**

BERT의 오픈소스 공개는 NLP 연구를 민주화했습니다. 누구나 자신의 작업에 미세조정할 수 있었습니다.

## 한계와 진화

### BERT 한계:

1. **단방향 생성**: 좌에서 우로 텍스트를 효율적으로 생성할 수 없음
2. **고정 시퀀스 길이**: 최대 512 토큰 (확장 가능하지만)
3. **독립성 가정**: 마스크된 위치가 독립적으로 예측됨 (공동이 아님)
4. **정적 임베딩**: 토큰별 표현이 시퀀스 내에서 변경되지 않음

### 후속 개선:

이 한계들이 다음 개발에 동기를 부여했습니다:
- **GPT 모델**: 생성을 위한 생성 (단방향) 사전훈련
- **T5**: 모든 NLP를 텍스트-투-텍스트로 취급, 생성을 더 잘 처리
- **XLNet**: 순열 언어 모델링 (더 유연한 맥락)
- **ELECTRA, DeBERTa**: 더 효율적이고 효과적인 사전훈련

## 현대적 영향

2024년에, BERT의 직접적 영향은 새로운 모델 (GPT-4, Claude, Llama)이 지배함에 따라 감소했습니다. 하지만:

1. **개념적 기초**: 사전훈련-후-미세조정은 여전히 지배적 패러다임
2. **인코더 전용 모델**: 효율성 중요 애플리케이션에서 여전히 사용
3. **다국어 변형**: mBERT, XLM-R은 비영어에서 여전히 가치
4. **교육적 가치**: 현대 NLP를 이해하기 위해 BERT 이해 필수

## 결론

BERT는 마스크드 언어 모델링에 대한 사전훈련 후 작업 특화 미세조정이 뛰어난 결과를 달성할 수 있음을 보여줌으로써 NLP를 근본적으로 변화시켰습니다. 핵심 통찰—양방향 맥락, 대규모 사전훈련, 간단한 미세조정—은 현대 언어 모델의 청사진이 되었습니다.

특정 BERT 모델이 현재 프로덕션에서 덜 일반적이지만, 이것이 설정한 패러다임은 2024년 NLP에 중앙에 있습니다. BERT를 이해하는 것은 현대 언어 모델, 그들의 능력, 그들의 한계를 이해하기 위해 필수적입니다. BERT의 유산은 특정 아키텍처에 있지 않고, NLP의 미래가 특정 작업을 위해 미세조정된 대규모 사전훈련 모델에 속한다는 것을 보여주었다는 점에 있습니다.
